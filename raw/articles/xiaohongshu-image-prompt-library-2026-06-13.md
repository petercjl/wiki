# 小红书生图提示词库

根据当前小红书知识库提炼，图片生成的核心不是“好看”本身，而是让图片承担笔记逻辑：搜索关键词、品类识别、场景代入、痛点呈现、证据展示、收藏价值。

## 1. 通用提示词公式

```text
小红书风格商业种草图片，竖版 3:4，手机端高可读性，主体是【产品/人物/场景】，面向【目标人群】，表达【痛点/结果/场景/卖点】，画面有明确品类信号和使用场景，真实生活感，干净明亮，轻商业感，柔和自然光，高质感产品摄影，画面中心突出一个主视觉，背景简洁不杂乱，适合小红书封面，留出上方或中间文字区域，中文大标题排版清晰醒目，标题文案：【封面文案】
```

## 2. 封面图提示词

### 痛点型封面

```text
小红书爆款封面风格，竖版 3:4，真实生活场景，主体是【产品/问题场景】，面向【人群】，画面突出【具体痛点】，情绪真实但不夸张，明亮干净，高对比但不刺眼，主体清晰，背景少量生活细节，手机端一眼可读，顶部大字标题：【痛点标题】，副标题：【解决方向】，红色或黑色重点字强调，避免杂乱拼贴
```

适合文案：

```text
【人群】别再乱买【品类】了
用了才知道，原来【痛点】可以这样解决
【场景】总踩坑？先看这张
```

### 结果型封面

```text
小红书种草封面，竖版 3:4，主体是【产品】和【使用后结果】，画面展示清晰可见的变化或理想状态，真实自然，不像广告大片，干净背景，柔和光线，质感高级，突出一个核心结果：【结果】，中文标题醒目，标题：【结果型文案】，保留文字安全区，移动端清晰
```

适合文案：

```text
坚持【周期】，终于把【问题】改善了
【人群】想要的【结果】，重点看这几个
这才是【品类】该有的效果
```

### 场景型封面

```text
小红书生活方式种草图，竖版 3:4，真实【场景】中正在使用【产品】，人物或手部动作自然，场景有代入感，适合【人群】，画面明亮、温暖、干净，产品可见且不生硬，突出【场景需求】，标题区域清晰，中文大标题：【场景标题】，整体真实、有用、可收藏
```

适合文案：

```text
【场景】真的离不开它
【人群】的【场景】好物清单
这个【场景】用【品类】才舒服
```

### 清单/攻略型封面

```text
小红书攻略封面，竖版 3:4，信息图和实拍结合，主体是【品类/产品集合】，排版清楚，有 3-5 个要点模块，白底或浅色背景，红色重点标注，黑色清晰中文标题，适合收藏，手机端可读，标题：【攻略标题】，包含关键词：【品类词】、【人群词】、【场景词】，整体像真实博主整理的实用攻略
```

适合文案：

```text
【品类】选购避坑清单
【人群】买【品类】先看这 5 点
【场景】好用的【品类】怎么选
```

### 对比测评型封面

```text
小红书测评封面，竖版 3:4，左右或上下对比构图，展示【产品A】和【产品B/使用前后/不同方案】，清晰标签区分，真实测评感，桌面自然光，背景简洁，重点突出差异，中文标题醒目：【对比标题】，有小标签：【适合人群】、【核心差异】，画面可信、克制、不夸张
```

适合文案：

```text
【产品A】和【产品B】到底怎么选
【品类】别只看价格，差别在这里
实测后才发现【核心差异】
```

## 3. 组图提示词结构

### 图 1：封面

```text
小红书封面图，竖版 3:4，一个清晰主体，包含【品类关键词】、【核心结果】、【目标人群/场景】，手机端强可读性，标题：【封面文案】，画面真实、明亮、干净、有记忆点
```

### 图 2：痛点/前情

```text
小红书笔记第二张图，展示【人群】在【场景】中遇到【痛点】的真实状态，生活化、自然、不摆拍，画面带一点问题感但不负面，配少量中文标注：【痛点短句】，让用户产生代入
```

### 图 3：使用场景

```text
小红书场景图，展示【产品】在【具体场景】中的自然使用方式，人物动作真实，产品露出清晰但不硬广，光线自然，画面干净，表达【场景价值】，适合种草
```

### 图 4：细节/证据

```text
小红书产品细节图，微距或近景展示【关键细节/材质/成分/结构/功能点】，干净桌面，高质感，清晰标注 2-3 个卖点，文字短而具体，标题：【证据标题】，让用户相信这个产品真的解决问题
```

### 图 5：对比/方法

```text
小红书对比信息图，展示【使用前后/不同产品/选择标准】的差异，排版清晰，浅色背景，红色重点标记，3-5 个对比点，手机端可读，适合收藏，标题：【对比或方法标题】
```

### 图 6：总结/购买理由

```text
小红书总结图，竖版 3:4，清单式排版，归纳【适合谁】、【解决什么】、【怎么选】、【注意事项】，风格干净实用，像真实博主整理的收藏卡片，标题：【总结标题】，不强卖，带轻提示
```

## 8. 知识库增强提取版

来源依据：`10_图片语义转译.md`、`03_内容选题与爆文方法.md`、`04_人群洞察与需求分析.md`、`02_账号定位与品牌策略.md`。

核心原则：
- 图片先服务笔记目标：点击、收藏、搜索承接、种草转化。
- 封面主图要浓缩痛点/痒点，并与标题信息统一。
- 信息流素材偏真实场景、兴趣感、弱营销；搜索素材偏答案感、利益点、选购决策。
- 卖点不要说明书式罗列，要转译成人群、场景、问题和结果。
- 标题、主图、关键词要共同覆盖品类词、人群词、场景词、功效词。

### 信息流素材提示词

适合“闲逛”场景，用户没有明确搜索需求，要靠画面抓住视线。

```text
小红书信息流种草封面，竖版 3:4，真实生活场景，主体是【产品/人物/空间】，目标人群是【人群】，画面呈现【具体场景】里的自然使用瞬间，弱营销感，不像广告摆拍，有兴趣点和轻微反差，产品露出清晰但不硬，融入【热点/季节/生活方式】元素。画面明亮干净，一个主视觉，背景有少量真实生活细节，手机端一眼能看懂。中文大标题：【场景化标题】，标题突出【人群词/场景词/痛点词】，保留文字安全区。负面提示：不要电商详情页，不要强促销，不要品牌硬广，不要复杂拼贴，不要密集小字。
```

### 搜索承接素材提示词

适合“找答案”场景，用户已经有问题或购买意向，图片要直接给答案。

```text
小红书搜索承接封面，竖版 3:4，主题是【品类/产品】，目标用户正在搜索【关键词】，画面直接回答【痛点/选购问题】，包含清晰的产品/方案展示和 3-5 个可读要点。风格真实、干净、可信，像专业博主整理的实用攻略，标题：【答案型标题】，副标题：【利益点/避坑点】，画面中出现【品类词】、【人群词】、【场景词】或【功效词】标签，手机端高可读性，适合收藏。负面提示：不要空泛氛围图，不要只拍美图不讲信息，不要错误中文，不要小字过多。
```

### 三类卖点转译模板

兴趣型卖点：

```text
小红书兴趣种草图，竖版 3:4，主题是【产品】，突出【有趣/差异化/有记忆点的卖点】，画面有轻微反差或惊喜感，让用户愿意停留。场景是【具体生活场景】，人物状态自然，产品与场景融合，标题：【没想到/原来/谁懂】式表达，整体真实、有记忆点、弱营销感。
```

功能型卖点：

```text
小红书功能解释图，竖版 3:4，主题是【产品功能/成分/功效】，画面用细节特写、局部放大、前后对比或步骤示意展示【功能如何解决痛点】。背景干净，文字标注 2-3 个关键证据点，标题：【痛点】看这个【功能点】，风格可信、克制、专业但不冰冷。
```

场景型卖点：

```text
小红书场景种草图，竖版 3:4，主题是【产品】在【使用场景】中的真实体验，目标人群是【人群】，画面突出【场景需求】和【使用后的好处】，人物动作自然，产品露出不突兀。标题：【人群】在【场景】真的需要它，整体生活化、真实代入、可收藏。
```

### 爆文封面点击率模板

```text
小红书爆文首图，竖版 3:4，封面主题是【选题】，核心人群是【人群】，主图直接浓缩【痛点/痒点/结果】，标题和画面信息统一。画面包含一个强主视觉，关键词醒目，中文大标题：【标题】，重点词：【人群词/功效词/场景词】，画面不要复杂，第一眼能判断这篇笔记解决什么问题。风格真实利他，像真实博主经验分享，不像品牌广告。负面提示：不要说明书式卖点罗列，不要大量品牌名，不要营销感过强，不要牛皮癣式排版。
```

### 关键词预埋型提示词

用于让生图天然覆盖搜索词。

```text
小红书关键词封面，竖版 3:4，围绕【核心品类词】生成一张适合搜索承接的笔记首图。画面中自然体现【品牌/产品词】、【特征词】、【类目词】、【场景词】、【人群词】中的 3-4 类信息。目标人群是【人群】，需求是【需求/痛点】，场景是【场景】。标题：【包含关键词的标题】，标签文字：【关键词1】、【关键词2】、【关键词3】。画面真实、清晰、可信，像用户正在找答案时会点开的封面。
```

### 评论区转化感图片模板

```text

```
