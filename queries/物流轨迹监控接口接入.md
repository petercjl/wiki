# 物流轨迹监控接口接入

当用户询问“如何读取订单物流轨迹”“旺店通能不能监控揽收/签收”“快递100实时查询怎么接”“发货后无揽收/在途超时/派件超时怎么判断”时，先读：

1. [[domains/电商运营/30-ERP与系统工具/01-旺店通开放平台API/03-旺店通订单类API使用手册|旺店通订单类 API 使用手册]]
2. [[domains/电商运营/30-ERP与系统工具/02-快递100实时查询API|快递100实时查询 API]]

## 标准路线

1. 从旺店通读取订单、出库单、物流公司和运单号。
2. 将旺店通 `logistics_name/logistics_type_name` 映射为快递100 `com` 编码。
3. 用快递100实时查询接口读取 `state`、`data[].time`、`data[].status`、`data[].statusCode`。
4. 按业务 SLA 输出异常表：未及时发货、发货后无轨迹、待揽收过久、在途未更新、派件后未签收、疑难件、拒签/退回。

## 默认约束

- 不读取或展示收件人信息。
- 不开启 `needCourierInfo`。
- 同一运单至少间隔半小时再查。
- 不在 wiki、日志、Excel 中保存快递100密钥。
