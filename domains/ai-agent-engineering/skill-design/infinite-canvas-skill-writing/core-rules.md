# 无限画板 Skill 写作核心规则

## 核心定位

无限画板中的 skill 是给无限画板 AI 读取和执行的单文件 Markdown 说明，不是 SealSeek/OpenClaw 的工具型 skill。

必须牢记三条边界：

1. 无限画板 skill 通常只有一个 `skill.md`。
2. 不写脚本、Shell、Python、外部命令或本地目录执行逻辑。
3. 不把 SealSeek Skill 的目录结构、工具调用方式、README 交付规范强行套入无限画板。

## 交互顺序

默认顺序是：

```text
理解需求 → 输出规划 → 等用户确认 → 生成完整 skill.md
```

除非用户明确说“不要规划，直接生成”，否则不应在第一轮直接输出完整 skill。

规划阶段至少说明：

- 用户目标
- 输入内容
- 输出内容
- 涉及能力
- 复杂度
- 核心流程
- 工具策略
- Prompt 结构
- 需要用户确认的点

## 文件内容边界

生成的 `skill.md` 应是 Markdown 指令文档，推荐结构：

```markdown
# Skill 中文名称

## 触发条件
## 核心目标
## 输入要求
## 执行流程
## 工具选择规则
## Prompt 模板
## 输出规范
## 设计约束
## 参数建议
```

复杂 skill 可以增加：

- 批次规则
- 产品保真规则
- 文案入图规则
- QA / 质检标准
- 失败与回退机制
- 多轮优化协议

## 默认模型规则

除非用户明确指定，否则必须显式固定模型：

| 任务 | 默认模型 |
| --- | --- |
| 生图 / 图生图 / 图片编辑 | `gpt-image-2` |
| 文生视频 / 图生视频 / 参考视频生成 | `doubao-seedance-2-0` |

禁止写“让系统自动选择模型”或提供一堆模型选项让无限画板自行判断。

## 产品与事实边界

电商视觉类 skill 必须区分三类内容：

- 可以优化：背景、场景、光线、构图、氛围、文案层级、利益表达。
- 必须保留：产品主体、品类、结构、材质、比例、关键部件、用户给定事实。
- 禁止编造：检测报告、认证、专利、销量、评价、医生推荐、功效承诺、未提供功能。

有产品参考图时，必须强调：产品以参考图为准，产品主体不能变。

## 自主判断原则

当用户只说“优化、重做、提升、营销优化、详情页优化”且输入图片足够识别时，skill 应先自主诊断，不应第一轮追问过多。

允许先问的情况：

1. 产品完全无法识别。
2. 关键文字无法识别且决定卖点判断。
3. 用户要求严格匹配某平台/品牌规范但未提供规范。
4. 无法区分风格参考图和待优化图。
5. 必需风格图缺失。

## 批量处理原则

多图任务应分批处理：

- 每张图单独读取、分析、生成和质检。
- 不把多张待优化图混合成一张生成任务。
- 大批量任务应限制单批数量，历史 skill 常用每批最多 4 张。
- 每批完成后询问是否继续下一批。

## 质量闭环

生成型 skill 不应只到“调用工具”为止，还要规定生成后检查：

- 产品是否变形。
- 风格是否符合。
- 文案是否真实、清楚、有层级。
- 图位任务是否完成。
- 是否存在虚构事实或夸大承诺。
- 是否需要重新生成、局部编辑或让用户确认。
