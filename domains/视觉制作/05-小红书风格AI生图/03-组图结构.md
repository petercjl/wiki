# 小红书组图结构

小红书组图可以按“点击 → 代入 → 种草 → 信任 → 收藏 → 转化”的顺序设计。源文件给出的基础模型是 6 张图。

## 6 图任务模型

| 图位 | 任务 | 画面内容 | 用户心理 |
| --- | --- | --- | --- |
| 图 1 | 封面 | 品类关键词 + 核心结果 + 人群/场景 | 我为什么点开 |
| 图 2 | 痛点/前情 | 目标人群在真实场景中遇到问题 | 这说的是我 |
| 图 3 | 使用场景 | 产品自然进入具体使用场景 | 原来可以这样用 |
| 图 4 | 细节/证据 | 材质、结构、成分、功能、局部特写 | 这个东西可信 |
| 图 5 | 对比/方法 | 前后对比、不同方案、选择标准 | 我知道怎么判断了 |
| 图 6 | 总结/购买理由 | 适合谁、解决什么、怎么选、注意什么 | 值得收藏/考虑购买 |

## 图 1：封面

```text
小红书封面图，竖版 3:4，一个清晰主体，包含【品类关键词】、【核心结果】、【目标人群/场景】，手机端强可读性，标题：【封面文案】，画面真实、明亮、干净、有记忆点
```

## 图 2：痛点/前情

```text
小红书笔记第二张图，展示【人群】在【场景】中遇到【痛点】的真实状态，生活化、自然、不摆拍，画面带一点问题感但不负面，配少量中文标注：【痛点短句】，让用户产生代入
```

## 图 3：使用场景

```text
小红书场景图，展示【产品】在【具体场景】中的自然使用方式，人物动作真实，产品露出清晰但不硬广，光线自然，画面干净，表达【场景价值】，适合种草
```

## 图 4：细节/证据

```text
小红书产品细节图，微距或近景展示【关键细节/材质/成分/结构/功能点】，干净桌面，高质感，清晰标注 2-3 个卖点，文字短而具体，标题：【证据标题】，让用户相信这个产品真的解决问题
```

## 图 5：对比/方法

```text
小红书对比信息图，展示【使用前后/不同产品/选择标准】的差异，排版清晰，浅色背景，红色重点标记，3-5 个对比点，手机端可读，适合收藏，标题：【对比或方法标题】
```

## 图 6：总结/购买理由

```text
小红书总结图，竖版 3:4，清单式排版，归纳【适合谁】、【解决什么】、【怎么选】、【注意事项】，风格干净实用，像真实博主整理的收藏卡片，标题：【总结标题】，不强卖，带轻提示
```

## 使用规则

- 如果只生成封面，优先读取 [[domains/视觉制作/05-小红书风格AI生图/02-封面图类型|小红书封面类型库]]。
- 如果生成整组图，先按本页分配每张图的任务，再到 [[domains/视觉制作/05-小红书风格AI生图/07-Prompt模板|小红书生图 Prompt 模板库]] 取模板。
- 如果产品卖点还停留在说明书语言，先做 [[domains/视觉制作/05-小红书风格AI生图/04-卖点转译|小红书卖点转译]]。
