# 04｜图片工作流：提示词结构与参考控制

AI 图片工作流不是直接写一句提示词让工具碰运气，而是先让 AI 帮你理解参考，再由人定义框架，最后用工具修正质感和细节。

## 第一步：精准获取提示词

很多生成结果不符合需求，是因为团队没有告诉 AI “需要提取什么”。课程建议先把参考图拆解成核心元素，再让 AI 按资深美术指导的方式反推提示词。

提示词反推时，要明确要求 AI 分析底层风格约束，而不是只描述画面表层物体。重点包括：

- 色彩范围：主色、辅助色、冷暖、饱和度、明度。
- 光影关系：柔光、硬光、逆光、侧光、自然光、棚拍光。
- 材质质感：金属、玻璃、布料、木质、皮革、食物、液体等。
- 构图结构：中心构图、对角线、留白、前中后景、人物和产品比例。
- 情绪气质：高级、松弛、科技、自然、童趣、运动、专业等。

如果有多张参考图，应让 AI 提炼共同核心元素，避免误把某一张图的偶然道具当成必须元素。

## 第二步：搭建结构框架

AI 的随机性适合用来生成多个构图骨架。团队可以尝试不同模型和工具，把多张参考图的核心元素重组，再由人选择最终框架。

这一阶段不要急着修细节。正确顺序是先判断：

1. 构图是否能承接商品或人物。
2. 场景是否符合品牌调性。
3. 画面是否能支撑主卖点。
4. 图位是否适合主图、详情页、内容图或海报。

如果出现人物崩坏、场景不对、产品比例不准，可以先优化关键词；如果问题是局部结构不符合要求，也可以手动修改草图后再喂给 AI。

## 第三步：优化质感与修正细节

构图成立后，再进入质感润色和局部修复。课程提到，线稿也可以结合参考调性生成商业大片；不满意的局部可以用 SD、即梦等工具修复。

细节生成程度建议控制在 50-60 左右。开得太高，AI 会过度重绘，破坏原本构图和品牌约束；开得太低，又可能修不动问题区域。

## 工具选择原则

偏创意生成时，可以使用非细节控制类 AI，让模型给出更多意外构图和氛围。

需要固定构图时，应使用细节控制类 AI，并管理参考图权重。课程中提到，参考图权重在 40%-60% 往往更合适：太低会不像参考，太高会丧失 AI 的重新组织能力。

AI 生成图无法直接商用时，也不要急着丢弃。它仍可以作为拍摄构图、光影、道具、场景和合成素材的参考。

## 输出检查

一张 AI 商业图进入下一步前，要检查：

1. 是否符合品牌符号、颜色、版式和调性。
2. 产品信息是否真实，没有误导性结构和功效。
3. 主卖点是否被看见，而不是只追求氛围。
4. 是否能和同一系列其他图保持一致。
5. 是否需要实拍产品、人物或局部素材来补真实度。

## 产品参考图保真：不要把视觉识别结果写成产品重绘提示词

在商品图生图、详情页场景图、品牌场景图中，产品参考图的作用是锁定商品本体。提示词的任务是描述场景、光线、构图、版式、品牌气质和文案，而不是用文字重新描述产品结构。

一个常见错误是：Agent 先识别参考图中的产品细节，再把识别结果原样写进生图提示词。这样看似是在强调“产品不能变”，实际会让生图模型进入“按文字重新画商品”的模式，导致产品变形。

### 典型案例

某次包袋详情页生图中，原参考图里的包有奶白色帆布、卡其色提手、小熊印花和磁扣结构。Skill 自动生成的提示词写了：

```text
正面两颗黑色磁扣，正面三只小熊印花的内容、大小、位置关系与排布完全一致……
```

结果模型把“正面两颗黑色磁扣”理解成外侧正面必须画出两颗扣子，从而新增或移动扣件。问题不在单个词写错，而在于：Skill 把“产品视觉识别结果”当成了“生图 Prompt 描述”。

### 根因

产品保真有两种完全不同的写法：

| 写法 | 表现 | 风险 |
|---|---|---|
| 结构复述型 | 描述商品有几颗扣子、几只图案、几条线、几层纹理、什么位置 | 模型按文字重绘商品，容易新增、删减、移动结构 |
| 参考锁定型 | 要求结构数量、位置、比例、可见关系严格以参考图为准 | 模型更倾向于保留参考图，不用文字重构商品 |

因此，商品图生图时，产品细节可以用于内部理解，但不应以细节复述的方式进入最终生图提示词。

### 正确规则

进入生图提示词的产品保真内容，应写成“锁定规则”，而不是“细节描述”。

推荐写法：

```text
商品本体严格以参考图为准。包身轮廓、提手、印花、扣件、开口比例等可见结构不得新增、删减、移动或重新设计。扣件、图案、提手等局部结构的数量、位置、大小和可见关系均以参考图为准。不要根据文字重新绘制商品结构。
```

不推荐写法：

```text
奶白色厚帆布材质，梯形托特包轮廓，宽口比例，两侧卡其色提手，正面两颗黑色磁扣，正面三只小熊印花……
```

### 产品保真 Prompt 分层

在 Agent 或 Skill 中，应把产品相关信息分为三层：

| 信息层 | 用途 | 是否进入生图 Prompt |
|---|---|---|
| 产品识别信息 | 给 Agent 理解商品，例如扣件、印花、提手、图案、孔位、纹理 | 不直接进入，尤其不要写具体数量、位置、形状 |
| 产品锁定规则 | 约束模型不新增、不删减、不移动、不重绘 | 可以进入，应以“以参考图为准”表达 |
| 场景与版式信息 | 描述咖啡厅、桌面、光线、构图、标题位置、品牌色等 | 可以进入，是生图 Prompt 的主要内容 |

### 细节结构的安全表达

如果必须提到产品局部结构，只能使用关系锁定语句：

```text
扣件的数量、位置、大小和可见关系严格以参考图为准，不新增、不删减、不移动。
```

不要写成：

```text
正面两颗黑色磁扣。
```

同类规则适用于：

- 包袋：扣子、提手、拉链、印花数量和排布。
- 鞋服：鞋带孔、鞋底纹路、拼接线、口袋位置。
- 灯具：灯珠数量、支架结构、灯罩纹理。
- 收纳盒：开口、盖子、卡扣、格子数量。
- 玩具：眼睛、耳朵、四肢、角色图案。
- 家纺：纹理、缝线、褶皱、图案排布。

### Agent / Skill 设计要求

设计生图 Skill 时，应避免让“不可变要素”自动扩写成细节描述。推荐把不可变要素表改成以下结构：

| 不可变要素 | 锁定方式 | 是否允许进入生图 Prompt |
|---|---|---|
| 扣件 | 数量、位置、大小、可见关系以参考图为准，不新增、不删减、不移动 | 只允许写锁定规则，不允许写具体数量 |
| 印花 / 图案 | 内容、比例、位置关系以参考图为准，不重新绘制、不重排 | 只允许写锁定规则 |
| 提手 / 把手 | 形态、连接关系、比例以参考图为准 | 只允许写锁定规则 |
| 开口 / 轮廓 | 开口比例和整体轮廓以参考图为准 | 只允许写锁定规则 |

一句话原则：

> 产品细节只用于内部识别，进入生图 Prompt 的只能是“以参考图为准、不新增、不删减、不移动”的锁定规则。

## Agent 使用提示

请先读取品牌定位、目标人群、图位任务、产品卖点和参考图，再按“反推参考图约束 -> 生成构图方向 -> 选择框架 -> 局部修复 -> 商业检查”的顺序输出图片方案。涉及商品本体时，不要把产品识别细节直接写成生图提示词；应把细节转换为参考图锁定规则。
