# 制片周期与AI资产管理

来源锚点：Ch5 The Production Cycle，以及 Ch4 中数字工具、素材库、预演工具部分。

## 制片周期的导演任务

Katz 的制片周期观点是：可视化必须进入真实协作，而不是停留在纸上。传统拍摄中，导演要与制片、美术、摄影、演员、剪辑和现场条件不断协商。AI 视频中，协作对象换成了 Agent、图像模型、视频模型、剪辑工具、素材库和平台规范。

| 传统变量 | AI 视频变量 |
| --- | --- |
| 演员表演 | 人物一致性、表情、动作稳定性 |
| 摄影执行 | 模型对景别、机位、运镜的响应 |
| 美术落地 | 参考图一致性、风格漂移 |
| 道具/服装 | 产品结构、Logo、材质准确度 |
| 现场限制 | 模型时长、画幅、参考图数量、成本 |
| 剪辑余地 | coverage 备选镜头和反应镜头 |

## AI 制片资产结构

推荐一个项目用下面结构：

```text
00-brief/
01-reference/
02-product-character/
03-visual-bible/
04-storyboard/
05-first-frames/
06-video-generations/
07-coverage/
08-edit/
09-final/
10-postmortem/
```

## 命名规则

镜头文件名应包含镜头编号、任务、景别、运动和版本：

```text
S01_world_wide_static_v03.png
S03_friction_medium_tracking_v02.mp4
S05_product_insert_static_v04_pick.mp4
S08_endcard_centered_v01.png
```

这样后期能快速判断一条素材为什么存在，而不是面对一堆随机生成文件。

## coverage 管理

传统拍摄会为重要动作准备多角度 coverage。AI 视频也应该这么做，尤其是高风险镜头。

| 风险 | coverage 类型 |
| --- | --- |
| 手部变形 | 手部不入镜、背影、插入产品静帧、POV |
| 产品结构不准 | 产品静态特写、参考图锁定、局部切镜 |
| 人物身份漂移 | 背影、侧脸、遮挡、统一服装道具 |
| 复杂运镜失败 | 拆成两个静止或慢运动镜头 |
| 多人混乱 | 主体减少、景别拉远、剪反应镜头 |

## AI 视频制片流程

```text
brief
-> 案例检索
-> 视觉圣经
-> 镜头任务表
-> 分镜/首帧
-> 单镜头生成
-> coverage 生成
-> 粗剪
-> 问题回修
-> 成片
-> 案例库回写
```

## 质量检查

- 文件名是否能看出镜头任务。
- 每个镜头是否有 prompt、首帧和生成版本记录。
- 关键产品镜头是否有 coverage。
- 失败素材是否记录失败原因。
- 成片后是否回写可复用案例。
