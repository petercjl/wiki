# AI视频控制资产系统：身份板、故事板、导演台与素材库

AI 视频稳定性不是靠把提示词越写越长，而是靠一组可被模型读取的控制资产，把“谁在画面里、镜头怎么拍、空间怎么摆、画面什么风格、商业结构怎么复刻”分配给不同材料。提示词只是最后的执行说明，不应该承担全部控制任务。

这套系统把原来的“故事板铁三角”扩展为五类控制资产：

| 控制资产   | 核心任务                 | 典型失控问题             |
| ------ | -------------------- | ------------------ |
| 身份板    | 锁定角色、产品或主体的一致性细节     | 脸变、服装变、产品变形、主体身份漂移 |
| 故事板    | 锁定镜头顺序、景别、动作和镜头运动    | 镜头乱跳、节奏断裂、动作方向不清   |
| 导演台    | 锁定人物在空间中的位置、朝向、距离和机位 | 多人关系混乱、左右不清、人物站位错误 |
| 商业素材库  | 锁定可替换的产品、人物、服装、场景资产  | 复刻对标视频时素材混杂、产品不稳定  |
| 风格/效果图 | 锁定光影、色彩、质感和整体审美      | 每个镜头都能看，但整条片子风格散   |

## 视觉证据原则

短视频知识入库时，凡是知识点依赖画面形态，截图本身就是知识单元。角色身份板到底是三视图、行动格、电影感角色母卡，故事板箭头如何标注镜头运动，导演台如何摆人物位置，这些信息不能只靠 ASR 转写恢复。

正文中的图片使用 vault 相对嵌入，Agent 和 Obsidian 读取本页时应把图片与文字规则一起理解。完整截图索引见 `_meta/extraction-notes/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence-index.md`。

## 1. 身份板：把角色变成可读取的档案

身份板不是装饰图，而是给视频模型读取的角色档案。它应该把角色的外貌、服装、姿态、表情、身份气质和关键微特征固定下来，让后续故事板和视频生成知道“这始终是同一个人/同一个产品”。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7643404202686020081-identity-board-motion-details.jpg]]

这张身份板的关键不只是“有一只猫”，而是把猫的多姿态、眼睛、爪子、毛发、局部特写和动作状态压进同一张参考图。模型在生成跳舞、旋转、近景、远景时，就有一套稳定的主体档案。

身份板可以分层使用：

| 身份板类型   | 解决的问题             | 适用场景            |
| ------- | ----------------- | --------------- |
| 角色三视图   | 正面、侧面、背面和服装结构稳定   | 动作简单、角色一致性优先    |
| 面部三视图   | 五官、表情、近景识别稳定      | 口播、情绪镜头、脸部特写    |
| 多姿态行动格  | 肢体动作、运动姿态和局部细节稳定  | 跳舞、打斗、运动、复杂动作   |
| 产品款式图   | 产品外观、材质、细节、角度稳定   | 商品广告、产品替换、功能展示  |
| 电影感角色母卡 | 气质、职业、服装、表情、世界观稳定 | 短剧角色、品牌角色、高质感广告 |

制作身份板时，先判断“需要稳定的变量”是什么。角色视频重点是脸、服装和动作姿态；商品视频重点是产品外形、材质、比例和使用角度；短剧和品牌角色还要补气质、身份和世界观线索。

## 2. 故事板：把视频拆成可执行镜头

故事板不是最终大片截图，而是视频的施工图。它要告诉模型每个镜头拍什么、从哪里看、人物如何运动、镜头如何运动，以及前后镜头如何衔接。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7643404202686020081-storyboard-arrows-00-02-18.jpg]]

这张故事板的价值在于结构信息：

- 12 格分镜把视频拆成可执行镜头。
- 每格有景别、动作或镜头说明，例如低机位全景、特写、仰拍慢动作、全景远去。
- 红色箭头表达人物或肢体运动。
- 蓝色箭头表达摄影机运动或视角方向。
- 橙色箭头表达光线、高光或轮廓光方向。

故事板越靠近控制层，越应该清楚、简化、低歧义。它不需要像最终成片那样精美；它需要让模型少猜。过度精细的故事板容易被模型当成最终参考图，导致身份、风格和镜头任务混在一起。

故事板至少要包含：

| 元素 | 作用 |
| --- | --- |
| 镜头编号 | 保持视频顺序和节奏 |
| 景别 | 控制远景、中景、近景、特写 |
| 主体位置 | 控制人物或产品在画面中的方位 |
| 动作方向 | 控制主体移动、旋转、视线和肢体 |
| 镜头运动 | 控制推、拉、摇、移、跟拍、俯仰 |
| 节奏提示 | 控制停顿、爆发、转场和收束 |

## 3. 导演台：把空间关系摆出来

当镜头涉及打斗、多人对话、群像调度、复杂走位或镜头角度时，单靠故事板仍然可能不够。模型不一定理解“左边是哪边、距离有多远、人物面向哪里”。导演台解决的是空间关系。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7651618528713871355-director-table-position.jpg]]

导演台的做法是：

```text
准备空场景
-> 建立可旋转/可截图的空间
-> 放入人物或产品模型
-> 调整位置、前后景、朝向、动作、机位
-> 截图成为分镜位置参考图
-> 再交给视频模型执行
```

导演台适合补足三类问题：

| 问题 | 导演台怎么解决 |
| --- | --- |
| 多人位置混乱 | 把每个人放在明确的前景/中景/远景位置 |
| 朝向错误 | 直接设置面对镜头、侧对镜头、背对镜头 |
| 机位不清 | 通过截图固定低机位、俯拍、侧拍、越肩等角度 |

简单单人镜头可以只用故事板；多人互动、打斗和群像镜头应增加导演台截图。导演台本质上是 [[domains/视觉制作/06-AI视频/03-导演式镜头设计方法|导演式镜头设计方法]] 的可视化前置层。

## 4. 商品素材库：产品一致性与正确性优先

电商视频的故事板不能只服务“好看”，还要先服务“产品正确”。商品图一旦变形、颜色错误、结构错位、材质失真，视频再有节奏也不能用。短视频中提到的“素材资产库”对电商尤其关键：它决定模型是不是在用一个清楚、干净、完整的产品档案生成视频。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7647464806470962447-asset-library.jpg]]

素材库不是临时找几张图，而是在做视频前先明确“要替换什么”。对标视频可以复刻结构，但不能搬运素材；安全且可长期复用的做法是复刻镜头语言、内容节奏和成交结构，再替换为自己的产品、人物、服装和场景。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7647464806470962447-product-asset-library-00-02-10.jpg]]

商品素材库至少分四类：

| 素材层 | 应准备什么 | 作用 |
| --- | --- | --- |
| 产品素材 | 白底图、细节图、使用场景图、不同角度图 | 防止产品变形，保证卖点和外观正确 |
| 人物素材 | 正面、侧面、表情、动作、服装上身图 | 保持出镜人物一致 |
| 场景素材 | 室内、街景、店铺、货架、办公室、家庭空间 | 统一真实感和商业语境 |
| 服装/道具素材 | 面料细节、模特上身、道具角度 | 避免穿搭和配件漂移 |

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7647464806470962447-product-asset-library-00-02-20.jpg]]

对电商而言，产品素材应优先做到“清楚、干净、可校验”：

| 产品素材要求 | 具体含义 | 为什么重要 |
| --- | --- | --- |
| 白底图 | 正面主图、轮廓清楚、无复杂背景 | 给模型稳定的产品外形和比例 |
| 细节图 | 材质、纹理、接口、功能部件、卖点特写 | 防止关键卖点在视频中丢失或生成错误 |
| 使用场景图 | 产品在真实使用场景中的位置和关系 | 让产品进入场景时不突兀 |
| 不同角度图 | 正面、侧面、背面、俯视、局部角度 | 支撑镜头切换和运镜，不让产品凭空变形 |
| 尺寸/比例参照 | 与手、人、桌面、空间或包装的比例关系 | 避免产品大小不合理 |
| 禁止变化项 | logo、颜色、结构、按钮、接口、材质、包装信息 | 保证商品合规和品牌一致性 |

视频中的关键判断是：产品图越清楚，产品生成视频越不容易变形；人物图越稳定，人脸和动作越不容易紊乱；场景图越明确，后面视频真实感越强。换句话说，视频质量很大程度取决于前面给模型的参考信息质量。素材越像一个完整资产库，生成结果越像一个完整项目，而不是临时拼接出来的 AI 视频。

商业复刻的正确顺序是：

```text
找对标视频
-> 拆前三秒钩子、产品展示、镜头切换、人物动作、结尾收束
-> 反推口播脚本、分镜画面、运镜节奏、镜头提示词
-> 准备自己的产品/人物/场景/服装素材库
-> 用 Image2 生成故事板
-> 用 Seedance 2.0 等视频模型按故事板执行
-> 后期对齐原视频节奏、转场和音乐
```

这里复刻的是结构，不是内容。真正要迁移的是镜头语言、节奏和成交路径。

商品视频交给视频模型前，prompt 也要围绕产品正确性写，而不是只写画面好看：

```text
保持产品外观稳定；
保持产品颜色、材质、结构、logo 和关键卖点不变；
人物动作自然；
镜头节奏参考故事板；
不要让产品变形、漂移或变成其他款式。
```

如果是服装类视频，还要单独准备服装素材。视频中的例子把服装拆成服装白底图、模特上身图、面料细节图、不同角度图；这对服装电商非常重要，因为服装不仅是“产品”，还同时涉及版型、面料、上身效果和人体动态。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7647464806470962447-product-asset-library-00-02-40.jpg]]

商品素材库可以作为电商 AI 视频生产的第一步，而不是故事板之后的附属动作。没有素材库，故事板只是在组织镜头；有了素材库，故事板才能把“正确的产品”放进“正确的镜头”。

## 5. 产品操作资产：让模型理解真实机构

商品素材库解决“产品长什么样”，但不一定解决“产品怎么用”。很多电商产品的卖点来自真实机构动作，例如保温杯的双饮杯盖、厨具的开合结构、收纳用品的折叠步骤、小家电的拆装和切换模式。静态多角度图只能让模型看见部件，不能保证它理解部件之间的先后关系、旋转轴、按压方向和最终状态。

双饮杯盖就是典型例子。静态主图大概能看出它有两种饮用/打开状态：一种像翻盖后露出的直饮口或大口饮水状态，另一种像打开吸嘴/吸管口的饮用状态。但仅靠图片，Agent 和视频模型很难稳定判断：

- 哪个盖片先打开。
- 哪个按钮或卡扣控制哪个口。
- 哪个部件是吸嘴，哪个部件是直饮口。
- 翻盖的旋转方向和终点位置。
- 手应该从哪里按、掀、拨、推。
- 两种饮用方式之间是否互斥，还是同一杯盖的两个独立出口。

这类产品不应该只建“产品外观素材库”，还要建“产品操作资产库”。

| 操作资产 | 解决的问题 | 推荐形式 |
| --- | --- | --- |
| 操作演示视频 | 真实展示机构如何运动 | 3-8 秒短视频，固定机位，手部清楚 |
| 操作分解帧 | 拆解起始、按压、翻起、饮用、复位 | 从视频截取 4-8 张关键帧 |
| 部件标注图 | 标清按钮、卡扣、吸嘴、直饮口、转轴 | 图片上加箭头和文字标签 |
| 状态对照图 | 闭合态、方式 A 打开、方式 B 打开、完全打开 | 一张图内并列展示 |
| 禁止误读清单 | 说明不能生成错的地方 | 例如吸嘴不能变成把手，直饮口不能消失 |

视频参考会比单张图片更好，但前提是它要足够“教学化”。如果视频是氛围片、手挡住杯盖、镜头晃动、动作太快，模型仍然只会学到表面动作。理想的视频参考应该像产品说明书：

```text
镜头固定在杯盖上
-> 手指按下/拨开第一个开关
-> 第一种饮用口完全打开并停留 1 秒
-> 关闭复位
-> 手指打开第二个饮用口
-> 第二种饮用口完全打开并停留 1 秒
-> 关闭复位
```

给 Agent 或视频模型使用时，不要只说“参考这个视频”。要把操作事实写成可检查的产品协议：

```text
产品操作协议：
1. 该杯盖有两种饮用方式。
2. 方式 A：打开直饮口，饮用口位于杯盖前侧，翻盖向后抬起。
3. 方式 B：打开吸嘴/吸管口，吸嘴从杯盖顶部抬起，形成倾斜饮用口。
4. 两种方式都属于同一个杯盖结构，不要生成两个不同杯盖。
5. 保持杯身、杯盖、提绳、装饰牌、颜色和比例不变。
6. 生成视频时手部动作只负责开合，不要遮挡关键部件。
```

如果工具支持视频参考，就把操作视频作为“机构动作参考”；如果只支持图片参考，就用操作视频先抽帧，做成“操作故事板 + 部件标注图”。核心目标不是让模型看见更多图片，而是让它理解产品的状态机：

```text
闭合态
-> 方式 A 打开态
-> 方式 A 饮用态
-> 复位
-> 方式 B 打开态
-> 方式 B 饮用态
-> 复位
```

这个层级应该并入商品素材库。对电商 AI 视频来说，产品一致性不只是外观一致，还包括操作一致、机构一致和卖点一致。

一次双饮杯盖视频参考测试验证了这个判断。抖音收藏中的一条保温杯视频不仅展示杯身和材质，还用近景连续说明了两种开盖方式：一边打开是直饮口/咖啡口，另一边通过卡扣打开隐藏式吸管。它比静态主图更适合作为操作资产，因为画面同时保留了手指作用点、翻盖方向、吸管抬起方式和最终打开状态。

![[raw/videos/douyin-cup-open-reference-test-2026-06-28/visual-evidence/00-06-5-direct-drink-open.jpg]]

![[raw/videos/douyin-cup-open-reference-test-2026-06-28/visual-evidence/00-09-0-straw-latch-open.jpg]]

对这类视频，不应只保存成“参考视频”。更好的入库方式是把它拆成一个可复用的操作协议：

```text
产品：双饮保温杯。
方式 A：翻开杯盖一侧，露出直饮口/咖啡口。
方式 B：打开卡扣或吸嘴结构，露出隐藏式吸管。
关键约束：两个饮用口属于同一个杯盖，不要生成两个杯盖；手部动作必须围绕杯盖开合，不要遮挡饮用口；视频生成时要保持杯身图案、杯盖颜色、提绳和装饰牌一致。
证据：raw/videos/douyin-cup-open-reference-test-2026-06-28/
```

## 6. 电影感角色身份板：从三视图升级为角色母卡

传统三视图能锁定角色结构，但对角色气质、职业身份、服装层次、表情和故事感表达较弱。短剧、品牌角色和高质感广告更适合使用电影感角色身份板，也可以理解为导演选角时看的角色母卡。

![[raw/assets/douyin-storyboard-identity-board-corroboration-2026-06-28/visual-evidence/7649315926764827322-cinematic-identity-board.jpg]]

电影感角色母卡应该包含：

- 角色正面或半身主视觉。
- 不同姿态和表情。
- 服装、材质、配饰和色彩信息。
- 与角色身份相关的场景或道具线索。
- 能传达气质的光影、镜头距离和构图。

它不替代三视图，而是补足三视图缺少的“角色丰满度”。如果任务是稳定建模或简单动作，三视图足够；如果任务是短剧人物、品牌代言角色、故事广告主角，就需要角色母卡。

## 7. 风格/效果图：统一整条片子的审美

风格/效果图负责光影、色彩、材质、画面质感和整体氛围。故事板管结构，身份板管主体，导演台管空间，素材库管可替换资产，风格图管最终画面应该长成什么样。

风格图不应和身份板、故事板冲突。它最好回答：

- 画面是写实、动画、手绘、广告大片还是纪实？
- 光线是柔光、硬光、逆光、霓虹、自然光还是棚拍？
- 色彩是冷调、暖调、低饱和、高对比还是品牌色？
- 质感是胶片、商业摄影、短剧、3D、动漫还是产品广告？

## 8. Prompt 的位置：执行说明，而不是主控制器

当控制资产已经准备好，prompt 应该变短、变具体、变一致。它只补充：

- 当前镜头的核心动作。
- 当前镜头的情绪和节奏。
- 必要的稳定性要求，例如保持人物一致、产品不变形、镜头节奏一致。
- 不能从图中读出的特殊限制。

不要在 prompt 里重复身份板已经给出的角色细节，也不要重复风格图已经给出的审美词，更不要写出和故事板箭头、导演台位置冲突的指令。

## 9. 场景选择模板

| 使用场景 | 必选资产 | 可选资产 |
| --- | --- | --- |
| 单人短镜头 | 身份板、故事板、短 prompt | 风格图 |
| 多镜头角色短片 | 身份板、故事板、风格图、短 prompt | 导演台 |
| 打斗/多人对话/群像 | 身份板、故事板、导演台、场景参考 | 风格图 |
| 商品广告 | 产品款式图、商业素材库、故事板、风格图 | 人物身份板、导演台 |
| 机构操作演示 | 商品素材库、操作演示视频、操作分解帧、部件标注图 | 故事板、手部动作参考 |
| 对标视频复刻 | 对标拆解、素材库、Image2 故事板、视频模型 prompt | 后期节奏表 |
| 短剧角色设定 | 电影感角色母卡、故事板、场景图 | 三视图、导演台 |

## 10. 失败模式与修正

| 失败模式 | 表现 | 优先修正 |
| --- | --- | --- |
| 角色漂移 | 脸、服装、身份不稳定 | 补身份板或角色母卡 |
| 产品变形 | 产品比例、材质、细节跑偏 | 补产品款式图和产品素材库 |
| 操作错误 | 开盖方式、按钮、卡扣、饮用口、转轴关系不对 | 补操作视频、操作分解帧和部件标注图 |
| 镜头乱 | 景别、动作、镜头方向不对 | 改故事板和箭头标注 |
| 空间乱 | 多人位置、左右关系、距离错误 | 增加导演台截图 |
| 风格散 | 单帧能看，整条片子不统一 | 补风格/效果图 |
| prompt 污染图像 | 文字和图像参考互相打架 | 缩短 prompt，只保留执行说明 |
| 对标复刻像拼贴 | 产品/人物/场景不统一 | 先建素材库，再生成故事板 |

## 11. Agent 使用模板

当 Agent 帮用户制作 AI 视频时，不应直接从文字剧本跳到视频 prompt。应先输出控制资产计划：

```text
1. 视频目标：
2. 场景类型：单人 / 多人 / 商品 / 短剧 / 对标复刻
3. 需要稳定的变量：
4. 身份板类型：
5. 素材库清单：
6. 产品操作资产：是否需要操作视频、分解帧、部件标注图：
7. 故事板格数：
8. 每格镜头任务：
9. 每格景别、主体位置、动作方向：
10. 人物运动箭头：
11. 镜头运动箭头：
12. 是否需要导演台：
13. 风格/效果图方向：
14. 单镜头短 prompt：
15. 生成前 QA：
```

生成前 QA：

- 是否有身份板或产品款式图？
- 故事板是否清楚表达了镜头顺序、景别和运动？
- 是否有箭头区分人物运动、镜头运动和光线方向？
- 复杂空间关系是否有导演台截图？
- 商业视频是否已经建立产品、人物、服装、场景素材库？
- 产品机构或使用方式是否已经用操作视频/分解帧说明？
- 风格图是否统一了光影、色彩和质感？
- prompt 是否只做执行说明，没有和图片资产冲突？
