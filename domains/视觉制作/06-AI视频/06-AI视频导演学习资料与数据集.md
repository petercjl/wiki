# AI视频导演学习资料与数据集

本页记录公开资料、GitHub 项目、论文、数据集和推荐书单，用于补充 [[domains/视觉制作/06-AI视频/03-导演式镜头设计方法|导演式镜头设计方法]]、[[domains/视觉制作/06-AI视频/04-商业短视频摄影与运镜语法|商业短视频摄影与运镜语法]] 和 [[domains/视觉制作/06-AI视频/05-AI视频导演Prompt模板|AI视频导演 Prompt 模板]]。

## 调研结论

AI 视频导演知识不应该只从 prompt 博客里学。更稳的来源组合是：

```text
电影导演/摄影基础
-> 商业广告分镜和镜头任务
-> AI 视频模型 prompt 官方示例
-> 论文 benchmark 的镜头维度
-> 公开视频/电影片段数据集
-> 自己拆片形成案例库
```

近年的 AI 论文和数据集把电影镜头拆成了可标注维度：景别、构图、机位、镜头运动、光线、色彩、焦段、剪辑关系。这些维度正好可以变成商业短视频的分镜字段和 prompt 字段。

## 公开网页资料

| 资料 | 用途 | 适合怎么学 |
| --- | --- | --- |
| StudioBinder shot size guide | 景别、镜头大小、镜头选择 | 用来建立景别词表和“景别服务叙事”的意识 |
| StudioBinder camera movement / storyboard movement | 运镜类型、分镜中如何表达运动 | 用来把运镜写进分镜表，而不是只写进 prompt |
| StudioBinder mise-en-scene | 场面调度、画面内元素 | 用来理解服装、道具、空间、光线和表演都属于导演控制 |
| Runway Camera Terms, Prompts & Examples | AI 视频中的摄影术语和 prompt 示例 | 用来观察模型如何响应镜头术语 |
| Kling / Eachlabs / LetsEnhance AI camera movement guides | AI 视频运镜 prompt 经验 | 只作为执行层参考，不作为导演思维来源 |
| Utah Film Center visual storytelling material | 入门镜头、构图、情绪表达 | 适合做中文速查表和训练题 |

## GitHub 和数据集

| 项目 | 内容 | 对 AI 商业短视频的价值 |
| --- | --- | --- |
| PRIS-CV/CineTechBench | 电影摄影技术理解与生成 benchmark，覆盖景别、角度、构图、运镜、光线、色彩、焦段 | 可作为“镜头语言维度表”的来源 |
| Vchitect/ShotBench | 电影语言理解 benchmark，覆盖 8 个摄影维度，并提供 ShotQA / ShotVL 方向 | 适合学习如何把镜头画面转成问答和标签 |
| MovieNet | 大规模电影理解数据集，含镜头尺度、镜头运动等标注 | 适合理解电影镜头分类维度 |
| PardoAlejo/MovieCuts | 剪辑类型识别数据集和代码 | 适合研究跨镜头剪辑关系，不只是单镜头 |
| robincourant/the-exceptional-trajectories | E.T. camera trajectory 数据集 | 适合研究镜头轨迹如何和人物运动同步 |
| robincourant/DIRECTOR | 基于 E.T. 的文本到镜头轨迹生成方法 | 适合理解“导演 prompt”不仅是画面描述，也包括镜头和人物关系 |
| magcil/movie_shot_classification_dataset | 电影镜头运动分类小数据集 | 适合做轻量学习样本：静止、手持、平摇、推进、拉出等 |

这些数据集不一定都适合直接商用训练，但很适合做学习材料：看它们如何定义镜头标签，再把标签转成自己的 prompt 字段。

## 论文方向

| 论文/方向 | 关键信息 | 可转化知识 |
| --- | --- | --- |
| CineTechBench | 用专家标注评估模型对电影摄影技术的理解和生成能力，维度包括景别、角度、构图、运镜、光线、色彩、焦段 | 证明 prompt 字段应该覆盖完整摄影维度 |
| ShotBench / ShotQA | 从 200 多部优秀电影中构造电影语言理解问答，覆盖 8 个摄影维度 | 可把优秀广告拆成 QA：这是什么景别？为什么这样拍？ |
| MovieNet | 大规模电影数据集，包含镜头尺度和运动等电影理解任务 | 用于建立镜头分类的基础词表 |
| CameraBench | 面向任意视频的镜头运动理解 benchmark，强调专家标注和运动原语 | 提醒 Agent 区分 zoom in 和 dolly in，不要混写 |
| E.T. Exceptional Trajectories / DIRECTOR | 用电影片段构造摄像机轨迹、角色信息和文字描述，研究文本到镜头轨迹 | 对 AI 视频很重要：运镜要和人物调度同步 |
| CameraDiff / Cinematic2K | 约 2000 个电影语言视频，覆盖取景、角度和运镜二十类子模式 | 可作为镜头语言小样本下载候选 |
| MovieCuts | 研究 cut 类型识别，强调剪辑不是只有画面，也和声音、说话人、空间关系有关 | 商业短视频后续应补“剪辑与声音设计” |

## 推荐下载/收藏的数据

优先级从高到低：

1. **CineTechBench**：最适合补“AI 视频摄影维度”和模型评估视角。
2. **ShotBench / ShotQA**：最适合训练自己识别镜头语言，尤其是看图问答式拆片。
3. **MovieNet**：大而全，但下载和使用门槛更高，适合后续做系统研究。
4. **MovieCuts**：当你开始研究剪辑节奏、转场和声音驱动剪辑时再下载。
5. **E.T. / DIRECTOR**：适合研究复杂镜头轨迹，不是第一阶段必需。
6. **movie_shot_classification_dataset**：小而直接，适合快速看镜头运动分类样本。

## 学习路线

### 第 1 阶段：镜头词表

目标：看到一个画面，能说出景别、机位、构图、运动、光线。

动作：
- 看 StudioBinder / Runway 术语资料。
- 把常用词翻译成中文和英文 prompt。
- 用 [[domains/视觉制作/06-AI视频/04-商业短视频摄影与运镜语法|商业短视频摄影与运镜语法]] 做速查。

### 第 2 阶段：镜头任务

目标：不只知道是什么镜头，还知道为什么这样拍。

动作：
- 读 *Film Directing: Shot by Shot*、*The Filmmaker's Eye*。
- 每拆一个广告，给每个镜头标注“镜头任务”。
- 更新 [[domains/视觉制作/06-AI视频/90-案例库/index|AI商业短视频案例库]]。

### 第 3 阶段：镜头关系

目标：理解镜头之间如何形成节奏、对比和情绪递进。

动作：
- 读 *The Visual Story* 和 *In the Blink of an Eye*。
- 用 MovieCuts / ShotBench 思路拆“为什么这里切”。
- 为短视频建立“远 -> 中 -> 近”“冷 -> 暖”“外部任务 -> 内心关系”等常用结构。

### 第 4 阶段：AI prompt 转译

目标：把导演决策稳定转成可生成的 prompt。

动作：
- 用 [[domains/视觉制作/06-AI视频/05-AI视频导演Prompt模板|AI视频导演 Prompt 模板]]。
- 每个镜头只保留一个主运镜。
- 分镜图片先确认构图，视频 prompt 只负责运动和情绪。
