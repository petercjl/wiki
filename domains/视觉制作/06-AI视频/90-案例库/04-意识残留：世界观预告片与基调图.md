## 案例结论

《意识残留》的关键不是“用 AI 做几个科幻镜头”，而是先拥有一个完整的世界观和剧本，再把其中最能传达冲突的片段剪成预告片。

它的构思链路是：

```text
完整剧本
-> 人物关系与动机
-> 世界规则与社会冲突
-> 选择一个代表性基调镜头
-> 批量生成图片并筛选
-> 图生视频，大量舍弃
-> 音乐和剪辑制造预告片节奏
-> 用预告片测试市场反馈
```

这条案例对电商广告也有启发：不是所有产品都适合直接讲功能。某些新品、品牌概念、IP 联名、高客单产品，可以先做“概念预告”，用一个有世界观的短片测试用户是否愿意进入这个品牌故事。

## 主题诊断

| 维度 | 项目判断 |
| --- | --- |
| 类型 | AI 科幻电影预告片 |
| 核心命题 | 意识复制和上传技术一旦脱离监管，会撕裂社会秩序 |
| 核心冲突 | 科学伦理、阶层权力、技术滥用、下一代生存 |
| 人物关系 | 年轻行动者、老科学家、机械警察、抗议人群、被卷入的普通人 |
| 视觉路线 | 好莱坞科幻大片感，暗调、警灯、机械执法、城市混乱、实验室和街头并置 |
| 制作重点 | 剧本先行、基调图、人物一致性、镜头取舍、音乐二次合成 |

## 构思路径复原

### 1. 先写完整剧本，而不是堆碎片镜头

博主在 `ch07` 里明确说，他是先写完完整剧本才去做预告，而不是杂乱做一些碎片。这个判断非常重要。

预告片只露出一部分信息，但它背后必须有完整结构：

- 谁发明或参与了意识上传技术。
- 为什么复制意识需要联邦授权。
- `V-1 mind upload` 为什么有缺陷。
- `Omega Life breach` 造成过什么灾难。
- 富人为什么会滥用这项技术。
- 主角为什么仍然要激活它。
- 机械警察为什么追捕主角。

这些信息不需要全部讲清楚，但观众必须感觉到“背后有一整部电影”。这种厚度不是画面风格给的，而是剧本给的。

### 2. 让台词承担世界规则

原片里的英文台词不是普通情绪对白，而是在快速建立世界规则：

| 台词信息 | 透露的世界规则 |
| --- | --- |
| 未经联邦授权复制他人意识 | 意识复制是受监管的高危技术 |
| 未受监管的数字意识上传公共网络 | 技术风险会扩散到公共基础设施 |
| V-1 mind upload 有缺陷 | 技术尚未成熟，但有人急于使用 |
| Omega Life breach 仍然困扰我 | 过去发生过重大技术灾难 |
| 富人会利用你的技术 | 科技不是中性的，会被阶层权力操控 |
| 法庭不再需要证词 | 意识/记忆技术已经改变司法秩序 |
| 我不是为你做，是为了他们 | 主角有公共性动机，而不是私人冒险 |

这给商业短片一个启发：高概念广告不要用旁白解释设定，可以把规则藏进人物冲突和对话里。

### 3. 选择最能代表全片的基调图

博主说他会先生成第一张基调图，用它定位全片画面调性。他在本片选择“主角被机械警察逮捕”的画面，因为它同时体现：

- 男主是谁。
- 机器人/机械警察是什么力量。
- 这个世界的压迫感。
- 科幻和警匪追捕的类型气质。
- 画面应该偏暗、紧张、硬光、警灯、城市夜景。

基调图不是封面图，也不是最好看的图。它是整个项目的视觉标准。

### 4. 图片生成阶段要允许大量抽卡

讲解里说，他在图片生成中用了大量抽卡，并反复修改提示词。很多图质量很糟糕，并不是一次就能生成标准图片。

这里的经验是：AI 图片不是“输入剧本自动出片”，而是一个导演筛选过程：

```text
剧情镜头需求
-> 生成大量候选图
-> 按人物关系、场景逻辑、类型调性筛选
-> 找到基调图
-> 用基调图约束后续场景
-> 继续为剩余剧情出图
```

### 5. 图生视频阶段继续大量舍弃

博主提到，图生视频阶段仍然有很多镜头要浪费掉，但没有关系，因为一切以剧情为主，导演最重要的能力是取舍。

这说明 AI 视频的核心不是“每个镜头都能用”，而是：

- 哪个镜头推进剧情。
- 哪个镜头建立世界。
- 哪个镜头强化冲突。
- 哪个镜头只是炫技。
- 哪个镜头虽然漂亮但会让故事变散。

### 6. 音乐不是直接生成即用

博主会根据影片基调生成 AI 预告片音乐，但音乐不能直接用。他认为 AI 可以生成很多不错片段，后期还需要二次合成。

预告片的音乐任务不是铺氛围，而是制造段落推进：

```text
设定抛出
-> 冲突升高
-> 追捕和混乱
-> 情绪停顿
-> 最后一击
```

### 7. 预告片可以作为市场测试

讲解结尾说，因为基础工作已经做得差不多，后续可以进入正片制作；同时，这样的影片也可以测试什么类型的片子对市场流量反馈是 OK 的，再决定是否要做。

这是这个案例最接近商业项目的一点：

```text
先做世界观和预告片
-> 用较低成本测试观众兴趣
-> 看反馈决定是否投入正片/系列/品牌 campaign
```

对电商品牌来说，这可以迁移成“概念片先行”：先测试一个视觉世界、人物关系或品牌故事是否有吸引力，再决定是否做完整系列。

## 原片故事结构

| 段落 | 画面/台词功能 | 情绪功能 | 方法意义 |
| --- | --- | --- | --- |
| 非法复制意识 | 质问“你未经授权复制了别人的意识？” | 开场即给出高危设定 | 用法律风险快速建立世界规则 |
| 机械追捕 | B-54 区域嫌疑人移动，机械警察响应 | 制造外部压力 | 用追捕让设定变成行动 |
| 科学家拒绝 | V-1 有缺陷，不能交给你 | 技术伦理冲突 | 让技术不是万能工具，而是危险选择 |
| 公共网络风险 | 未监管数字意识上传公共网络 | 世界级后果 | 把个人行动升级为社会风险 |
| 过去灾难 | Omega Life breach | 历史阴影 | 让世界有前史 |
| 富人滥用 | 富人会利用你的技术 | 阶层冲突 | 让科幻设定有现实隐喻 |
| 社会撕裂 | 这会撕裂社会 | 价值判断 | 明确科学家的恐惧 |
| 主角动机 | “我不是为了你，是为了他们” | 公共使命 | 让主角不是无脑冒险 |
| 最后逮捕 | Hands up, you are arrested | 动作高潮 | 用强动作收尾，留下悬念 |

## 博主讲解中的方法节点

### 节点 1：人物关系设计比后期更耗时

他花了比后期制作还长的时间来做人设、动机、行为逻辑和场景剧情线。这是本案例最核心的方法节点。

如果没有这个步骤，AI 只能生成“像科幻片的镜头”；有了这个步骤，镜头才会变成“某个世界里正在发生的事”。

### 节点 2：基调图要从剧情里抽

基调图不能随便选一张漂亮图。它要来自剧情中最能代表项目矛盾的镜头。本片选“主角被机械警察逮捕”，因为它同时压缩了人物、权力机器和世界冲突。

### 节点 3：图片候选要按关系筛，不只按美感筛

讲解里提到“控制人物关系”。这意味着选图时不是只看质感，而要看角色站位、权力关系、情绪方向是否正确。

### 节点 4：导演能力就是取舍

AI 会给你很多素材，但导演要删掉不服务剧情的镜头。越是大片感项目，越容易被炫技镜头诱惑。

### 节点 5：音乐和剪辑是最后的类型包装

当剧本、图片、视频素材都有之后，音乐和剪辑把它包装成“好莱坞预告片”。如果前面没有剧本和冲突，音乐再大也只是空。

## 可迁移灵感

### 适合迁移到哪些商业任务

- 新品牌世界观发布。
- 游戏、影视、动漫、IP 预告。
- 高客单新品概念片。
- 科技产品、智能硬件、AI 产品。
- 潮玩、运动、户外、汽车等适合建立视觉世界的品类。
- 品牌联名、限量系列、年度 campaign。

### 可迁移结构

```text
产品/品牌概念
-> 写出一个世界规则
-> 设计核心冲突
-> 设计 2-3 个角色立场
-> 找一个最能压缩冲突的基调镜头
-> 生成并筛选基调图
-> 扩展关键场景
-> 剪成预告片
-> 用流量反馈判断是否继续投入
```

### 迁移示例

| 产品/主题 | 不该只讲 | 可以转译成 |
| --- | --- | --- |
| AI 眼镜 | 轻便、翻译、拍摄 | 一个普通人突然能看见城市隐藏信息后的选择 |
| 智能家居 | 全屋联动 | 一个家庭搬进会学习他们习惯的房子，便利与失控并存 |
| 户外装备 | 防水、耐磨、轻量 | 未来极端天气里，一个人穿越城市边缘寻找安全区 |
| 运动品牌 | 透气、支撑、速度 | 地下训练联盟里，身体极限和身份突破的预告 |
| 潮玩/IP | 造型、限量 | 一个小角色在废土城市里寻找同类，形成世界观系列 |

## 反向拆解：如果做成硬广会怎样

错误做法可能是：

- 只生成一堆科幻城市、机器人、爆炸镜头。
- 没有人物动机，只有氛围。
- 台词都是“未来已经到来”“科技改变命运”。
- 每个镜头都很炫，但观众不知道谁在反抗谁。
- 产品或品牌强行贴在最后，没有进入世界规则。

这种做法的问题是：它有类型外壳，没有叙事压力。《意识残留》的优势在于，观众会感觉这个预告片背后真的有一部电影。

## Agent 调用模板

当 Agent 拿到适合做世界观、概念片、预告片的新产品时，可以调用本案例：

1. 这个产品能不能变成一个世界规则，而不只是一个功能？
2. 世界规则失控时，会产生什么冲突？
3. 谁支持它，谁反对它，谁被它伤害？
4. 主角为什么必须行动？
5. 哪个镜头能同时体现主角、反派力量和世界规则？
6. 基调图应该规定哪些视觉标准？
7. 哪些镜头服务剧情，哪些只是炫技？
8. 预告片要测试什么市场反馈？

输出结构：

```text
产品/概念：
世界规则：
核心冲突：
角色立场：
主角动机：
代表性基调镜头：
视觉基调：
关键场景清单：
预告片段落：
音乐节奏：
市场测试指标：
误用边界：
```

## 误用边界

- 不要把世界观片做成美术图合集。必须先有规则、冲突、人物和动机。
- 不要让基调图只承担“好看”。它必须同时压缩项目的核心矛盾。
- 不要把 AI 抽卡当成失败。高质量项目需要大量候选和筛选。
- 不要迷信图生视频一次成功。导演要有删镜头的能力。
- 不要用空泛旁白解释概念。尽量让台词、动作和场景自然透露世界规则。
- 不要把预告片当最终成片。它可以是低成本市场测试工具。
