# AI长视频连续性组接方法

AI 长视频不要把“延长 15 秒”当成唯一解法。多数断裂来自错误的组接方法：相似景别硬接、相似角度硬接、动作没有重叠、剧情被强行拉成长镜头。更稳的做法是把 AI 视频当成电影剪辑问题：每段生成只承担一个可完成的镜头或小剧情，再用连续性剪辑把它们组接起来。

这页补充 [[domains/视觉制作/06-AI视频/03-导演式镜头设计方法|导演式镜头设计方法]]、[[domains/视觉制作/06-AI视频/04-商业短视频摄影与运镜语法|商业短视频摄影与运镜语法]] 和 [[domains/视觉制作/06-AI视频/07-故事板铁三角AI视频控制法|AI视频控制资产系统：身份板、故事板、导演台与素材库]]。它解决的是：视频模型单段时长有限、续写段色彩突变、动作断掉、衔接奇怪时，怎样像导演和剪辑师一样设计镜头关系。

## 核心判断

限制 AI 长视频的通常不是单段 15 秒，而是把所有内容硬做成一个长镜头。电影本来就靠镜头组接完成叙事，AI 视频也应该先拆镜头，再设计段落之间怎么接。

| 问题 | 错误做法 | 连续性做法 |
| --- | --- | --- |
| 两段颜色、人物、动作突然跳 | 用上一段尾帧直接续写相似画面 | 换景别或换角度，让切换本身成为合理剪辑 |
| 动作突然断掉 | 第一段动作结束后再生成第二段动作 | 在动作中切，保留相同运动轨迹 |
| 15 秒之后剧情不够用 | 强行延长一个复杂长镜头 | 用分镜把剧情拆成多个 15 秒内的小段 |
| 镜头看起来像随机抽卡 | 只写 prompt，不写前后镜头关系 | 在故事板阶段写清每段的镜头任务、时长和衔接点 |

## 方法一：切换景别和角度

当两段视频从相似景别、相似角度硬接，观众会直接感到画面跳变。更稳的方法是让下一段变成“有理由的剪辑”：同一事件继续发生，但画面切到不同景别或不同角度。

操作流程：

1. 在上一段视频结尾截取尾帧。
2. 用多机位、九宫格或同类工具生成多个候选首帧。
3. 选择和上一段不同景别或不同角度的图作为下一段首帧。
4. 下一段 prompt 继续承接同一剧情动作，但不要复刻上一段构图。
5. 剪辑时让切点落在画面信息变化合理的位置。

选择原则：

| 上一段 | 下一段更适合 | 目的 |
| --- | --- | --- |
| 中景 | 近景、特写、远景 | 改变观众距离，避免同景别跳切 |
| 正面 | 侧面、背面、俯拍、低机位 | 改变空间视角，让切镜成立 |
| 人物动作 | 手部、物品、反应镜头 | 用细节或反应遮住模型不稳定 |
| 空间建立 | 人物推进、动作细节 | 从场景信息转入动作信息 |

这个方法适合轻中度连续性问题：角色、场景大体稳定，只是段与段之间有卡顿、颜色突变或机位僵硬。

## 方法二：在动作中衔接

更高级的做法是 match on action：让两段视频保留同一动作的重叠部分，在运动进行到一半时切镜头。观众的大脑会自动把前后运动轨迹接上，因此不容易察觉断点。

典型流程：

1. 在上一段中找到动作起势，比如人物开始拿起杯子、转身、迈步、伸手、打开门。
2. 在动作刚开始时截取当前帧，而不是等动作完成。
3. 基于这一帧生成不同景别或不同角度的候选首帧。
4. 下一段 prompt 不写“开始喝咖啡”，而写“继续抬起咖啡杯，举到嘴边轻轻喝一口，随后放下”。
5. 生成后把两个动作轨迹对齐，把剪辑点放在动作进行到一半的位置。

提示词关键词要体现“继续”，而不是“重新开始”：

```text
主体继续[上一段已经开始的动作]，沿着同一运动轨迹完成[动作结果]。
保持人物身份、服装和场景一致，切换为[新景别/新角度]。
```

可用动作锚点：

| 动作锚点 | 为什么好接 |
| --- | --- |
| 手拿物品、抬杯、递文件 | 物体轨迹明确，容易对齐 |
| 转身、回头、迈步 | 身体运动能遮住切点 |
| 推门、拉抽屉、打开包装 | 动作有清晰起点和终点 |
| 车辆、列车、人群移动 | 屏幕内运动提供连续方向 |

这类切点利用了视觉暂留和运动连续性感知。正式使用时不必执着于数值解释，关键是让前后帧的运动方向、速度感和动作目标保持一致。

## 方法三：用分镜组接剧情

当故事超过单段模型能力时，不要让一个镜头承担全部剧情。先把剧本拆成一组 15 秒以内可完成的小剧情，再让每段只负责一个剧情阶段。

分镜拆法：

1. 写出完整剧情目标。
2. 拆成若干镜头或镜头组，每组控制在模型稳定时长内。
3. 给每组标注时长、景别、动作、人物目标和必要参考图。
4. 每段同时喂入人物设定图和场景参考图，减少人物漂移和场景漂移。
5. 每段之间用景别/角度变化或动作中衔接，而不是机械续写。

示例结构：

| 段落 | 镜头组合 | 时长 | 任务 |
| --- | --- | --- | --- |
| 第一段 | 4 秒 + 3 秒 + 3 秒 + 4 秒 | 14 秒 | 建立车站、人物寻找、任务压力 |
| 第二段 | 3-5 个短镜头 | 15 秒内 | 递送文件、穿越人群、接近目标 |
| 第三段 | 反应镜头 + 结果镜头 | 15 秒内 | 完成交付，收束情绪 |

这样每个片段更容易生成，组接后又能超过单段时长限制，并且节奏更像电影。

## 三步决策表

| 当前问题 | 优先方法 | 追加控制 |
| --- | --- | --- |
| 只是尾帧续写卡顿 | 切换景别和角度 | 用多机位候选首帧 |
| 动作断裂明显 | 在动作中衔接 | prompt 写“继续动作”，剪辑点放在动作中段 |
| 剧情复杂、时间不够 | 分镜组接剧情 | 每段只负责一个剧情阶段 |

## Agent 使用模板

当用户要求“把 AI 视频做长、做顺、续写不跳、分镜组接”时，Agent 可以按下面的结构输出：

```text
长视频目标：
模型单段限制：
拆分原则：

段落表：
- 段落编号
- 剧情任务
- 推荐时长
- 景别/角度
- 动作锚点
- 参考图/身份板/场景图
- 首帧生成方式
- 视频 prompt
- 衔接方式
- 风险与备选

剪辑建议：
- 哪些切点用景别/角度变化
- 哪些切点用动作中衔接
- 哪些段落必须重新拆分
```

## 检查清单

- 是否还在试图硬续一个复杂长镜头？
- 每段是否能在模型稳定时长内完成？
- 前后段是否有景别或角度变化？
- 是否为关键动作保留了重叠轨迹？
- prompt 是否写了“继续抬起/继续转身/继续前进”等动作承接词？
- 切点是否放在运动中段，而不是动作结束后的静止点？
- 是否同时使用人物设定图和场景参考图？
- 复杂空间关系是否需要导演台或轴线检查？
