# 聚宽策略报错与排查

## 排查主线

1. 先确认错误发生在编译、回测运行、下单撮合、取数、模拟盘重启，还是研究环境。
2. 看错误是否由本地 Python 与聚宽环境差异导致。
3. 修代码后先本地运行检查器，再让用户复制到聚宽。
4. 若聚宽仍报错，把错误信息回写到本页和 `joinquant-strategy` skill。

## 常见问题矩阵

| 症状 | 可能原因 | 修复 |
| --- | --- | --- |
| `NameError: name 'xxx' is not defined` | 缺少 `from jqdata import *` 或 API 只在研究环境可用 | 补导入，或替换为回测环境 API。 |
| 定时函数报参数错误 | `run_daily` 回调不是 `func(context)` | 改为全局函数并只接收 `context`。 |
| `run_daily(handle_data, ...)` 报错或行为异常 | 把 `handle_data(context, data)` 当作定时函数 | 拆成 `market_open(context)`。 |
| 回测收益异常好 | 使用未来数据、静态前复权价格或当日不可见数据 | 开启 `use_real_price` 和 `avoid_future_data`，检查取数时间。 |
| 日线历史取不到当天 | `history`/`attribute_history` 日线本来不含当天 | 要当天数据时改用分钟数据或当前行情 API。 |
| 多标的 `get_price` 形状异常 | 默认 `panel=True` 返回 Panel | 设置 `panel=False` 并按 DataFrame 结构处理。 |
| 模拟盘替换代码后参数没变 | `initialize` 不会重新执行 | 使用 `after_code_changed` 修改持久化状态。 |
| 模拟盘重启后 `g.query` 丢失 | `query(...)` 等对象不能持久化 | 在 `process_initialize` 中初始化，并用 `g.__query`。 |
| 本地文件读不到 | 聚宽回测无法读本地路径 | 上传到研究环境，用 `read_file`。 |
| 下单无成交或被拒 | 停牌、零成交、涨跌停、现金不足、持仓不足、交易单位限制 | 下单前用 `get_current_data` 和持仓/现金检查。 |
| `create_backtest` 不能用 | 这是研究环境 API | 不要放入普通策略代码。 |

## 回写规则

当用户从聚宽复制真实报错回来：

1. 判断是代码 bug、API 用法、平台边界、数据时点、交易规则还是权限问题。
2. 修改策略。
3. 如果能静态识别，增强检查器。
4. 把新失败模式补到本页。
5. 如果是写作规则，补到 `/Users/pechen/.codex/skills/joinquant-strategy/references/joinquant-strategy-rules.md`。
