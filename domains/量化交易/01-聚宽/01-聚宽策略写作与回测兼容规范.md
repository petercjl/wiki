# 聚宽策略写作与回测兼容规范

## 目标

Codex 写聚宽策略时，必须把脚本当成“运行在聚宽回测/模拟引擎中的事件回调程序”，而不是普通本地 Python 程序。交付代码应能复制到聚宽策略编辑器中编译运行。

## 标准主线

1. 导入聚宽环境：

```python
from jqdata import *
```

2. 定义 `initialize(context)`，在这里设置全局参数、基准、真实价格模式、防未来函数模式、调度函数。
3. 用 `run_daily`、`run_weekly`、`run_monthly` 注册全局回调函数。普通策略优先使用定时函数，而不是 `handle_data`。
4. 在回调函数中取历史数据、生成信号、下单。
5. 用 `record` 和 `log` 输出回测观察指标。
6. 本地先运行语法检查和聚宽兼容检查，再让用户复制到聚宽。

## 初始化规范

常规 A 股策略建议：

```python
def initialize(context):
    set_benchmark('000300.XSHG')
    set_option('use_real_price', True)
    set_option('avoid_future_data', True)

    g.stock_pool = ['000001.XSHE']
    run_daily(market_open, time='09:30')
```

`set_option('use_real_price', True)` 应在 `initialize` 中调用。它让回测按历史真实价格成交，并降低静态前复权价格带来的未来函数风险。

`set_option('avoid_future_data', True)` 用于开启避免未来数据模式。若因此暴露报错，应修正取数逻辑，而不是默认关闭。

## 调度函数规则

- `run_daily`、`run_weekly`、`run_monthly` 注册的函数只能接受一个参数 `context`。
- 调度函数必须是全局函数，不能是类实例方法。
- 不要写 `run_daily(handle_data, ...)`。
- 不要无理由同时使用 `run_daily` 和 `handle_data`。
- `run_daily` 没有 `force` 参数。
- 使用具体时间如 `time='14:50'` 时，一般不要再设置 `reference_security`。期货、夜盘等需要参照标的时，应按期货规则单独处理。

## 数据获取规则

`attribute_history` 适合一只股票、多字段历史数据。

`history` 适合多只股票、单字段历史数据。

`get_price` 适合明确日期区间、多字段或更复杂取数。多标的取数时，建议显式设置 `panel=False`，避免 pandas Panel 兼容问题。

关键时间规则：

- 日线 `history` 和 `attribute_history` 不包含当天数据，即使在 15:00 后也不包含。
- 分钟数据不包含当前分钟。
- `get_price` 的 `end_date` 不应晚于 `context.current_dt`。
- 开启真实价格模式后，不要跨日期缓存 `history`、`attribute_history`、`get_price` 等数据 API 返回结果。

## 下单规则

常用接口：

- `order_target_value(security, value)`：把某标的调整到目标市值，适合仓位管理。
- `order_target(security, amount)`：把某标的调整到目标数量。
- `order_value(security, value)`：按金额买入或卖出。

下单前应考虑：

- 标的是否停牌。
- 是否涨停、跌停。
- 是否有足够现金或可卖持仓。
- A 股交易数量约束。
- 多个未完成订单可能被目标类接口取消。

## 平台环境边界

聚宽回测不是本地环境：

- 不能读取用户本地路径。
- 不能默认使用本地项目里的模块。
- 不能依赖未确认支持的第三方包。
- 如果需要外部 CSV/JSON，应先上传到聚宽研究环境，再用 `read_file` 读取。
- `create_backtest` 属于研究环境能力，不应写进普通回测策略。
