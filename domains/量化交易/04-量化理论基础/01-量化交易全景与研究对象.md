# 量化交易全景与研究对象

量化交易是把投资假设编码成可验证、可执行、可监控的交易系统。它不是单纯找指标，也不是只写回测脚本。Qlib 把量化投资链路概括为数据处理、模型训练、回测，并覆盖 alpha 寻找、风险建模、组合优化和订单执行；Machine Learning for Trading 则强调从数据来源到实盘执行的完整流程。两者共同说明：量化研究的对象是系统，不是单个买卖点。

## 量化系统的基本组件

| 组件 | 作用 | 典型问题 |
| --- | --- | --- |
| 市场与资产 | 决定交易对象和制度约束 | 股票、ETF、期货、期权、债券、外汇、加密资产分别怎么交易？ |
| 数据 | 提供历史状态和当前状态 | 价格、成交量、财务、基本面、新闻、盘口、宏观数据是否点时正确？ |
| 信号 | 把数据转成方向、排序或风险判断 | 动量、价值、质量、反转、波动率、事件、机器学习预测是否有效？ |
| 组合 | 把多个信号和资产转成权重 | 集中还是分散，等权还是风险预算，是否控制行业/风格暴露？ |
| 执行 | 把目标权重转成订单 | 何时交易、用什么订单、成交成本和市场冲击如何处理？ |
| 风控 | 限制单次、组合和系统级损失 | 仓位、杠杆、止损、回撤、流动性和极端事件如何约束？ |
| 回测 | 用历史模拟系统表现 | 是否有未来函数、幸存者偏差、成本低估和过拟合？ |
| 监控 | 实盘后持续检查失效 | 策略是否偏离预期，市场结构是否变化，数据是否异常？ |

## 研究对象的三个层次

量化研究至少有三层对象：

1. Alpha：是否存在可预测的收益、相对强弱或风险补偿。
2. Portfolio：如何把多个 alpha、资产和约束合成可承受的组合。
3. Execution：如何以足够低的成本把组合意图落到真实成交。

很多初学者只研究第一层，例如找一个看似有效的指标。但实盘收益通常在第二、第三层被稀释：信号相关性太高、组合权重不稳、换手过高、成交价格不理想，都会让纸面 alpha 消失。

## 收益来源分解

一个策略的收益可能来自：

- 市场 beta：整体市场上涨带来的收益。
- 风格风险溢价：价值、动量、质量、规模、低波动等长期风格。
- 资产配置：股票、债券、商品、现金、海外资产之间的配置。
- 择时：在不同市场阶段提高或降低风险暴露。
- 横截面选择：同一时点选择相对更好的标的。
- 微观结构或执行优势：盘口、成交、冲击成本管理。
- 杠杆和资金利用率：承担更多风险获得更高期望收益。
- 运气或过拟合：样本内碰巧有效，但没有可持续机制。

入库任何策略时，必须判断它主要赚哪一类钱。否则“收益高”无法转化为知识。

## 量化研究的最小闭环

一个研究闭环应按以下顺序执行：

1. 写出可证伪假设。
2. 明确交易宇宙和数据可见性。
3. 构造信号或规则。
4. 先做信号层验证，再做完整回测。
5. 加入成本、约束和订单执行。
6. 做分年度、分市场环境和样本外验证。
7. 记录失败版本和搜索空间。
8. 决定是否进入模拟盘或继续研究。

## 对 Peter 的使用规则

- 当策略只是社区代码时，先拆系统结构，再评价收益。
- 当策略结果很好时，先找收益来源和偏差风险。
- 当策略结果不好时，先定位是市场、信号、组合、执行还是成本问题。
- 当准备优化时，每次只改一个主假设，避免在回测曲线上雕花。
