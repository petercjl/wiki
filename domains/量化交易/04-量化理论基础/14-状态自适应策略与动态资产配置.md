# 状态自适应策略与动态资产配置

状态识别本身不赚钱。它只有被转成风险预算、策略选择、模型路由或资产配置动作时，才可能改善组合表现。

本页讨论状态识别之后如何行动。它不是推荐复杂模型，而是建立一个从“市场状态”到“组合动作”的设计空间。阅读前应先读 [[domains/量化交易/04-量化理论基础/13-市场状态识别与策略失效|市场状态识别与策略失效]] 和 [[domains/量化交易/04-量化理论基础/07-组合构建与仓位管理|组合构建与仓位管理]]。

## 状态到动作的四种模式

| 模式 | 做法 | 适用 | 主要风险 |
| --- | --- | --- | --- |
| Regime filter | 不利状态下降仓或停止交易 | 单一策略在高波/熊市明显失效 | 容易错过反弹，状态滞后。 |
| Strategy router | 不同状态启用不同策略 | 趋势、均值回复、防御策略互补 | 子策略太多会过拟合。 |
| Expert models | 每个状态训练专门模型 | 有足够数据和稳定标签 | 样本切小后训练不稳。 |
| Dynamic allocation | 不同状态配置不同资产 | 多资产组合、ETF/大类资产 | 状态和资产表现关系会漂移。 |

## Regime Filter：状态作为风险管理器

QuantStart 的 QSTrader 示例把 HMM 用作 risk manager：在高波动状态下禁止趋势策略开仓。这个思想比“状态直接预测涨跌”更稳健，因为它只回答一个问题：当前环境是否适合这个策略暴露风险？

常见动作：

- 高波/压力状态：降低股票或风险资产仓位。
- 震荡状态：降低趋势策略仓位，减少追涨杀跌。
- 趋势状态：允许趋势策略正常持仓。
- 极端状态：只平仓，不新开仓。

设计规则：

1. 先证明策略在某状态明显失效。
2. 再让该状态触发降仓或停手。
3. 不要为了提高收益而让状态模型预测短期涨跌。

## Strategy Router：不同状态用不同策略

固定策略在某些年份失效，常见原因是它只适合一种市场机制。例如：

| 市场状态 | 可能更适合的策略 |
| --- | --- |
| 低波趋势 | 趋势跟随、动量轮动。 |
| 高波下跌 | 防御资产、现金、短债、保护性策略。 |
| 区间震荡 | 均值回复、套利、低换手防御。 |
| 普涨牛市 | 高 beta、成长、动量。 |
| 风格切换 | 多策略分散或风格中性。 |

Strategy router 的重点不是“每个状态都找最优策略”，而是避免明显错配：

- 趋势策略不要在无趋势状态里高频换仓。
- 均值回复策略不要在单边趋势里硬抄底。
- 高 beta 策略不要在风险偏好崩塌时满仓。

## Expert Models：状态专用模型

QuantInsti 的 regime-adaptive trading 示例使用 HMM 检测状态，再为每个状态训练 Random Forest，并用当前状态选择模型。这个模式适合机器学习研究，但要非常谨慎：

- 每个状态内样本变少，模型更容易过拟合。
- 状态标签若由全样本训练，会引入看未来风险。
- 模型路由需要 walk-forward：先用过去识别状态和训练专家模型，再在未来验证。
- 状态专用模型必须和统一模型比较，不能只看复杂模型自身收益。

最低验证要求：

1. 同一特征集下比较 single model vs regime-specific models。
2. 状态标签必须在训练窗口内生成。
3. 测试窗口只允许使用当时可见的状态预测。
4. 报告每个状态下的交易次数、收益、回撤和命中率。

## Dynamic Allocation：状态驱动资产配置

多资产配置是状态识别最自然的落点。相关来源中，SPY/TLT/GLD 或股票/债券/黄金的配置框架都体现同一个逻辑：

- 稳定低波状态：股票等风险资产可能占优。
- 压力或高波状态：债券、黄金、现金类资产可能保护组合。
- 过渡状态：降低风险资产仓位，分散到防御资产。

状态驱动配置常见动作：

| 状态 | 权重动作 |
| --- | --- |
| risk-on | 提高权益、成长、动量、商品中顺周期资产。 |
| risk-off | 提高现金、短债、长债、黄金、防御资产。 |
| transition | 降低总风险暴露，增加分散。 |
| high-vol trend | 保留趋势资产但降低杠杆或单资产上限。 |
| high-vol crash | 停止风险策略或只保留防御组合。 |

但动态配置也有明显陷阱：

- 状态识别滞后会导致“跌完才防守、涨完才进攻”。
- 防御资产关系会变化，股债相关性不是永远负相关。
- 频繁切换会产生交易成本和税费。
- 如果资产池本身是事后挑选，也会形成 hindsight bias。

## No-Trade Regime：不交易也是策略动作

RobotWealth 对 regime-based split models 的讨论提醒：极端波动状态可能本身难以预测。在这种状态下，最优动作可能不是换一个更复杂模型，而是减少交易或不交易。

No-trade regime 的使用条件：

- 历史证据显示某状态下交易期望显著差。
- 状态持续时间足够长，停手不只是随机错过几天。
- 停手后收益/回撤/成本改善来自多个样本期，而非单次危机。
- 停手规则在交易时点可见，不依赖事后标注。

## 现金替代资产

降仓后现金闲置可以降低回撤，但也会拖累长期收益。状态自适应策略需要区分：

| 空出仓位去向 | 优点 | 风险 |
| --- | --- | --- |
| 现金 | 最简单，风险低 | 收益拖累，通胀/机会成本。 |
| 货币/短债 | 低波动，有基础收益 | 利率环境变化，ETF 流动性。 |
| 长债 | 压力期可能保护 | 利率上行时可能亏损。 |
| 黄金 | 部分危机和通胀环境保护 | 自身趋势波动大。 |
| 低波/防御权益 | 保留权益暴露 | 熊市仍可能下跌。 |

因此“降仓”不是唯一答案。更完整的问题是：状态不适合主策略时，组合应该持有什么风险？

## 设计约束

状态自适应策略最容易过拟合。必须设置约束：

1. 状态数量少：优先 2-3 个状态。
2. 动作少：满仓、半仓、防御、空仓等少数动作。
3. 特征少：先用收益、波动、回撤、相关性等低自由度特征。
4. 换手约束：状态切换必须有最小持续期或冷却期。
5. 延迟执行：至少测试一日延迟，避免隐性看未来。
6. 成本真实：状态策略通常增加交易，成本必须纳入。
7. 分状态报告：不能只报告总收益。

## Agent 使用模板

当用户问“策略固定会失效，怎么解决”时，按以下设计空间回答：

1. 先定位策略在哪些状态失效。
2. 如果只是高波/熊市亏损，优先测试 regime filter。
3. 如果不同状态适合不同收益假设，再考虑 strategy router。
4. 如果已有稳定状态标签和足够样本，再考虑 expert models。
5. 如果是多资产问题，优先考虑 dynamic allocation 和防御资产。
6. 每一步都必须用 walk-forward 和交易延迟验证。
