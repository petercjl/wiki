# 绩效评估、归因与报告

绩效评估要回答：策略赚了多少钱、承担了什么风险、收益来自哪里、失败发生在哪里、结果是否可信。Pyfolio 的 tear sheet 思路、Empyrical 的风险指标和 ffn 的绩效分析都指向同一点：回测报告不是收益截图，而是一套证据结构。

## 报告结构

1. 策略摘要：名称、资产、类型、周期、核心假设。
2. 回测设置：时间、资金、基准、成本、滑点、频率。
3. 核心指标：累计、年化、超额、波动、最大回撤、Sharpe/Sortino/Calmar。
4. 收益路径：净值曲线、月度/年度收益、回撤曲线。
5. 风险分析：回撤区间、波动变化、尾部亏损。
6. 交易分析：交易次数、换手、成本、订单失败。
7. 持仓分析：集中度、行业/类别/风格暴露、单标的贡献。
8. 归因分析：市场 beta、因子、资产类别、择时、选股、执行成本。
9. 稳健性：分区间、分市场环境、成本敏感性、样本外。
10. 结论：可用边界、主要风险、下一步实验。

## 归因层次

| 层次 | 问题 |
| --- | --- |
| 市场归因 | 策略是否只是跟随大盘？ |
| 资产归因 | 哪些资产贡献收益和亏损？ |
| 类别归因 | 宽基、行业、商品、债券、海外等类别贡献如何？ |
| 因子归因 | 收益来自动量、价值、质量、低波动还是其他暴露？ |
| 择时归因 | 仓位升降是否增加收益或降低回撤？ |
| 执行归因 | 成本、滑点、换手和失败订单影响多少？ |
| 参数归因 | 某个参数是否过度决定收益？ |

## 不能接受的报告结论

- “收益高，所以策略好。”
- “最大回撤低，所以策略安全。”
- “这版比上版好，所以新增过滤器有效。”
- “一年回测很好，所以可以实盘。”
- “人工池表现好，所以池子科学。”
- “Sharpe 高，所以没有问题。”

这些都缺少归因和稳健性。

## 分环境评估

一个策略至少要按以下环境拆分：

- 牛市、熊市、震荡市。
- 高波动、低波动。
- 利率上行、利率下行。
- 行业分化强、行业普涨/普跌。
- 流动性充裕、流动性紧张。
- 样本内、样本外。

不同策略有不同环境偏好。趋势策略通常怕震荡，均值回复怕单边趋势，价值策略怕长期估值压缩，机器学习策略怕分布漂移。

## 聚宽四件套分析模板

Peter 导出聚宽结果后，Agent 应读取：

- 收益概述：核心绩效指标。
- 交易详情：订单、成交、成本、失败。
- 持仓收益：持仓贡献和风险暴露。
- 日志：策略决策路径。

分析顺序：

1. 核对回测区间和设置。
2. 复算或校验核心指标。
3. 对照交易和持仓解释净值大波动。
4. 用日志核对策略是否按预期执行。
5. 输出归因和下一轮可证伪实验。
