# 市场、资产与交易制度

量化策略必须先落在具体市场和资产上。不同资产的交易制度、成本结构、数据可得性和风险来源不同，同一个信号在股票、期货、ETF、期权或加密资产上可能含义完全不同。

## 主要资产类别

| 资产 | 交易特征 | 量化关注点 |
| --- | --- | --- |
| 股票 | 公司权益，受基本面、行业、风格、流动性影响 | 横截面选股、因子、多空组合、财报事件、行业中性。 |
| ETF/基金 | 一篮子资产或策略的份额 | 资产配置、轮动、折溢价、流动性、跟踪误差、申赎机制。 |
| 期货 | 保证金、杠杆、到期换月 | 趋势跟随、期限结构、展期收益、保证金风险、夜盘。 |
| 期权 | 非线性收益，受波动率和希腊值影响 | 波动率交易、对冲、尾部保护、时间价值损耗。 |
| 债券 | 利率、信用、久期、流动性驱动 | 久期、信用利差、收益率曲线、流动性折价。 |
| 外汇 | 宏观利差、货币政策、全球风险偏好 | 套息、趋势、宏观事件、交易时区。 |
| 商品 | 供需、库存、季节性、期限结构 | 趋势、carry、库存周期、产业链数据。 |
| 加密资产 | 7x24、交易所分散、监管和流动性差异大 | 动量、资金费率、链上数据、交易所风险。 |

## 交易制度决定策略边界

一个策略在回测前必须回答：

- 是否允许做空？
- 是否允许杠杆？
- 是否 T+0 或 T+1？
- 是否有涨跌停、熔断、停牌？
- 是否有最小交易单位和最小变动价位？
- 是否存在夜盘、节假日和交易日历差异？
- 下单类型支持哪些：市价、限价、止损、止盈、跟踪止损、OCO？
- 数据频率和下单频率是否匹配？

Backtrader README 中列出的 broker simulation、commission、slippage、volume filling、订单类型和 trading calendars，说明成熟回测框架不会只关心信号，而会把交易制度纳入模拟。Zipline 作为事件驱动回测系统，也把算法对交易事件的响应作为核心。

## 可交易宇宙

交易宇宙是策略允许考虑的资产集合。它不是随手写一个列表，而是策略假设的一部分。

| 宇宙类型 | 例子 | 主要风险 |
| --- | --- | --- |
| 全市场宇宙 | 全 A 股、全 ETF、全期货主力合约 | 流动性差、数据噪声多、计算量大。 |
| 指数成分宇宙 | 沪深300、中证500、标普500 | 成分历史变化和幸存者偏差。 |
| 流动性宇宙 | 成交额前 N、上市满 N 日 | 可能偏向热门资产和短期拥挤。 |
| 基本面宇宙 | 盈利、规模、行业过滤 | 财报滞后、口径修订、数据发布时间。 |
| 人工主题宇宙 | 核心资产、红利、科技、商品 | 最容易引入 hindsight bias。 |

科学宇宙必须历史时点可复现。回测 2021 年时，只能使用 2021 年当时可见的上市状态、成分、流动性和数据。

## 市场微观结构

信号只告诉你想买什么，市场微观结构决定你能否以预期价格成交：

- 买卖价差：流动性越差，成交成本越高。
- 市场深度：大资金下单会推动价格。
- 成交量限制：高换手策略可能无法按回测价格成交。
- 交易时段：开盘、收盘、午间、夜盘的流动性不同。
- 停牌与涨跌停：想卖时可能卖不掉，止损可能失效。
- ETF 折溢价：ETF 价格可能偏离净值。

## Agent 检查清单

分析任何策略前先问：

1. 交易资产是什么？
2. 策略是否依赖做空、杠杆、T+0 或特殊订单？
3. 宇宙是历史时点生成，还是事后固定列表？
4. 流动性是否足以支持策略换手？
5. 成本、滑点、涨跌停、停牌是否在回测中体现？
6. 资产制度是否与策略逻辑冲突？
