# QMT四季发财 ETF 轮动策略案例

## 一句话结论

QMT四季发财v3 是一个聚宽 ETF 单标的轮动策略：用“长期动量年化收益 × 趋势稳定性 R²”给 ETF 池排序，再用短期动量、RSI、近 3 日下跌和 ATR 止损控制风险。2020-01-02 至 2025-12-31 回测中表现很强，但它仍是回测案例，不是投资建议；实盘化前必须处理高换手、现金不足订单失败和参数稳健性问题。

## 核心逻辑

### 标的池

其中 `g.defensive_etf = "511880.XSHG"`。策略无合格候选时会尝试买入它作为防御模式目标；但 ATR 止损默认豁免防御 ETF。

### 信号计算顺序

策略每日对 ETF 池计算候选得分：

```text
score = annualized_returns * r_squared
```

其中：

- `annualized_returns`：基于最近 `lookback_days = 25` 的价格序列，用对数价格线性拟合斜率年化。
- `r_squared`：同一趋势拟合的稳定性，趋势越平滑，得分越可信。
- 权重：拟合时使用线性递增权重，让近期价格权重更高。

信号计算有几个容易被忽略的实现细节：

1. 取数使用 `attribute_history(etf, lookback, '1d', ['close', 'high'])`，再把 `get_current_data()[etf].last_price` 追加到价格序列末尾。因此策略信号不是纯前一日收盘信号，而是包含当天 14:29 附近的当前价。
2. `lookback` 不是固定 25，而是取长期动量、短期动量、MA、RSI、MACD、成交量、布林带所需窗口的最大值再加 20 天。这样即使某些过滤器关闭，数据长度仍按所有模块的最大需求准备。
3. 长期动量用最近 `g.lookback_days + 1` 个价格点，默认 26 个点，其中最后一个点是当前价。
4. 年化收益来自对数价格线性拟合斜率：`annualized_returns = exp(slope * 250) - 1`。
5. R² 用加权残差平方和与加权总平方和计算，权重从 1 线性增加到 2，越近的价格权重越高。

### 候选过滤顺序

在得分之前和之后叠加过滤：

| 过滤/风控 | 参数 | 作用 |
| --- | --- | --- |
| 短期动量过滤 | `short_lookback_days = 10`, threshold `0.0` | 近 10 日收益为负时排除，避免追入短期转弱标的。 |
| RSI 过滤 | `rsi_period = 6`, `rsi_threshold = 95`, lookback `1` | 近 1 日 RSI 超高时过滤过热标的。 |
| 近 3 日跌幅过滤 | `g.loss = 0.97` | 任一日跌幅超过约 3% 时将 score 置 0。 |
| 异常高分过滤 | `0 < score < 5.0` | 排除可能由异常数据或极端行情导致的过高得分。 |
| ATR 动态止损 | `period = 14`, multiplier `2` | 持仓期间若现价低于 ATR 止损价则清仓。 |
| 固定止损 | `stop_loss = 0.95` | 现价低于成本 95% 时清仓。 |
| 防御 ETF | `511880.XSHG` | 无合格候选时尝试切换到防御资产。 |

MA、MACD、成交量异常、布林带过滤在脚本中有实现，但本次版本默认关闭。这个细节很重要：入库时不能把“代码里存在的过滤器”误认为“本次回测实际启用的过滤器”。

候选从 ETF 池进入可交易排名，实际经过这些步骤：

1. 数据不足则返回 `None`，该 ETF 当日不参与排名。
2. 计算 MA、RSI、MACD、成交量、布林带、短期动量和长期动量指标。
3. 近 3 日任一日价格比小于 `g.loss = 0.97`，则把 `score` 置为 0，而不是直接中断函数。
4. 若启用短期动量过滤且 `short_return < 0`，排除。
5. 若启用 MA 过滤且 MA 条件不满足，排除。本次默认关闭。
6. 若启用 RSI 过滤且近 `g.rsi_lookback_days` 日 RSI 超过 `g.rsi_threshold`，排除。当前代码中 `current_below_ma5 = True`，所以“当前价低于 MA5”这个条件在本版本等同于恒真；实质上只要 RSI 触发阈值就会过滤。
7. 若启用 MACD 过滤且条件不满足，排除。本次默认关闭。
8. 若启用成交量异常过滤且异常，排除。本次默认关闭。
9. 若启用布林带过滤且满足“近 N 日突破上轨且当前价低于 MA5”，排除。本次默认关闭。
10. 最后要求 `0 < score < g.max_score_threshold`，默认 `max_score_threshold = 5.0`。score 为 0、负数或过高都会被排除。
11. 剩余候选按 `score` 从高到低排序。

### 买入策略

买入并不是看到任意正信号就买，而是按“单目标 ETF 全仓轮动”执行：

1. 每日 14:29 调用 `etf_trade(context)`。
2. 先用 `get_ranked_etfs()` 得到过滤后的候选列表。
3. 如果候选列表非空，且第一名 `score >= g.min_score_threshold`，选择第一名作为 `target_etf`，进入 `normal` 模式。本版本 `min_score_threshold = 0.0`。
4. 如果没有合格候选，检查防御 ETF `511880.XSHG` 是否可交易；可交易则进入 `defensive` 模式，目标为防御 ETF。
5. 如果防御 ETF 不可交易，进入 `cash` 模式，目标列表为空。
6. 目标持仓金额 `target_value = context.portfolio.total_value`，即只要有目标 ETF，就以总资产作为目标金额。
7. 如果目标 ETF 当前市值与目标金额偏离超过 5%，或者当前无持仓，则调用 `smart_order_target_value(etf, target_value, context)` 调仓；如果偏离在 5% 容差内，则不下单。

买入前的下单保护：

| 检查 | 行为 |
| --- | --- |
| 停牌 | 跳过交易 |
| 当前价达到涨停价 | 跳过交易 |
| 当前价为 0 | 跳过交易 |
| 目标股数不足 100 股但目标金额大于 0 | 至少买 100 股 |
| ETF 数量 | 目标股数按 100 股整数倍取整 |
| 交易金额低于 `g.min_money = 5000` | 跳过交易 |
| 目标股数等于当前股数 | 跳过交易 |

注意：策略使用 `order(security, amount_diff)`，不是 `order_target_value`。它先把目标金额折算成目标股数，再下差额股数。

### 卖出策略

卖出来源有三类：换仓卖出、固定止损卖出、ATR 止损卖出。

#### 1. 换仓卖出

在 14:29 调仓阶段，确定目标 ETF 后，策略遍历当前持仓：

- 如果持仓标的属于 `g.etf_pool`，但不在 `target_etfs` 中；
- 且当前持仓数量大于 0；
- 则调用 `smart_order_target_value(security, 0, context)` 卖出至 0。

卖出成功后清除该标的的 `g.position_highs` 和 `g.position_stop_prices` 记录。

#### 2. 固定止损卖出

在 14:29 调仓函数中，选出目标 ETF 后、计算目标仓位前，策略检查当前持仓：

```text
if current_price <= cost_price * g.stop_loss:
    sell_to_zero
```

默认 `g.stop_loss = 0.95`，即当前价低于成本价 95% 时清仓。固定止损不区分是否为防御 ETF；只要持仓在 ETF 池内且满足价格条件，就会尝试卖出。

#### 3. ATR 动态止损卖出

ATR 检查在每日 09:35 单独执行。计算方式：

```text
TR = max(high - low, abs(high - prev_close), abs(low - prev_close))
ATR = 最近 period 日 TR 的简单平均
```

默认参数：

- `g.atr_period = 14`
- `g.atr_multiplier = 2`
- `g.atr_trailing_stop = False`
- `g.atr_exclude_defensive = True`

当 `atr_trailing_stop = False` 时：

```text
atr_stop_price = cost_price - 2 * current_atr
```

当 `atr_trailing_stop = True` 时：

```text
atr_stop_price = position_high - 2 * current_atr
```

本回测版本默认不是跟踪止损，而是基于成本价的固定 ATR 止损。若当前价低于等于 ATR 止损价，则清仓。防御 ETF `511880.XSHG` 默认豁免 ATR 止损。

### 卖出前的执行保护

`smart_order_target_value` 对卖出也做保护：

| 检查 | 行为 |
| --- | --- |
| 跌停 | 跳过卖出 |
| T+1 可卖数量为 0 | 跳过卖出 |
| 卖出数量超过可卖数量 | 限制为 `closeable_amount` |
| 交易金额低于 5000 | 跳过交易 |

因此，策略逻辑上“应该卖出”不等于平台一定成交。知识库评估该策略时，必须同时看 `order_decision`、`order_result` 和交易导出中的状态。

### 止盈策略

本策略没有显式止盈规则。盈利退出主要来自：

- 排名变化导致换仓；
- 当前目标 ETF 市值偏离目标值超过 5% 时再平衡；
- 风险过滤使原持仓不再是目标；
- 止损规则触发。

因此它不是“达到某个盈利百分比卖出”的止盈策略，而是“趋势轮动 + 风险退出”策略。收益来自持续切换到当前得分最高的趋势资产，而不是固定止盈。

### 仓位与资金管理

- 目标仓位：有目标 ETF 时使用 `total_value` 作为目标金额，接近满仓。
- 持仓数量：`g.holdings_num = 1`，但实际逻辑也只构造一个 `target_etf`。
- 再平衡容差：目标 ETF 当前市值与目标金额偏离超过 5% 才调仓。
- 最小交易金额：低于 5000 跳过。
- 股数单位：按 100 股整数倍。
- 现金缓冲：没有主动预留现金缓冲。本次回测仍有 3 次可用资金不足订单失败，说明“全仓目标金额 + 市价/手续费/取整”会产生轻微超买风险。

## 风险与失效边界

1. 集中持仓风险：策略只持有一只 ETF，收益曲线高度依赖当期目标 ETF 的趋势延续。
2. 高换手风险：六年买卖成交额均接近 28.78 亿，手续费约 111.71 万；交易成本或滑点假设改变会影响结果。
3. 订单执行问题：日志中仍有 3 次可用资金不足订单失败，说明全仓目标金额在市价、手续费、取整后可能轻微超买。
4. 参数依赖：25 日趋势窗口、10 日短期动量、RSI 95、ATR 2 倍等参数需要样本外和滚动窗口测试。
5. 基准选择边界：策略基准为纳指 ETF，但策略池横跨商品、海外、港股和 A 股 ETF，超额收益解释要注意资产配置差异。
6. 回测不等于实盘：停牌、涨跌停、T+1、成交深度、ETF 折溢价和申赎机制都可能让实盘偏离回测。

## 复盘时必须问的问题

未来 Agent 复盘或优化这个策略时，不应只看总收益，而要逐项回答：

| 问题 | 需要看的证据 |
| --- | --- |
| 当天为什么买这个 ETF？ | `rebalance_universe_summary.top_candidates` 和 `target_selection.top_candidate` |
| 为什么没有买其他 ETF？ | `filter_summary.rejected_by_filter`、短期动量、RSI 和 score 边界 |
| 是主动卖出、换仓卖出还是止损卖出？ | `risk_event`、`rebalance_plan`、交易详情中的卖出行 |
| 是否真的成交？ | `order_result` 和交易详情 `状态` |
| 订单失败是否影响净值？ | `order_decision`/`order_result`、持仓收益、现金变化 |
| 收益来自胜率还是盈亏比？ | 交易详情 `平仓盈亏`、标的贡献表 |
| 策略是否依赖少数年份？ | 年度收益和月度收益 |
| 策略是否可实盘化？ | 现金不足订单、换手、滑点敏感性、ETF 流动性和折溢价 |

## 知识库结论

这个策略值得入库，原因不是“收益高”本身，而是它提供了一个完整样本：

- 一个社区策略如何在不改变策略语义的前提下增强审计日志。
- 一个 ETF 轮动策略如何把趋势强度与趋势稳定性合成打分。
- 一个高收益回测如何通过交易、持仓和日志三类导出进行证据核对。
- 一个策略案例如何同时沉淀策略卡、平台经验和后续 skill 模板。

后续如果要继续研究，应优先做：

- 滚动样本外测试：例如 2020-2022 训练观察，2023-2025 验证。
- 成本敏感性测试：提高滑点和手续费，看收益是否仍稳健。
- 现金缓冲实验：例如目标金额打 99% 折扣，验证能否消除可用资金不足订单失败。
- 参数扰动：lookback、RSI、ATR、短期动量窗口小范围变化后看排名和收益是否稳定。
