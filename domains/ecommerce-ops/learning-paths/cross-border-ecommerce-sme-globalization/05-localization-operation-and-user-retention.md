# 05｜本地化运营与用户留存

## 问题定义

本地化不是翻译文字，而是让海外消费者感觉产品、页面、支付、物流、客服和内容都“为我而设计”。

## 核心模型

**本地化运营 = 语言文化适配 × 本地团队/伙伴 × 节日营销 × 用户体验 × 售后服务 × 社群留存。**

## 操作流程

1. **语言本地化**：避免机器直译；产品描述、客服话术、FAQ、政策说明要符合本地表达。
2. **文化适配**：根据当地节日、宗教、礼赠习惯、颜色偏好、审美和禁忌设计内容。
3. **购物体验本地化**：语言、货币、支付方式、配送范围、到货承诺、退换货说明清晰。
4. **用户黏性设计**：个性化推荐、会员权益、邮件/社媒触达、UGC 和社群运营。
5. **售后服务**：本地化响应时间、退货便利性、清晰补偿规则，降低跨境不信任。
6. **动态调整**：根据评论、客服问题、转化漏斗和退货原因快速改产品、页面和服务。

## 判断标准

| 场景 | 好的本地化 | 坏的本地化 |
| --- | --- | --- |
| 文案 | 像本地人写的产品表达 | 生硬直译、语义错误 |
| 节日 | 按本地礼赠/自用场景设计 | 照搬中国大促日历 |
| 支付 | 支持本地常用方式 | 只提供陌生支付方式 |
| 物流 | 清楚承诺和跟踪 | 到货时间模糊 |
| 社群 | 用户参与和反馈闭环 | 只发促销信息 |

## 案例与证据锚点

书中提到 Lazada、阿里国际、速卖通、小红书等案例，用于说明文化适配、本地运营、退货服务和社区策略如何提升信任与留存。

## Agent 使用提示

给本地化建议时，不能只说“翻译成当地语言”。必须拆成页面、支付、物流、客服、内容、节日、社群和售后八个维度。
