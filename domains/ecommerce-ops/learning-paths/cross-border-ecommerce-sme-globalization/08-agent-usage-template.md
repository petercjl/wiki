## 诊断输入

向用户收集或明确假设：

- 品类、产品、价格带、毛利结构
- 目标国家/地区
- 当前渠道：国内电商、外贸、平台、独立站、线下
- 供应链能力：交期、 MOQ、质检、认证
- 预算：首批货、广告、物流、服务商、团队
- 团队能力：内容、投放、客服、英语/本地语、数据分析
- 风险约束：现金流、库存、知识产权、合规、税务

## 诊断步骤

1. 判断品类和企业是否适合跨境试点。
2. 选择目标市场和进入模式。
3. 设计产品本地化和认证清单。
4. 规划渠道组合：平台、独立站、社媒、达人、搜索、邮件。
5. 设计物流和售后体验。
6. 设计支付、结算、汇率和资金风险方案。
7. 建立 IP、合规、平台规则和风险预警清单。
8. 给出 30/60/90 天试点计划和验证指标。

```md
## 必须实时复核的信息
```

## 不得推断

- 不得把书中平台规则当作当前规则。
- 不得承诺某平台一定能入驻或放量。
- 不得忽略认证、税务、隐私、广告和平台合规。
- 不得用大企业案例直接套中小企业。

## 需要复核的当前平台信息

- 平台入驻规则、费用、类目限制、账号健康规则。
- 目标市场产品认证、税务、隐私、广告合规。
- 物流价格、清关时效、海外仓费用和退货政策。
- 支付费率、到账周期、汇率服务、资金冻结规则。

## 来源与边界

来源锚点：全书。此模板是 Agent 应用入口，不替代具体市场实时调研。
