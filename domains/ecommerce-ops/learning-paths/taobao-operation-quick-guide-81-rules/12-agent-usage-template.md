# Agent 使用模板：淘宝店铺运营诊断

## 先读哪些页面

1. 店铺方向不清：读 [[02-category-positioning-and-product-selection]] 和 [[11-taobao-operation-success-model]]。
2. 新品推不动：读 [[03-product-development-and-new-product-testing]]、[[04-search-title-and-keyword-operations]]。
3. 点击和转化差：读 [[05-main-image-detail-page-and-selling-point-planning]]。
4. 流量渠道混乱：读 [[06-paid-traffic-and-channel-operations]]。
5. 复购和服务差：读 [[07-customer-service-private-domain-and-repurchase]]。
6. 活动和价格问题：读 [[08-promotion-pricing-and-major-campaigns]]。
7. 后端承接差：读 [[09-supply-chain-warehouse-and-finance]]。
8. 团队执行问题：读 [[10-team-organization-and-performance]]。

## 诊断输入

Agent 至少需要：品类、Top 商品、价格带、销量/收货人数、主图/详情页、关键词或流量来源、活动信息、客服/售后反馈、库存发货、团队分工。缺少平台实时数据时，必须标注推断边界。

## 输出结构

1. 一句话判断：当前最大约束是定位、商品、搜索、视觉、渠道、客服、促销、供应链还是团队。
2. 证据：引用用户提供的数据、图片、标题或业务描述。
3. 来源模型：说明调用了本 learning path 哪些页面。
4. 诊断：按原因链解释，而不是只列建议。
5. 行动：给出 7 天、30 天、90 天动作。
6. 复核：列出必须用当前淘宝/天猫后台确认的规则和数据。

## 不得推断

不能凭本书直接推断当前淘宝搜索算法、直通车出价规则、直播排名规则、营销活动报名规则、最低价规则或生意参谋指标口径。这些必须用当前平台资料复核。
