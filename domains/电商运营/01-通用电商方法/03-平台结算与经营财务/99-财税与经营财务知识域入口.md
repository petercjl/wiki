# 财税与经营财务知识域入口

本页是电商运营知识域通往 [[domains/财税与经营财务/domains/视觉制作/05-小红书风格AI生图/index|财税与经营财务知识域]] 的桥接入口。它不是旧 learning path 的残留，而是为了保留人的记忆关联：电商运营问题经常会延伸到收入确认、利润核算、税务合规、股权激励和战略财务，但这些知识的长期家园不应该埋在电商运营目录里。

## 什么时候留在电商运营域

如果问题主要是在回答“电商业务如何运营、平台如何结算、账单如何核对、数据从哪里来”，优先留在电商运营域。

- [[domains/电商运营/01-通用电商方法/03-平台结算与经营财务/01-电商平台结算时间口径财务核算方法|电商平台结算时间口径财务核算方法]]：平台结算时间确认收入、平台账单核对、快麦成本匹配、保证金、推广费和特殊账户处理。
- [[domains/电商运营/01-通用电商方法/02-平台渠道与入驻合作/domains/视觉制作/05-小红书风格AI生图/index|平台渠道与入驻合作]]：进入平台渠道时的合作模式、货权、费用口径和适配判断。
- [[domains/电商运营/30-ERP与系统工具/01-旺店通开放平台API/domains/视觉制作/05-小红书风格AI生图/index|旺店通开放平台 API 知识库]]：通过 ERP/API 获取订单、库存、货品、售后和采购数据。

## 什么时候跳到财税与经营财务域

如果问题主要是在回答“这套经营数据如何形成财务报表、如何合规、如何设计组织和激励”，应跳到财税与经营财务知识域。

- [[domains/财税与经营财务/01-电商财税合规/02-电商税务合规路径与红线|电商税务合规路径与红线]]：判断业务是否踩到税务合规红线。

- [[domains/财税与经营财务/02-经营财务与业财一体/03-平台利润表与边际贡献率|平台利润表与边际贡献率]]：按平台、店铺、品类和费用结构看经营利润。

## 记忆关系

电商运营域保存的是“业务发生了什么、平台怎么结算、数据怎么拿”；财税与经营财务域保存的是“这些业务如何被确认、归集、合规、分析和激励”。两者应互链，而不是互相吞并。这样既保留旧知识的上下文，也让新知识成为更常用的综合入口。
