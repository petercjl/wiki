# 03｜选品库与月度打款机制

选品库不是资料表，而是公司每个月增长动作的管理对象。它要把“看到了一个机会”变成“谁负责、怎么打、打到什么结果”。

## 选品库应该记录什么

| 字段 | 作用 |
| --- | --- |
| 候选款/品类线 | 明确本月可能打的对象 |
| 市场体量 | 判断空间是否足够 |
| 竞品链接 | 明确直接对标对象 |
| 竞品销售额/价格/毛利 | 判断可打空间和利润承受力 |
| 竞品主图/标题/卖点 | 拆解点击和转化来源 |
| 评价/晒图/问大家 | 挖掘真实需求和顾虑 |
| 我方优势/劣势 | 判断不是“想打”，而是“能不能打” |
| 路径动作 | 产品、视觉、上新、评价、付费、供应链动作 |
| 负责人和周期 | 避免只讨论不执行 |

## 空间-条件-路径

课程中最重要的选品评审框架是“空间、条件、路径”。

1. **空间**：市场体量是否足够，竞品销售额是否证明需求存在。
2. **条件**：我们是否具备产品、价格、供应链、视觉、链接数量、评价和投放能力。
3. **路径**：如果要打，先做哪些链接，先解决哪个卖点，先补哪些评价，投放从哪个阶段开始。

只有空间没有条件，是诱惑；只有条件没有空间，是内耗；有空间、有条件但无路径，会变成会议共识而不是增长结果。

## 竞品链接数和品类线完整度

课程中特别强调：要看对手店铺的链接数和品类线完善度。如果对手有数百条甚至上千条链接，而我们只有很少链接，就不能简单说“我也打这个品类”。

判断逻辑：

- 对手只有几十条链接，说明覆盖度有限，小团队可以通过集中上新和对标快速追赶。
- 对手已有大量链接和完整品类线，说明竞争不是单款竞争，而是品类线和店铺权重竞争。
- 我方需要判断能否在短期内补齐链接、评价、图片、价格和供货，否则会被对手矩阵压住。

## 月度选款会

建议形成固定节奏：

1. 运营先从选品库中筛出候选款。
2. 老板或负责人带运营复盘旧品数据：优势、劣势、空间、条件、路径。
3. 共同确认本月主推品类线和 AB 款。
4. 给每个链接打标：需求链接、测试链接、AB 款、重点打款、放弃款。
5. 每天摘录数据、反馈问题、由负责人解决卡点。
6. 月底复盘：哪些款进入下一阶段，哪些款停止投入，哪些问题进入产品或供应链改造。

## 选品库和人效

选品库最终要服务人效和利润，而不是服务“上了多少链接”。如果运营长期只铺需求链接，不进入 AB 款打款、转化优化和利润产出，就会形成低效忙碌。
