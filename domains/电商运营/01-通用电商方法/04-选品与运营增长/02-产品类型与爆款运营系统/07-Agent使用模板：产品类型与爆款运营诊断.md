# Agent 使用模板：产品类型与爆款运营诊断

当用户要诊断一个电商店铺、产品线、爆款打造计划或投放方向时，先按产品类型和生命周期判断，而不是直接给投放建议。

## 必要输入

1. 产品类目和具体 SKU。
2. 产品更接近标品、非标品、半标品、高频、低频、高客单、ToB 还是 ToC。
3. 当前销售额、客单价、毛利率、复购率。
4. 关键词、搜索流量、推荐流量、内容/直播流量占比。
5. 点击率、转化率、收藏加购、询单率、客服响应。
6. 竞品 Top10/Top20 的销量、价格、主图、链接数和卖点。
7. 当前生命周期阶段：新品期、成长期、爆款期、衰退期。
8. 当前组织能力：客服、视觉、供应链、投放、数据复盘、助理执行。

## 诊断顺序

### 1. 定位

- 这个店铺/产品解决什么问题？
- 核心用户是谁？
- 企业优势是什么？
- 这个优势能不能落到链接、主图、价格、客服或供应链？

### 2. 产品类型

- 标品：重点看搜索、参数、价格、销量信任、点击率和转化率。
- 非标品：重点看场景、审美、推荐/内容、停留、收藏加购。
- 高频：重点看获客成本、复购和回本周期。
- 高客单：重点看信任、售后、停留、加购、询单。
- ToB 定制：重点看精准询单、咨询成本、客服响应和销售承接。

### 3. 行业结构

- 是否头部寡头。
- 是否品牌词主导。
- 是否可以切规格、参数、人群、场景、价格带。
- 是否属于动销趋势型。

### 4. 生命周期

- 新品期：验证需求。
- 成长期：建标签、提转化、递增支付人数。
- 爆款期：放大规模、抢份额。
- 衰退期：控费、清库存、老客收割、新品衔接。

### 5. 动作建议

输出动作时必须区分：

- 产品动作。
- 主图/详情页动作。
- 关键词/投放动作。
- 客服/服务动作。
- 组织排期动作。
- 数据复盘动作。

## 输出格式

```text
1. 当前阶段判断
2. 产品类型与行业结构
3. 当前最大短板
4. 应看哪些指标
5. 未来 7 天动作
6. 未来 30 天动作
7. 禁止动作
8. 需要补充的数据
```

## 禁止动作

- 不判断产品类型就直接给投放方案。
- 用爆款期指标考核新品期。
- 把 ToB 定制当作静默成交商品。
- 只看投产，不看标签、加购、询单和客服承接。
- 只模仿竞品图，不拆竞品背后的价格带、参数和用户问题。
- 只做低价引流，不设库存、关停和利润承接。
