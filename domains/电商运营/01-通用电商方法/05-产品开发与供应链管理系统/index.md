# 产品开发与供应链管理系统

本目录基于 **山峰组研修4班6-23/6-24课程**，把电商产品开发、供应商开发、打样品质、成本利润、库存补货和部门协同整理成一套可诊断的经营系统。

核心判断是：电商竞争进入同质化和高流量成本阶段后，单靠运营技巧很难持续增长。产品开发决定可卖性，供应链决定交付稳定性，库存和成本决定利润是否真实。

## 核心模型

```text
竞争困局
-> 产品开发思维
-> 市场机会判断
-> 供应商开发与分级
-> 谈判、打样、品质
-> 成本穿透与 SKU 角色
-> 库存周转、补货与部门协同
```

## 阅读顺序

1. [[01-产品开发思维与市场机会判断|产品开发思维与市场机会判断]]：从电商内卷、流量成本、产品是 `1`、五大产品开发原则和机会四象限理解为什么要从卖货转向产品开发。
2. [[02-供应商开发与分级管理|供应商开发与分级管理]]：识别源头工厂、贸易商和定制能力，建立供应商开发流程、评估维度和分级体系。
3. [[03-供应商谈判打样与品质管理|供应商谈判打样与品质管理]]：把定价、交付、质量、权益、OEM/ODM、自有品牌、打样和三检制度落成合作标准。
4. [[04-成本结构利润模型与SKU角色|成本结构利润模型与 SKU 角色]]：从净利、隐性成本、包装流程、AI 增效和 SKU 角色判断产品是否真赚钱。
5. [[05-库存周转补货与跨部门协同|库存周转补货与跨部门协同]]：用库存分类、周转率、安全库存、7/14/21 天补货阈值和部门数据同步控制断货与积压。
6. [[06-Agent使用模板：产品开发与供应链诊断|Agent 使用模板：产品开发与供应链诊断]]：给 Agent 诊断店铺产品开发、供应商、成本和库存问题时使用。

## 何时调用

当问题出现以下信号时，优先读取本目录：

- 店铺销售额能做但利润不清楚。
- 产品同质化严重，只能靠降价。
- 想开发新品、定制款、独家款或自有品牌。
- 供应商很多但没有分级和淘汰机制。
- 打样靠老板个人审美拍板。
- 爆款断货、滞销积压、库存周转不准。
- 运营、仓库、采购、财务互相甩锅。
