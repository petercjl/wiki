# 国际站后台十关键词运营系统

CH07 把阿里巴巴国际站后台重构为十个关键词：店铺、产品、客户、内容、营销、数据、商机、订单、资金、物流。它的价值不是背目录，而是让运营把分散后台板块串成一个可调用的运营系统。

这套框架也可迁移到其他电商平台：每个平台的后台名称不同，但大多数平台都会围绕店铺资产、商品供给、客户关系、内容承接、营销投放、数据诊断、商机/线索、订单交易、资金结算和物流履约组织功能。

## 十关键词地图

| 关键词 | 国际站后台对应内容 | 运营问题 | 常见动作 |
| --- | --- | --- | --- |
| 店铺 | 公司信息、店铺动态、模板、贸易记录等。 | 买家如何理解商家可信度和经营状态。 | 完善公司信息、店铺装修、贸易记录展示、实力信息。 |
| 产品 | 商品发布、商品管理、优品运营、行业定向征品、新品和认证商品等。 | 平台如何识别你的供给，买家能否找到合适产品。 | 发布产品、维护标题/属性/详情页、管理优品和新品。 |
| 客户 | 客户信息、客户分组、客户分类、客户营销。 | 如何沉淀和再触达潜在买家。 | 建立客户分层、跟进重点客户、设置客户营销动作。 |
| 内容 | 媒体中心、视频、直播等。 | 如何用内容增强产品和商家表达。 | 发布视频、直播素材、强化产品应用和商家能力展示。 |
| 营销 | 直通车、问鼎、顶展、橱窗、全域引擎、品牌直达、明星展播等课程提到的营销入口。 | 如何用付费或资源工具获取增量曝光和询盘。 | 选择合适营销工具，匹配预算、关键词、人群和产品。 |
| 数据 | 数据参谋、产品参谋、选词参谋、流量参谋、买家参谋等。 | 问题发生在哪个环节，下一步该改哪里。 | 看曝光、点击、询盘、买家、产品和流量来源。 |
| 商机 | 询盘、TM、商机线索等。 | 买家表达意向后如何承接。 | 分析询盘质量、响应速度、跟进链路和转化动作。 |
| 订单 | 订单、起草信保、交易数据等。 | 询盘如何进入成交和履约。 | 跟进订单状态、信保订单、交易数据和转化复盘。 |
| 资金 | 资金金融、付款到账、发票等。 | 交易资金是否正常流转，运营需要知道哪些节点。 | 查询客户付款、发票和资金状态；具体账务交由财务负责人。 |
| 物流 | 物流设置、发货、物流单、物流费用。 | 交付承诺和履约体验是否支撑成交。 | 设置物流费用、填写物流单、检查发货和履约信息。 |

课程还提到认证中心、商家成长和服务中心等板块。它们可作为支撑层：认证中心影响实力和信任展示，商家成长影响等级与成长任务，服务中心处理投诉、违规、知识产权、申诉和发票等事务。

## 从目录到系统

十个关键词之间不是并列清单，而是一条经营链：

```text
店铺/产品/内容 -> 营销/数据 -> 商机/客户 -> 订单 -> 资金/物流 -> 再回到数据复盘
```

运营学习时要避免“这里看一下、那里看一下”。每进入一个板块都要问：

1. 这个板块控制什么资产或行为？
2. 它影响曝光、点击、询盘、转化、成交、履约还是复购？
3. 它和哪些板块发生联动？
4. 当前店铺最短板的问题，需要先看哪个板块？

## Agent 诊断用法

当用户只说“国际站店铺效果不好”时，Agent 可以用十关键词快速定位：

| 用户现象 | 优先检查板块 |
| --- | --- |
| 曝光少 | 产品、数据、营销、店铺、认证中心。 |
| 点击少 | 产品标题、主图、关键词、内容、首页流量入口。 |
| 询盘少 | 产品详情页、店铺信任、商机、买家匹配、SEO 布局。 |
| 询盘不准 | 产品关键词、标题相关性、店铺偏好、客户画像、SEO。 |
| 有询盘不成交 | 客户、商机、订单、物流、资金、沟通效率。 |
| 不知道从哪里学 | 先按十关键词建立后台板块地图，再进入对应问题页。 |

## 使用边界
