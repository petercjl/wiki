# 店铺流量暴涨暴跌分析SOP

店铺流量暴涨或流量暴跌，不能只看当天数据。课程强调日数据随机性强，应优先用周对比、月对比，必要时再下钻到日级异常点。

分析顺序是：

```text
店铺综合数据
-> 店铺状态和风险
-> 流量渠道
-> 活动和平台资源位
-> 优品/爆品数量
-> 产品贡献
-> 直通车词、国家、人群变化
```

## 先看店铺综合数据

主链路仍然是：

```text
曝光 -> 点击 -> 访客 -> 询盘/TM -> 订单
```

暴涨或暴跌要先定位发生在哪一段。

| 现象 | 可能原因 | 下一步 |
| --- | --- | --- |
| 曝光跌，点击和访客同步跌 | 排名、资源位、活动、产品权益或渠道流量变化。 | 拆渠道和产品。 |
| 曝光不跌，点击跌 | 主图、标题、价格、排名位置、词品匹配或竞争变化。 | 拆高曝光产品和关键词。 |
| 点击不跌，询盘跌 | 访客国家、人群、详情页、价格/MOQ 或询盘入口变化。 | 拆国家、人群、承接。 |
| 询盘不跌，订单跌 | 询盘质量或业务承接问题。 | 联动业务复盘报价、回复和跟进。 |

## 排查店铺状态和风险

流量异常不一定来自运营动作，也可能来自店铺状态。

需要检查：

1. 星等级是否变化。
2. 是否存在侵权、违规、处罚或限权。
3. 发货率、回复率、保障力等指标是否低于平台要求。
4. 店铺类型、服务能力或活动资格是否变化。
5. 产品是否掉优品、掉爆品、掉橱窗或信息质量下降。

这些因素会影响搜索、推荐、活动和付费推广承接，不应等到产品层才发现。

## 拆流量渠道

如果总流量变了，下一步看哪个渠道变了。

| 渠道 | 重点判断 |
| --- | --- |
| 自然搜索 | 关键词排名、搜索指数、产品层级、优品/爆品状态是否变化。 |
| 推荐流量 | 标签、买家喜好度、活动资源位、系统选品是否变化。 |
| 付费流量 | 预算、点击词、国家、人群、计划结构是否变化。 |
| 活动流量 | 是否参加或退出活动，活动是否结束，是否拿到资源位。 |
| 场景流量 | 榜单、新品、会采购等场景是否进入或掉出。 |

活动结束、平台大促结束、资源位奖励结束，都会造成看似“突然”的流量下降。

## 找到具体产品

店铺数据变化最终会落在具体产品上。要找到贡献暴涨或暴跌的产品，而不是平均看全店。

产品层按以下顺序拆：

1. 哪些产品贡献了流量变化。
2. 这些产品的曝光、点击、访客、询盘、TM、订单哪项变化最大。
3. 自然流量和直通车流量占比是否变化。
4. 关键词排名是否下降或搜索热度是否降低。
5. 是否失去优品、爆品、橱窗、活动或榜单资格。
6. 是否有同行进入同一关键词或活动场景并挤压位置。

如果一个产品原来 90% 来自自然流量，后来变成 70% 自然、30% 付费，说明自然能力可能已经下降，不能只看总访客还在。

## 直通车流量变化诊断

直通车层要做前后对比，不只看预算。

| 对比项 | 风险信号 |
| --- | --- |
| 点击词 | 原来 B/C/D 词高转化，后来 E/F/G 词高点击低询盘。 |
| 点击国家 | 原来美国、英国、澳大利亚等目标国，后来变成非目标国。 |
| 人群标签 | 原来高意向标签点击多，后来低意向标签消耗高。 |
| 产品组合 | 原来推高转化产品，后来预算被低承接产品消耗。 |

如果直通车点击增长但询盘下降，优先怀疑词、国家、人群偏移，而不是简单加预算。

## 复盘模板

每次出现流量异常，建议记录：

| 字段 | 内容 |
| --- | --- |
| 异常周期 | 周/月对比，不只写单日。 |
| 变化段 | 曝光、点击、访客、询盘、TM、订单。 |
| 变化渠道 | 自然、推荐、付费、活动、场景。 |
| 变化产品 | 贡献主要涨跌的产品链接。 |
| 变化原因 | 排名、权益、活动、计划、国家、人群、风险等。 |
| 处理动作 | 要改什么，什么时候改。 |
| 验证指标 | 改后看什么数据，多久看一次。 |
