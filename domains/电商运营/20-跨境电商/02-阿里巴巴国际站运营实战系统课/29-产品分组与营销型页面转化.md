# 产品分组与营销型页面转化

买家进入店铺后，不会耐心研究一个混乱的产品目录。产品分组、详情页和旺铺首页共同决定买家能否快速理解店铺、找到产品、相信供应商，并愿意发询盘。

## 产品分组原则

产品分组的目标是方便买家查找，同时突出核心品类和爆品。

### 不超过两级

分组不要做太多层级。课程建议分组不超过两级：第一层放核心品类，第二层做必要细分。层级过深会让买家找不到产品，也会削弱核心产品曝光。

### 重要品类放前面

旺铺前端通常只能优先展示前几个分类，因此第一层前 3 个位置要放核心主营品类、主推品类或高质量产品线。分组内部也要把优质产品、爆款、实力优品和主推链接排在前面。

### 按买家需求或公司品类归属分

分组可以按买家需求、使用场景、产品系列、材质、功能或公司内部品类归属组织。判断标准不是运营自己觉得整齐，而是买家能不能一眼找到想要的产品。

## 分组后台动作

在商品管理的商品分组排序里，先调整第一层和第二层分组顺序，再进入分组内部调整产品排序。每次调整后要保存，并回到前台刷新检查展示顺序。

```text
确定分组逻辑
-> 调整第一层分组顺序
-> 调整第二层分组
-> 把优质产品排到分组前排
-> 保存
-> 前台检查
```

产品分组页面也可以做装修，添加自定义模块和内容，让买家在进入分组后继续看到产品卖点、关联产品和转化引导。

## 营销型详情页

课程反复提醒，很多详情页是运营或设计师“自嗨”，并没有围绕买家需求组织。营销型详情页要先回答买家为什么需要、为什么信任、为什么现在发询盘。

详情页可包含以下模块，顺序按买家关注度调整：

| 模块 | 作用 |
| --- | --- |
| 优惠券/折扣 | 提升点击后的行动意愿，并带动更多产品曝光。 |
| 参数表和规格表 | 让买家快速确认尺寸、材质、功能、包装、MOQ 等信息。 |
| 产品细节 | 证明产品质量、工艺、使用效果和差异化。 |
| 关联产品矩阵 | 引导买家继续浏览新品、热销品和同系列产品。 |
| 包装、物流和定制能力 | 回答交付、样品、定制和运输疑虑。 |
| 生产过程和工厂场景 | 建立供应商专业度和履约信任。 |
| 公司实力、团队、证书 | 增强高客单、定制和长期合作信任。 |
| 询盘或回首页引导 | 在页面末端承接行动。 |

## 详情页制作方法

先找高转化同行详情页，再拆解其结构、文案、展示顺序、客户关注点和评论反馈。把核心内容、案例、卖点、工厂能力和模块顺序交给设计，而不是只让设计“做得好看”。

详情页图片不要做成一张超长大图。过长大图加载慢，可能导致买家跳出或数据异常。应切成多张短图，便于加载、替换和复用。

主推链接优化时要保留原模板。新详情页可以先在 3-4 个相似点击量的链接上测试，观察询盘和转化，再决定是否推广到更多产品。

## 营销型旺铺首页

旺铺首页是店铺门面，要让买家快速知道你卖什么、有什么优势、能解决什么问题、是否专业可信。RTS 店铺和定制店铺的首页结构应不同：RTS 更强调可交易产品、活动、订单和现货，定制店铺更强调解决方案、定制能力、工厂实力和服务。

旺铺首页可包含：

- 1-3 张首焦图，展示主产品、服务能力、公司优势和专业度。
- 优惠券、折扣和活动入口。
- 产品线、热销品、新品和主推分组。
- 企业实力、生产场景、团队和认证。
- 服务能力、交付能力和定制流程。
- 合作客户、品牌客户或案例。
- 橱窗模块和高质量产品展示。
- 视频营销模块。

旺铺装修时，从全球旺铺装修入口进入，打开产品、公司信息和动态模块，按页面目标拖拽模块、编辑内容、保存并前台检查。

## 诊断信号

| 现象 | 优先检查 |
| --- | --- |
| 首页访客不少但询盘低 | 首焦图、产品线、优势表达和信任模块是否清晰。 |
| 分组页访问后流失 | 分组是否过深，核心品类是否靠前，分组内产品是否优质。 |
| 详情页点击多但询盘少 | 参数、卖点、定制能力、价格、样品、物流和信任证据是否缺失。 |
| 旺铺看起来很完整但没转化 | 是否围绕买家需求，而不是围绕公司自我介绍。 |
