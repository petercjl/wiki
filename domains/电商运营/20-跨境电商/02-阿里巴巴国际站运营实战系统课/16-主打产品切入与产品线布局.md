# 主打产品切入与产品线布局

CH24 和 CH29 共同解决一个问题：店铺产品很多、预算有限时，应该先推什么。课程的答案是单点突破，先选少数公司有优势、市场有需求、容易成交的品类，通过直通车和订单绑定拉起全店流量。

不要把直通车预算平均撒给几十个品类。资源越有限，越要先找到主打产品切入点。

## 贸易型和工厂型的选品差异

| 店铺类型 | 核心任务 | 选品方式 |
| --- | --- | --- |
| 贸易型公司 | 先选行业，再选产品，依赖数据判断机会。 | 看市场容量、增长率、供应能力、榜单热品和竞争强度。 |
| 工厂型公司 | 主营类目通常固定，在既有产品内选功能、属性、款式和材料。 | 参考同行热卖款、关键词数据、客户需求和自身产能优势。 |

贸易型店铺更需要“行业选择 + 产品选择”；工厂型店铺更需要在固定赛道内找出能快速成交和快速积累数据的产品。

## 五个实战选品路径

| 路径 | 后台/数据来源 | 适合问题 |
| --- | --- | --- |
| 市场参谋选类目 | 查看行业、市场容量、增长率、买家和卖家分析。 | 不确定行业或类目机会。 |
| 榜单选品 | 参考平台榜单、热卖款、同行爆品。 | 需要快速找到市场验证过的款式。 |
| 流量参谋热门品类 | 查看类目热度、行业优品、流量趋势。 | 已有类目方向，想找更高流量产品。 |
| 买家参谋偏好 | 看目标买家行业偏好、订单买家、询盘买家和国家分布。 | 需要匹配真实买家需求。 |
| 优秀同行产品组 | 观察金品店铺或优秀同行的产品分组、标题、价格、MOQ 和详情结构。 | 需要建立参考样本和产品线结构。 |

课程提醒，这些路径是参考，不是照抄。尤其是小众行业，平台数据可能不完整，要结合同行、外部平台和公司历史订单一起判断。

## 主打产品选择条件

CH29 给出三条选择标准：

1. 公司有优势且容易成交：价格、供应量、交期、质量、款式、服务或历史成交证明有优势。
2. 关键词指数高、流量稳定、关键词总量多：说明市场容量足够，获取流量的可能性更高。
3. 符合当前市场迫切需求：季节性、采购周期和目标国家需求都要匹配，避免有点击无商机。

如果产品只是“有人看”，但当前阶段没有迫切采购需求，就不适合作为短期主推。

## 单点突破打法

```text
从 10-20 个品类中选 1-3 个或 1-5 个主推
-> 用直通车集中推广
-> 获取精准访客和商机
-> 及时绑定订单和补充交易数据
-> 打成爆品、优品或实力优品
-> 提升店铺权重
-> 反向带动其他品类自然流量
```

这套打法的关键不是“只卖一个产品”，而是先用有限资源把一个小范围打透。主打品类有商机后，要及时通过订单绑定、评价、交易数据和产品层级提升形成突破。

## 产品线布局

| 产品线角色 | 应承担的任务 | 运营动作 |
| --- | --- | --- |
| 主推线 | 打开流量和权重。 | 直通车集中预算、橱窗/资源绑定、订单绑定。 |
| 关联线 | 承接同一买家其他需求。 | 通过产品分组、详情页推荐和业务跟进关联销售。 |
| 利润线 | 提升成交质量和利润。 | 在主推线带来客户后由业务或页面引导。 |
| 测试线 | 验证新产品和新国家机会。 | 小预算测试曝光、点击、询盘和订单。 |

示例逻辑：目标买家可能同时需要 B、C、D、F 产品。运营可以先推 B/C，把精准买家拉进店铺，再通过页面和业务承接 D/F，而不是一开始四条线平均投放。

## Agent 使用提示

当用户的店铺品类很多但预算少时，先帮他做“主推品类筛选表”，字段包括：公司优势、关键词指数、流量稳定性、当前需求、历史成交、目标国家、供应能力、预计推广成本。
