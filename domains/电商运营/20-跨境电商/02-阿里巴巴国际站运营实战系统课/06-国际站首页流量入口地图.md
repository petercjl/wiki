# 国际站首页流量入口地图

CH08 的核心不是介绍首页长什么样，而是让运营识别买家从哪些入口进入平台、如何寻找产品，以及店铺怎样通过不同入口拿到流量。课程主要讲 PC 端首页，APP 端要求按 PC 端模板对照学习。

首页入口要从“流量模块”理解，而不是从页面美观理解。不同模块代表不同的买家路径和平台分发规则；如果运营不知道流量从哪里来，就很难判断应该优化产品、店铺、活动、榜单、类目还是制造商能力。

## 两大主入口

| 入口 | 买家路径 | 运营含义 |
| --- | --- | --- |
| 产品入口 | 买家通过关键词、类目、图片搜索或推荐进入具体产品。 | 产品标题、关键词、属性、详情页、价格、榜单和场景标签决定能否被匹配。 |
| 制造商入口 | 买家以供应商/制造商为对象浏览，一个卖家可带出多个产品。 | 店铺交易额、供应商实力、服务能力和店铺综合效果会影响展示机会。 |

一个店铺不应只在一个入口有表现。课程强调，产品入口和制造商入口都应尽量布局，因为它们相当于两条不同的流量渠道。

## 买家找产品的方式

买家可能通过多种路径找产品：

- 关键词搜索。
- 图片搜索，上传一款产品图寻找同类产品。
- 类目入口，一层一层进入细分类目。
- 系统基于搜索和浏览历史的推荐。
- 首页活动、榜单、新品、汇采购等场景入口。
- 制造商入口，从供应商卡片进入产品和店铺。

运营要理解买家的搜索习惯和浏览路径，才能知道产品标题、属性、类目、详情页、活动报名和店铺实力分别在什么地方起作用。

## 首页流量模块

| 模块 | 课程中的规则/现象 | 运营动作 |
| --- | --- | --- |
| 首焦图/活动图 | 首页有多张活动图，会随平台活动变化；大促或资源位可能给产品板块引流。 | 关注平台活动资源和报名机会，判断是否适合参与。 |
| 浏览历史与推荐 | 系统根据买家历史浏览和偏好推荐内容。 | 产品信息越准确，越有机会被推荐给相似偏好的买家。 |
| 热门榜单 | 包括爆款商品、热卖榜、好评榜等；课程提到爆款按询盘量、热卖按订单量、好评按评分排序。 | 根据榜单规则反推要提升的指标，避免只看销量或只看曝光。 |
| 细分类目榜单 | 类目会按属性、用途或产品类型继续拆分。 | 标题、属性和类目要准确，才能进入正确细分场景。 |
| 新品上架 | 符合平台新品规则的产品有机会进入新品展示模块。 | 新品发布要满足规则，属性和标题要让系统识别准确。 |
| 汇采购 | 通过报名、圈品和二次排序获得位置；课程提到靠前位置流量更好。 | 先判断报名条件，再看产品数据是否支撑排序。 |
| 180 天最低价等活动 | 符合条件并报名后可参与展示。 | 活动前核对价格、规则、利润和库存，避免只追资源位。 |
| 制造商场景 | 准时交货、快速响应时间、畅销品、爆款产品、趋势、新品发布等场景。 | 店铺中满足场景条件的产品越多，被系统圈选进入排序的机会越大。 |

## 产品发布和流量入口的关系

课程反复强调，产品属性、标题关键词和类目填写会影响系统分类是否准确，也会影响能不能进入榜单、场景或细分流量入口。

最小检查表：

1. 标题是否准确描述同一类产品。
2. 关键词是否覆盖核心词和相关长尾词。
3. 属性是否完整、正确、与标题和类目匹配。
4. 详情页是否强化同一产品，而不是混入多个产品信息。
5. 是否满足新品、榜单、活动或场景入口的规则。
6. 店铺是否具备制造商入口所需的交易、响应、交付和实力信号。

## 流量入口诊断

当店铺流量不足时，不要只问“要不要开车”。先问：

| 诊断问题 | 可能动作 |
| --- | --- |
| 产品是否能被系统准确分类？ | 检查标题、类目、属性和详情页一致性。 |
| 产品是否有机会进入榜单或场景？ | 对照询盘量、订单量、评分、新品规则、活动报名条件。 |
| 店铺是否只依赖产品入口？ | 检查制造商入口、实力标签、交易额、服务能力和店铺综合效果。 |
| 买家是否有多种搜索路径？ | 补足关键词、图片搜索相关性、类目入口和推荐入口。 |
| 当前资源位是否值得争取？ | 看报名门槛、排序规则、产品数据、利润和供给能力。 |

## 使用边界

首页结构、榜单规则、活动位置和制造商入口会随平台版本变化。课程提供的是“从首页识别流量入口”的方法，具体入口名称、排序指标和报名条件必须以当前后台和前台页面为准。
