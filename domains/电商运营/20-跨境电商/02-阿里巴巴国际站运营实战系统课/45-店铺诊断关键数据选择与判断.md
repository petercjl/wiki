# 店铺诊断关键数据选择与判断

店铺诊断要选对数据入口。只看总曝光、总访客或总询盘，容易把问题判断错。课程把诊断数据分成两个核心问题：国家和人群是否精准，词和产品是否匹配。

底层主线仍然是：

```text
曝光 -> 点击 -> 访客 -> 询盘/TM -> 订单
```

所有数据入口都服务于这条主线，目的是判断每一段断在哪里。

## 国家和人群精准度

从店铺维度看国家精准度，可用多个入口交叉验证。

| 入口 | 看什么 | 判断 |
| --- | --- | --- |
| 数据概览访客 | 访客国家分布。 | 访客排名靠前的国家是否是目标国家。 |
| 数据概览询盘 | 询盘国家分布。 | 询盘是否来自目标国家；若不是，询盘质量可能偏差。 |
| 买家参谋/访客画像 | TM、询盘、订单维度下的国家分布，建议看 30 或 90 天。 | 成交、询盘、沟通买家是否与目标国家一致。 |
| 流量参谋 | 流量承接及去向、国家地域偏好。 | 流量来源和承接人群是否偏离目标市场。 |

如果访客国家精准但询盘国家不精准，说明页面或产品可能吸引了目标国家浏览，却没有让目标国家买家行动；如果访客国家本身不精准，先纠正关键词、推广国家、人群和推荐标签。

## 词品匹配判断

产品维度要看高点击产品和点击词是否匹配。

判断入口：

1. 产品分析：看高点击产品对应的点击词。
2. 引流关键词：看高点击词是否真正匹配产品。
3. 关键词指数：看这些词在目标国家是否有搜索偏好。
4. 前台或 Google 搜索：辅助确认关键词对应的产品款式和买家预期。
5. 同行点击词：看优秀同行在哪些词上获得更多点击。

高点击不代表高价值。如果某词点击多但搜索指数低、目标国家偏好弱、产品款式不匹配，就不应持续重点推广。

## 用关键词指数验证国家偏好

关键词指数不仅用来找词，也可用来判断国家偏好。把引流关键词或高点击词放入关键词指数，观察国家地域搜索占比：

1. 目标国家占比高：词可进入标题、直通车和重点产品承接。
2. 非目标国家占比高：除非店铺准备拓展该国家，否则谨慎投放。
3. 搜索指数很低：即使有点击，也要判断是否偶然流量或小盘词。
4. 涨幅明显：可用于新品发布、标题补充和趋势产品测试。

## 必须理解的分层逻辑

课程提醒要理解卖家分层、买家分层、渠道分层和赛道场景。平台分层的目的，是让产品、买家、流量入口更精准匹配，提高转化和交易效率。

| 分层 | 运营含义 |
| --- | --- |
| 卖家分层 | 星级、店铺类型、能力分和经营数据决定可获得的流量机会。 |
| 买家分层 | 不同买家国家、采购规模、偏好和行为决定推荐和搜索匹配。 |
| 渠道分层 | 搜索、推荐、活动、付费和内容渠道有不同承接逻辑。 |
| 赛道场景 | RTS、定制、半托管、普通询盘品对应不同买家行为。 |

店铺诊断时，要把问题放回分层系统，而不是只看单个页面表现。

## 产品、市场、词和人群的关系

课程结论是：产品与市场是绑定的。一个产品在美国可能热销，在英国或东南亚可能没有效果。流量问题本质上经常是词、品、国家和人群的匹配问题。

可用以下问题复盘：

1. 产品是否符合目标市场需求。
2. 款式是否符合目标国家偏好。
3. 关键词是否准确表达产品和买家需求。
4. 产品是否匹配目标人群和采购规模。
5. 直通车推广是否把预算投给了精准词、精准国家和精准人群。
6. 询盘质量是否足以支持订单成交。

当知识点混乱时，可画一张图，把曝光、点击、访客、询盘、订单作为主干线，再把国家、词、产品、人群和渠道挂到对应节点上。这样诊断不会跑偏。

## 数据选择速查表

| 诊断问题 | 先看入口 | 再拆 |
| --- | --- | --- |
| 访客不精准 | 数据概览访客国家、买家参谋、流量参谋地域偏好 | 关键词、推广国家、人群标签。 |
| 询盘质量差 | 询盘国家、TM/询盘/订单买家画像 | 产品、国家、关键词、业务反馈。 |
| 高点击低询盘 | 产品分析、引流关键词、产品 360 | 词品匹配、价格/MOQ、详情页、目标国家。 |
| 推广花费无效果 | 直通车计划、关键词指数、产品 360 | 高点击低转化词、国家偏移、产品承接。 |
| 产品在不同市场表现差异大 | 国家市场偏好、关键词指数、产品偏好 | 国家、款式、价格带和页面表达。 |
