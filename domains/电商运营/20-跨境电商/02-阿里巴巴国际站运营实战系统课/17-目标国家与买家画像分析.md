# 目标国家与买家画像分析

CH25-CH26 把定位进一步落到两个问题：目标国家选哪里，产品卖给谁。课程的核心判断是：产品可以全球卖，不代表运营应该全球投。有限预算下，应先找到更有增量、更容易成交的国家和买家群体。

目标国家决定市场潜力，买家画像决定页面表达、推广匹配和业务跟进。

## 目标国家选择原则

课程提醒，一个店铺往往由少数 3-5 个国家贡献大部分交易额。全球铺开会导致精力、产品和直通车预算分散，最后很多国家有一点访客和询盘，却没有订单。

选择目标国家时看五类数据：

| 数据 | 作用 | 优先级 |
| --- | --- | --- |
| 订单买家国家 | 已经证明成交能力。 | 最高 |
| 询盘买家国家 | 证明存在需求和沟通机会。 | 高 |
| TM 买家国家 | 证明有即时咨询意愿。 | 中 |
| 访问买家国家 | 证明有流量兴趣，但不一定成交。 | 中低 |
| 市场参谋国家数据 | 判断行业在各国的容量和增长。 | 高 |

课程建议优先看订单买家，因为访问、询盘和 TM 不一定代表真实成交。

## 国家筛选流程

1. 在市场参谋进入行业分析，选择对应类目和国家维度。
2. 提取询盘买家、交易买家等国家分布，放入表格并降序排序。
3. 在买家参谋查看 30 天或 90 天订单买家、询盘买家、TM 买家国家。
4. 找交集：订单、询盘和 TM 都较强的国家优先进入测试池。
5. 先保留 10-20 个国家测试，再根据成交和推广数据收敛到 5-10 个核心国家。

没有固定标准说必须选 10 个或 15 个国家。关键是在有限资源里，找到更高概率成交的国家。

## 国家数据如何指导运营

| 运营动作 | 国家选择的影响 |
| --- | --- |
| 直通车投放 | 决定投放国家、人群和预算集中度。 |
| 产品选择 | 不同国家对价格、MOQ、款式、功能和认证要求不同。 |
| 页面表达 | 目标国家关注的痛点、物流、清关、交期和案例不同。 |
| 业务跟进 | 报价、沟通时差、样品策略和交付承诺不同。 |
| 店铺标签 | 长期精准国家成交会强化系统匹配，反之会让推荐流量变差。 |

## 买家画像分析

CH26 的核心问题是：如果不知道产品卖给谁，就无法写出让买家愿意询盘的首页、主图和详情页。

买家画像至少包括：

| 画像字段 | 需要回答的问题 |
| --- | --- |
| 买家类型 | 批发商、零售商、品牌商、工程商、代理商，还是终端采购？ |
| 国家/地区 | 主要在哪些市场采购？价格和交付习惯怎样？ |
| 采购痛点 | 他为什么需要这个产品？当前供应商哪里不能满足？ |
| 决策因素 | 价格、MOQ、交期、认证、质量、定制、售后哪项最重要？ |
| 成交门槛 | 买家最担心什么？信任、样品、质检、付款还是售后？ |
| 页面需求 | 他进入详情页最希望先看到什么证据？ |

## 买家画像数据来源

1. 公司线下历史客户和成交资料。
2. 业务员和销售的客户沟通记录。
3. 后台询盘、客户列表、TM 和订单买家。
4. 同行详情页、评论和高转化页面。
5. 外部电商平台评论中的高频需求和差评痛点。

画像不是写完放着，而要用于旺铺首页、详情页、主图、直通车推广和业务沟通。页面没有呈现买家想看的解决方案，就容易出现“有访客、无询盘”。

## Agent 使用提示

诊断询盘质量时，把目标国家和买家画像一起看。如果国家不精准，买家画像会失真；如果画像不清楚，页面和推广即使投对国家，也难以转化。

## 使用边界

国家和买家数据必须以当前后台和店铺真实成交为准。本页提供筛选方法，不替代行业市场调研和业务判断。
