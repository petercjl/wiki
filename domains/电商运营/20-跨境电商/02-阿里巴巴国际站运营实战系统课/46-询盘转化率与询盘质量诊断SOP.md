# 询盘转化率与询盘质量诊断SOP

询盘问题要分成两个层面：询盘转化率低，说明访客进入后没有产生足够咨询；询盘质量差，说明产生了咨询，但买家、国家、需求或业务承接不适合成交。两个问题不能混在一起判断。

课程给出的主线是从大到小拆：

```text
店铺流量层级
-> 产品流量层级
-> 直通车/P4P 流量层级
-> 关键词、国家、人群、价格/MOQ、页面承接
```

## 询盘转化率低的店铺层诊断

先不要直接改详情页。要先确认流量是否值得承接。

| 诊断项 | 看什么 | 常见结论 |
| --- | --- | --- |
| 引流关键词是否精准 | 引流关键词、产品分析、关键词指数。 | 词与产品不匹配时，点击越多，询盘转化越低。 |
| 访客国家是否是目标国家 | 数据概览、买家参谋、流量参谋。 | 非目标国家访客多，询盘少或询盘质量差。 |
| 价格和 MOQ 是否符合市场 | 榜单、买家参谋、商品洞察、同行价格带。 | 价格/MOQ 偏离目标买家预期，会卡住咨询和成交。 |
| 店铺是否流量太少 | 曝光、点击、访客基础量。 | 访客基数太小，先补流量，不要过早判定页面失败。 |
| 高点击产品是否低询盘 | 高点击、高访客、低询盘产品。 | 进入产品层继续拆词、国家、页面和价格。 |

如果店铺本身没有足够访客，优先做产品数量、优品/爆品、关键词排名、活动和推广；如果访客足够但询盘少，再进入承接质量诊断。

## 产品层诊断

高点击、高访客、低询盘的产品要重点分析。

1. 拆自然流量和直通车流量占比。自然流量持续增长，才更容易在预算固定时拉动总流量。
2. 看点击词是否真正匹配产品款式、属性、价格和目标国家。
3. 看访客国家是否与目标国家一致。
4. 看价格、MOQ、运费、样品和交期是否让买家愿意发起咨询。
5. 看详情页是否回答买家关心的参数、场景、认证、服务、案例和下单方式。
6. 看产品是否仍有优品、爆品、橱窗、关键词排名等平台权益。

如果头部 3-5 个产品贡献主要流量，要特别关注这些产品的关键词排名稳定性和关键词搜索指数。头部产品掉排名，会直接传导到店铺询盘。

## 直通车层诊断

直通车不是只看花费和点击。要拆高点击、高消耗背后的词、国家和人群。

| 入口 | 判断 |
| --- | --- |
| 高点击词 | 是否与产品、目标国家和买家意图匹配。 |
| 高消耗国家 | 是否是目标市场，是否产生询盘、TM 或订单。 |
| 人群标签 | L1+、意向商机、目标行业买家占比是否足够。 |
| 产品承接 | 推广产品是否适合被推广国家和人群点击。 |

如果直通车把预算花在不精准词、非目标国家或低意向人群上，询盘转化率会下降，还会污染产品和店铺标签。

## 如何判断询盘质量

不能只凭业务一句“质量差”下结论。询盘质量至少看四个维度。

| 维度 | 判断方法 | 注意 |
| --- | --- | --- |
| 询盘来源国家 | 是否来自目标国家和重点市场。 | 非目标国家即使账号等级高，也可能不是当前店铺的高质量询盘。 |
| 买家账号层级 | L1+ 通常比普通买家更有价值，L2-L4 更可精细判断。 | L1+ 不是绝对高质量，仍需看国家和需求。 |
| 买家采购类目 | 买家主营类目是否与产品匹配。 | 如果买家采购范围很泛，而本品只是边缘类目，成交概率可能低。 |
| 买家回复和互动 | 是否已读、是否回复、是否继续沟通。 | 高等级买家不回复，可能是业务话术、报价或产品承接问题。 |

询盘质量诊断必须让运营、业务和负责人共同参与，也就是把“业务协作”纳入诊断流程。运营负责流量、人群、产品和页面，业务负责回复质量、报价、跟进节奏和成交反馈，负责人负责资源和方向取舍。

## 询盘质量差的常见原因

1. 店铺星等级太低，高质量买家的匹配机会和占比不足。
2. 店铺、产品、关键词、国家标签混乱，推荐流量偏向非目标买家。
3. 产品选择没有围绕目标国家偏好，款式和价格带不适合核心市场。
4. 用过低价格或过低 MOQ 做引流，吸引了难成交买家。
5. 直通车投放国家和人群过宽，把预算打给非目标流量。
6. 业务承接弱，高质量询盘没有被有效回复、报价和跟进。

## 修复询盘质量的动作

店铺层先统一定位：定制、RTS、半托管或普通询盘品要清楚，价格/MOQ 应接近真实可成交区间，目标国家和核心人群要稳定。

产品层优先处理把店铺标签拉偏的产品。如果某些产品流量和询盘很多，但主要来自非目标国家，要判断是优化详情页、调整款式表达、暂停推广，还是下架重发。自然流量下降不一定是坏事，如果下降的是低质量非目标流量，反而可能让后续标签更干净。

推广层要锁定目标国家、目标词和目标人群。对低质量高点击标签降低出价，对高意向、高转化标签追加测试。若一个计划长期只有低质点击、国家不准、无意向商机，优先暂停并重建，而不是继续加预算。

订单层只能绑定真实客户、真实订单和合规线上交易。课程强调订单反馈会影响系统判断，但正式执行时必须避免虚假交易、虚假绑定或误导性数据。
