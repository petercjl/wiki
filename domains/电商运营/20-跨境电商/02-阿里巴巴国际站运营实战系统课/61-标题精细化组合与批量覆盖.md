# 标题精细化组合与批量覆盖

CH100 和 CH112 把前面标题规则落成一个可批量执行的方法：先确定多个核心结尾词，再用搜索指数从低到高的属性词往前填，批量生成低重复度标题。

## 核心公式

标题从左到右的搜索指数逐步上升：

```text
低指数属性词 -> 中指数精准词 -> 高指数核心产品词
```

新标题不要一开始就追求 128 个字符。课程建议多数新标题先控制在 100 个字符以内，甚至 60-80 个字符也可以，后续根据数据再增加关键词。

## 批量组合步骤

1. 选定同类或同款产品。
2. 用词根在关键词指数里搜索，导出相关关键词。
3. 从第一页、第二页优先挑选近 12 个月流量稳定的大词作为结尾核心词。
4. 如果要做 5 个标题，就确定 A/B/C/D/E 五个结尾词。
5. 把属性词、规格词、功能词按搜索指数从低到高排列。
6. 对每个结尾词向前填不同属性词，形成多条标题。
7. 检查多条标题之间重复度，尽量不要只换一个词。
8. 将标题放入不同链接或推广计划中测试覆盖流量。

示意：

```text
属性词1 + 属性词2 + 属性词3 + A核心词
属性词4 + 属性词5 + 属性词6 + B核心词
属性词7 + 属性词8 + 属性词9 + C核心词
```

## 为什么低重复度重要

同款或同类产品如果标题高度重复，多个链接可能抢同一批关键词和同一批人群，覆盖范围并不会真正扩大。

低重复度标题的价值：

1. 扩大关键词覆盖面。
2. 更容易发现不同词组的真实转化。
3. 后期能筛出更匹配产品的点击词和询盘词。
4. 降低多个链接互相内耗。

## 结尾词选择标准

结尾词通常是核心产品词或核心流量词，应满足：

1. 与产品类目和实物高度匹配。
2. 在关键词指数中有稳定搜索数据。
3. 近 12 个月流量相对稳定。
4. 是买家真实会搜索的词，不是运营凭感觉造词。
5. 能作为标题最后的产品识别锚点。

如果关键词池足够大，可以每 5 个结尾词一组，连续生成多组标题，直到覆盖主要关键词池。

## 属性词填充标准

属性词应来自关键词指数、同行有效词、产品真实属性和买家需求，不应凭空编写。

优先使用：

1. 有搜索指数的材质、功能、风格、规格、场景词。
2. 与产品主图和详情页能对应的词。
3. 不同标题中可互补的词。
4. 搜索指数逐渐上升的词序。

谨慎使用：

1. 纯营销词。
2. 没有搜索数据且无法证明买家会搜的词。
3. 与产品不匹配但看起来流量大的词。
4. 与其他标题高度重复的词。

## 实战检查表

发布前逐条检查：

1. 标题是否描述同类或同款产品。
2. 结尾核心词是否都是真实产品词。
3. 属性词是否按搜索指数从低到高大致排列。
4. 多条标题之间是否有明显差异。
5. 标题是否控制在 100 个字符以内。
6. 标题、K1/K2/K3、属性、详情页是否一致。
7. 是否能在推广后根据曝光、点击、询盘和排名词复盘。

## 常见误区

1. 只用一个核心词，复制出多条几乎一样的标题。
2. 为了写满字符而堆无搜索指数词。
3. 结尾词不是产品词，而是过宽类目词。
4. 只看词量，不看词品匹配。
5. 一开始就追求完美，迟迟不发布测试。
