# 店铺诊断框架与综合数据SOP

店铺诊断的目标是找到流量、转化和成交问题，并把问题转成具体修复动作。不能一上来就问“为什么没有询盘”，因为没有询盘可能来自没访客、访客不精准、产品承接差、询盘质量差或业务转化弱。

课程给出的核心方法是：先定位问题，再拆指标，再回到产品、关键词、国家、人群和推广计划。

## 诊断前先问四个问题

| 问题 | 判断含义 | 下一步 |
| --- | --- | --- |
| 店铺有没有买家进来 | 先看点击和访客，不只看曝光。 | 如果没有点击/访客，先解决流量进入问题。 |
| 买家进来后是否产生询盘和订单 | 区分询盘转化问题和订单转化问题。 | 访客多询盘少看承接；询盘多订单少看询盘质量和业务转化。 |
| 买家从哪些渠道和关键词来 | 判断渠道、词和国家是否精准。 | 拆流量来源、引流关键词、产品和推广计划。 |
| 数据和谁比较 | 数据只有比较才有意义。 | 对比自己历史、行业平均、优秀同行和不同时间周期。 |

这四问决定诊断顺序。没有流量时先做流量；流量不精准时先纠偏；询盘质量差时先修人群和产品匹配；询盘质量好但订单少时再进入业务承接。

## 店铺诊断循环

诊断不是一次性动作，而是循环：

```text
定向找问题
-> 确认问题
-> 解决问题
-> 评估效果
-> 有效则复制
-> 无效则回到问题定位
```

每次动作都要绑定验证指标。例如换主图看点击率，换关键词看点击和询盘质量，调整国家看访客国家和询盘国家，调整详情页看询盘转化和 TM 质量。

## 两个诊断层级

### 店铺布局和平台趋势

这层决定店铺流量天花板，常见问题包括：

1. 店铺赛道不清：RTS、定制、半托管混杂，资源分散。
2. 目标国家和人群不清：推广和页面没有明确承接对象。
3. 产品结构过窄：普通询盘品、询盘可交易品、RTS 或半托管产品覆盖不足。
4. 星等级过低：影响高质量买家分配、排序权重和活动机会。
5. 类目覆盖太少：只做少数类目，导致流量池太窄。

如果店铺布局错了，局部优化只能缓解，不能真正突破天花板。

### 店铺综合数据

综合数据按主链路分析：

```text
曝光 -> 点击 -> 访客 -> 询盘/TM -> 订单
```

常用指标：

| 指标 | 公式或含义 | 诊断方向 |
| --- | --- | --- |
| 点击率 | 点击 / 曝光 | 买家是否愿意点，受主图、标题、价格、排名和词品匹配影响。 |
| 访客点击比 | 访客与点击之间的关系。 | 判断自然访客、重复访问和流量承接质量。 |
| 询盘转化率 | 询盘 / 访客 | 判断访客是否精准，产品页是否能承接。 |
| 订单转化率 | 订单 / 询盘或访客 | 判断询盘质量、价格预期和业务跟进。 |

课程提到一些店铺点击率、询盘转化率参考范围，但这些数字会随行业、类目、时间和后台口径变化，实操时应以当前后台、行业平均和优秀同行为准。

## 综合数据分析顺序

建议按“大到小”拆：

```text
店铺综合数据
-> 产品数据
-> 直通车/P4P 数据
-> 关键词、国家、人群、计划
```

如果店铺流量下滑，不要只看店铺总访客。要继续问：

1. 是自然、付费还是推荐下滑。
2. 是哪些产品贡献下滑。
3. 是产品曝光下滑、点击下滑、访客下滑，还是询盘下滑。
4. 是哪些关键词、国家、计划或人群标签变化。
5. 是否与类目变化、星级变化、产品掉级或同行竞争有关。

## 流量增长动作

店铺流量增长可以从以下方向拆：

| 方向 | 动作 |
| --- | --- |
| 关键词数量 | 增加有曝光、有点击、有询盘的有效关键词数量。 |
| 关键词排名 | 将高价值词从靠后位置推到更靠前位置。 |
| 产品线 | 扩充有市场机会的品类和产品线。 |
| RTS/半托管 | 适合时增加可交易入口，打开不同流量池。 |
| 资源位 | 使用橱窗、问鼎、顶展、直通车等放大重点产品。 |
| 优品/爆品 | 增加有平台权益和数据沉淀的产品总量。 |

## 转化增长动作

转化提升要优先保证精准：

1. 关键词精准：词是否对应买家真实需求。
2. 国家精准：访客国家和目标国家是否一致。
3. 人群精准：访客是否为目标买家，而不是泛批发商或非核心品类买家。
4. 产品精准：款式、价格、MOQ、运费是否符合目标市场。
5. 页面承接：详情页、首页、公司优势和信任证据是否清楚。

只有流量精准后，主图、详情页和优惠动作才更容易有效。
