# 标题底层逻辑与搜索抓取

CH93 文件名像直通车学习指南，但实际转写内容与 CH94 基本一致，都是“标题的底层逻辑：排序、权重、系统抓取”。因此正式页合并两节内容，保留两份来源。

## 标题是第一匹配要素

课程把标题定义为产品名称，也是买家搜索时系统最先匹配的核心文本。

标题的作用：

1. 告诉系统产品是什么。
2. 承接买家的搜索词。
3. 决定产品能参与哪些关键词排名。
4. 影响产品曝光、点击和后续数据沉淀。

买家通过关键词表达需求，系统先从产品标题中提取关键词组合，再结合类目、K1/K2/K3、详情页文本、店铺权重和单品权重排序。

## 关键词、需求和标题的关系

课程给出四个概念：

1. 关键词等于产品信息。标题里的词，本质是在告诉系统和买家产品信息。
2. 关键词也是买家需求表达。不同关键词背后代表不同需求。
3. 标题是多个关键词组成的集合。一个标题可以拆出多种搜索组合。
4. 标题是详情页中最核心的搜索权重文本。

所以标题不是把词塞满，而是把买家会搜索、产品能承接、平台能识别的词组合好。

## 权重不等于排名

权重表示某个指标在平台系统中的相对重要程度。标题权重高，说明标题对搜索匹配和排序影响大。

但要区分：

1. 权重不等于排名。权重高只是更容易获得排序优势。
2. 权重对排名有加分作用。同等条件下，权重高的产品更容易靠前。
3. 店铺权重提高，会提升全店产品和关键词的排序基础。
4. 单品权重提高，会增强该产品对应关键词的排序能力。

标题是重要因素，但不是唯一因素。类目、产品质量、买家喜好度、店铺综合实力和交易/询盘数据也会参与。

## 产品 ID 与标题优化

课程用产品链接中的 ID 解释：产品 ID 存储曝光、点击、询盘、订单、访客等数据。修改标题通常不会改变产品 ID，但会影响这个 ID 后续匹配到哪些关键词和流量。

因此：

1. 爆品或有效产品的标题不要随意大改。
2. 不要轻易删除已经带来流量和排名的关键词。
3. 标题优化前先看引流关键词、询盘词、排名词和成交词。
4. 新品标题可以大胆测试，老爆品标题要小步验证。

## 系统抓取顺序

课程给出的抓取优先级：

```text
标题关键词 -> K1/K2/K3 -> 详情页文本 -> 店铺权重/单品权重加分
```

理解为三层匹配：

1. 第一层：标题中是否包含买家搜索词或可拆分组合词。
2. 第二层：K1/K2/K3 是否补充相关关键词，店铺和单品权重是否足够。
3. 第三层：详情页文本是否也有相关内容，进一步增强匹配和相关度。

如果标题、K1/K2/K3、详情页都围绕同一产品和同一需求表达，系统更容易判断产品相关，排名也更容易获得加分。

## 标题诊断问题

诊断一个标题，先问：

1. 标题里是否有买家真实会搜索的词？
2. 标题是否能拆出多个有效关键词组合？
3. 标题和类目、属性、详情页是否一致？
4. K1/K2/K3 是补充标题，还是重复堆词？
5. 产品当前已有排名词是否被保留？
6. 店铺和单品权重是否足以支撑大词竞争？
