# 爆品总量提升与优品权益

爆品总量提升不是简单“多发产品”，而是从优品、准爆品、平台新品和已有数据产品里，找离爆品门槛最近的产品，集中补足曝光、点击、询盘、TM、订单和买家数。它承接前一页的优品模型，也为后续橱窗和星等级提升提供产品池。

## 爆品判断口径

课程中给出的爆品门槛是后台规则型指标，不同类目、行业和时间会变化，实操前要以商品运营工作台当前规则为准。

常见判断逻辑：

| 产品类型 | 课程中的判断方向 | 实操注意 |
| --- | --- | --- |
| 定制产品 | 信息质量分约 4.2 以上，并在近 90 天积累足够询盘/TM 人数。 | 具体询盘/TM 阈值以后台准爆品规则为准。 |
| RTS 产品 | 信息质量分约 4.2 以上，并在近 90 天积累足够支付买家数。 | 不同类目对买家数和支付转化要求不同。 |
| 准爆品 | 离爆品门槛不远、已有一定询盘/TM/订单基础。 | 优先投入资源，因为晋级成本更低。 |

爆品是动态身份。课程反复强调近 90 天窗口：如果后续数据下滑，爆品可能掉回准爆品或普通产品，所以爆品运营必须持续维护。

## 优先产品池

提升爆品总量时，不要平均用力。先从四类产品里筛：

1. 准爆品：离爆品门槛最近，最适合短期冲刺。
2. 实力优品：已有产品质量和基础数据，资源承接能力更强。
3. 平台新品：新品期有流量扶持，可用 1-2 周数据判断潜力。
4. 其他优品：已有平台质量认可，但需要进一步补数据。

筛选时可进入产品参谋或商品运营工作台，按最近 7 天或 30 天的询盘、TM、点击、订单、买家数排序，找“数据已有起势但曝光不足”的产品。

## 十个提升动作

课程将爆品提升拆成一组可组合动作。并不是每个店铺都要全部执行，而是根据产品当前短板选择动作。

| 动作 | 适用场景 | 关键判断 |
| --- | --- | --- |
| 直通车、顶展、关键词计划 | 产品基础好但曝光不足。 | 投放词要与产品、目标国家和页面承接匹配。 |
| 爆品助推或优爆品助推 | 准爆品、实力优品离门槛不远。 | 用资源补最后一段数据，不要推低质量产品。 |
| 新品加速 | 平台新品或近 30 天新发品。 | 观察 1-2 周询盘/TM/点击趋势。 |
| 普通品推广计划 | 普通优品需要先补初始数据。 | 先验证点击和询盘，不要长期高消耗无商机。 |
| 真实订单线上化 | 线下已有真实客户和履约能力。 | 引导真实业务进入信保或平台链路，不能做虚假交易。 |
| 蓝海产品开发 | 有上涨词、细分需求或竞品供给少。 | 用关键词趋势和前台验证市场需求。 |
| 市场迫切需求产品 | 行业短期需求明确、转化快。 | 页面和交付要能立即承接。 |
| 访客营销、EDM、RFQ | 已有访客、老客或 RFQ 商机。 | 用真实买家触达补充初始商机。 |
| 活动、折扣、场景报名 | 产品符合平台活动或场景规则。 | 活动前先确认库存、价格和履约。 |
| 直播、短视频绑定产品 | 产品适合内容展示或场景讲解。 | 绑定主推品，避免内容和产品脱节。 |

## 合规边界

原始课程中出现了“补询盘”“补订单”“补买家账号”等口语表达。正式知识库只保留合规理解：把真实客户、真实订单和真实业务沟通引导到平台完成询盘、TM、信保交易或合法营销触达。

不能把虚假询盘、虚假交易、刷单、操控买家账号或规避平台规则作为运营动作。爆品提升要和违规处罚规则一起使用，任何可能触发处罚的做法都应先停止。

## 优品权益与基础门槛

优品、爆品、国家甄选品、行业服务力品、内容竞争力品和价格竞争力品会获得不同权益，例如搜索、推荐、活动、场景、站外和资源使用机会。权益名称和展示位置会随后台变化，实操时要看当前商品运营工作台的权益说明。

课程特别强调三个基础门槛：

| 基础门槛 | 课程口径 | 含义 |
| --- | --- | --- |
| 近 90 天发货率 | 不低于 90% | 履约能力不过关会拖累优品和爆品。 |
| 近 90 天信保退款申请率 | 低于 10% | 交易风险高会影响产品质量判断。 |
| 商品评价分 | 不低于 4 分 | 买家评价差会削弱产品权益和转化。 |

如果这些底线指标没有达标，即使产品标题、图片和推广做得好，也可能无法稳定进入优品或爆品池。

## 运营节奏

建议用周节奏维护爆品池：

```text
每周查看准爆品和平台新品
-> 找离门槛最近的产品
-> 分配推广、营销、活动或内容资源
-> 观察 7 天点击、询盘、TM、订单和买家数
-> 晋级产品继续维护，失败产品换池测试
```

爆品总量的本质是梯队运营：平台新品负责发现机会，实力优品负责稳定承接，准爆品负责短期冲刺，爆品负责拉动店铺流量和权重。
