# 店铺规划与定位方法

CH21-CH22 把店铺运营从“做动作”提升到“先定方向”。课程借用“先胜而后求战”的逻辑：好的店铺规划能缩短成长周期，节省人力、时间和直通车预算，让后续动作都朝同一个方向积累。

店铺定位不是一句口号，而是行业、产品、人群、国家和店铺类型五个维度共同决定的经营方向。

## 为什么先做定位

| 定位价值 | 课程含义 | 如果缺失会怎样 |
| --- | --- | --- |
| 找到差异化打法 | 明确行业、产品、客户和国家，不和同行盲目同质化竞争。 | 操作没有主线，今天推 A、明天推 B，数据无法沉淀。 |
| 提升询盘质量 | 买家属性、地区属性和客户类型更精准。 | 询盘有量但质量差，业务转化困难。 |
| 决定推广策略 | 直通车应该投什么产品、投哪些国家、匹配什么人群。 | 预算分散，推广成本高，难以验证模型。 |
| 对齐公司资源 | 运营方向要和供应链、销售、产品优势一致。 | 运营拿不到有效素材，销售承接也不顺。 |
| 形成正向积累 | 店铺标签、产品标签、国家标签和成交数据同向增长。 | 标签混乱，推荐流量和自然流量不精准。 |

一句话：在正确方向上，用正确方法实现店铺运营目标，并持续正向积累。

## 五个定位维度

| 维度 | 关键问题 | 操作要点 |
| --- | --- | --- |
| 行业定位 | 主营类目是否正确？行业竞争和平台权重如何？ | 主营类目一年调整次数有限，变更会影响原有数据和标签，不要轻易跨类目跳转。 |
| 产品定位 | 店铺主推什么产品线？哪些是引流品、活动品、利润品、主推品？ | 选择 1-5 条核心产品线，结合市场需求、关键词指数、竞争度和公司供应链优势。 |
| 人群定位 | 产品卖给谁？他们接受什么价格、MOQ、风格和功能？ | 从 Amazon 评论、业务沟通、国际站同行评价和询盘记录里提炼痛点。 |
| 国家定位 | 哪些国家需求强、成交概率高、物流和清关匹配？ | 用历史成交、询盘国家、市场参谋和买家参谋收敛目标国家。 |
| 店铺类型定位 | 店铺更像定制、RTS 现货、品牌，还是混合型？ | 店铺类型决定流量入口、详情页表达、产品数量结构和订单绑定方式。 |

## 行业定位

行业定位决定流量天花板、竞争强度和系统匹配方向。课程特别强调，主营类目不能随意改，因为类目一旦变化，原有的人群、国家和店铺权重都可能被削弱。

判断行业时看四件事：

1. 行业是否有足够市场需求和增长空间。
2. 平台是否给该类目足够流量和资源位置。
3. 公司供应链、价格、交付和产品能力是否能支撑。
4. 关键词和产品池是否能让店铺较快切入。

工厂型店铺通常行业更清楚；贸易型店铺更需要用数据判断行业和产品组合。

## 产品定位

产品定位要回答：店铺到底靠哪些产品拿流量、拿询盘、拿利润和拿转化。

课程建议把产品分成四类：

| 产品角色 | 作用 | 选择依据 |
| --- | --- | --- |
| 引流产品 | 获取曝光和点击，打开店铺流量。 | 搜索量、价格吸引力、买家关注度。 |
| 活动产品 | 配合平台活动、榜单、场景流量。 | 活动适配度、供货稳定性、价格竞争力。 |
| 利润产品 | 承接高质量客户和订单利润。 | 毛利、交付能力、复购或定制空间。 |
| 主推产品 | 资源集中投入，形成爆品或优品。 | 公司优势、关键词容量、成交概率和竞争强度。 |

不要平均用力。店铺里可以有很多产品，但推广资源应集中给少数能形成突破的产品线。

## 人群定位

人群定位决定页面怎么写、主图怎么拍、详情页怎么讲、业务怎么跟进。

可从这些来源提炼买家画像：

- 公司历史客户和线下成交资料。
- 后台询盘、TM 和订单买家。
- Amazon 等平台评价里的高频痛点。
- 国际站同行产品评论和详情页表达。
- 业务团队和客户沟通记录。

提炼后要回写到旺铺首页、详情页、直通车推广计划和业务谈判话术里。买家关心的问题没有被页面呈现，访客就容易只浏览不询盘。

## 国家定位

目标国家不是越多越好。课程提醒，全球都能卖不等于全球都值得投。有限预算下，应优先找到需求强、成交概率高、公司履约匹配的国家。

判断目标国家时看：

1. 哪些国家有强需求和增长。
2. 当地偏好什么款式、价格、MOQ 和交期。
3. 哪些国家历史成交更容易。
4. 后台询盘和高转化国家是否集中。
5. 市场容量、竞争强度和新增机会是否足够。

国家定位会影响直通车投放、人群标签和推荐流量。如果长期投向不匹配国家，点击和询盘可能存在，但转化和店铺标签会变差。

## 店铺类型定位

课程把店铺类型和产品类型结合来看：定制型、RTS 现货型、品牌型或混合型店铺，对应不同流量入口和页面表达。

| 店铺类型 | 运营重点 | 产品结构建议 |
| --- | --- | --- |
| 定制型店铺 | 强调生产能力、方案能力、公司实力和定制服务。 | 定制产品占主导，RTS 产品做补充，可参考 7:3。 |
| RTS 现货型店铺 | 强调热卖款、订单数、评价、买家地区和快速交易。 | RTS 产品占主导，定制产品做补充，可参考 7:3 反向配置。 |
| 混合型店铺 | 先确定当前阶段主打法，不让系统识别混乱。 | 避免定制和 RTS 平均用力。 |

店铺类型还会影响详情页结构、订单绑定、活动报名和系统匹配的买家层级。若店铺主打定制，却长期只推广 RTS 产品，询盘质量和标签精准度都会出问题。

## 落地流程

1. 写出行业、产品、人群、国家、店铺类型五个定位结论。
2. 选择 1-5 条主推产品线，并标注引流品、活动品、利润品、主推品。
3. 建立目标国家列表，先用数据圈定，再用成交结果收敛。
4. 把买家画像转成首页、详情页、主图、直通车计划和业务话术。
5. 按月和周拆成动作，持续看曝光、点击、询盘、订单和标签变化。

## Agent 使用提示

诊断店铺规划时，不要只问“流量为什么少”。先问店铺是否已经明确五个定位。如果五个定位缺失，SEO、直通车和详情页优化都会变成局部修补。
