# 竞争对手店铺与产品分析

竞争对手分析不是为了照搬同行，而是为了知道优秀店铺在卖什么、主推什么、靠什么拿流量和转化，再反推自己店铺的方向、产品和动作。

课程强调：对标对象必须足够相似。类目、产品款式、属性、目标市场和买家类型越接近，对标价值越高。差异过大的店铺只能提供参考，不能直接当作经营标准。

## 找到相似优秀同行

可用以下路径建立同行店铺表：

1. 插件或第三方工具：课程提到可用商品/店铺分析类工具辅助找同行，但工具名称和权限需以当前可用版本为准。
2. 平台榜单：从产品榜单、行业榜单或活动榜单进入产品，再进入店铺。
3. 制造商入口：观察制造商入口或工厂类板块中排名靠前的店铺。
4. 前台自然搜索：用核心关键词搜索，筛选自然排名好、星级高、产品相似的店铺。
5. 高投入店铺：搜索结果中可观察高年框、高投入或高资源位店铺，但仍要先过滤产品和买家是否相似。

建议建立表格，记录店铺链接、主营类目、主推产品、星级、店铺类型、主要国家、交易数据、询盘线索、橱窗产品和可学习点。

## 店铺维度分析

分析同行店铺时，优先看自己可以改变的指标。店铺年限这类不可改变信息可作背景，不应成为主要动作；星级、产品结构、橱窗、主推产品、页面承接和推广结构才是可操作项。

| 维度 | 看什么 | 如何使用 |
| --- | --- | --- |
| 星级和店铺层级 | 星级、能力分、店铺类型。 | 判断自己在平台分层上的差距。 |
| 经营状态 | 交易金额、近 6 个月订单、主要市场。 | 判断同行是否真的有结果。 |
| 店铺赛道 | RTS、定制、半托管或混合结构。 | 避免用定制店铺直接对标 RTS 流量。 |
| 主营产品 | 首页产品、产品组、橱窗、主营认证、活动产品。 | 识别同行真正重点推的产品。 |
| 页面表达 | 首页、详情页、证书、案例、服务和询盘入口。 | 反推同行如何建立信任和转化。 |

同行首页和产品组里反复出现、排序靠前、绑定橱窗或主营认证的产品，通常是重点产品。不要只看店铺“有什么”，要看店铺“把什么放在最重要位置”。

## 高转化产品来源

课程给出多条找高转化产品的路径：

| 来源 | 用法 |
| --- | --- |
| 市场参谋商机分析 | 从产品榜单和效果榜看行业热品。 |
| 买家参谋 | 看买家行业偏好产品和相关榜单。 |
| 平台榜单 | 找订单、询盘或热度较好的产品。 |
| 优秀同行店铺 | 看同行产品分组、首页、橱窗和主营产品。 |
| 汇采购/活动产品 | 活动门槛较高，能进入的产品往往有一定数据基础。 |

找产品时只对标同类产品。同类目但不同风格、不同买家、不同价格带的产品，不能直接得出“我也应该这样做”的结论。

## 产品维度分析

对同行高转化产品，需要拆到能落地的字段：

| 字段 | 诊断问题 |
| --- | --- |
| 款式和属性 | 热销点是外观、功能、材质、规格还是应用场景。 |
| 价格和 MOQ | 买家预期价格带和起订量是否与自家产品冲突。 |
| 询盘数和订单线索 | 该产品要达到怎样的数据量才可能有排名或转化优势。 |
| 主要销售国家 | 哪些国家更适合推广，哪些国家不必优先烧预算。 |
| 标题和关键词 | 关键词是否和目标国家、类目、款式匹配。 |
| 主图和详情页 | 买家第一眼看到什么，信任证据如何组织。 |
| 优惠和物流 | 是否通过样品、优惠券、折扣、物流承诺降低行动门槛。 |

课程中涉及“需要多少询盘”的表述，应理解为用真实询盘量判断竞争强度和排名门槛，不是制造虚假询盘。所有对标动作都应遵守平台规则。

## 输出自己的行动表

竞争对手分析最后要输出动作，而不是只保存链接。

```text
确定相似优秀同行
-> 拆店铺结构和主推产品
-> 找高转化产品共性
-> 对比自己的产品、国家、关键词、价格和页面
-> 形成可执行差距清单
-> 设定 7/14/30 天验证指标
```

行动表可包含：

1. 要补的产品线或款式。
2. 要测试的目标国家。
3. 要加入或排除的关键词。
4. 要调整的主图、详情页、价格、MOQ 或样品策略。
5. 要加入橱窗、直通车、活动或重点首页展示的产品。
6. 要持续跟踪的同行和数据字段。
