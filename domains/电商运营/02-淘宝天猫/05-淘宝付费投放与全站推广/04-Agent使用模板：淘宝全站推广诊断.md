# Agent 使用模板：淘宝全站推广诊断

当用户询问淘宝/天猫全站推广、万相台、智能计划、投产目标、短视频拉新或打款放量问题时，先判断问题处在“产品基础、链接测试、拿成交标签、转投产、预算放大、拉新收割、内容补充”哪一层。

## 输入信息

尽量向用户索取：

1. 商品类目、客单价、毛利率、保本投产。
2. 当前链接数量、主图数量、是否同款多链接。
3. 近 7 天或 30 天的访客、成交、转化率、加购、收藏、投产。
4. 当前计划类型、目标、预算、消耗、成交金额、ROI。
5. 是否已有成交标签，成交人群是否符合预期。
6. 竞品访客、推广花费、价格、主图、销量和链接数。
7. 是否有短视频、直播、内容素材和收割计划。
8. 最近是否频繁改目标、改预算、切计划。

## 诊断顺序

### 1. 判断能不能打

- 市场有没有足够空间。
- 竞品是否证明需求。
- 产品基础是否能承接。
- 价格和毛利是否允许前期试错。

如果这里不成立，不要直接建议加预算。

### 2. 判断有没有给产品足够机会

- 是否只上了一个链接。
- 是否测试了不同场景、人群、主图、颜色、规格或套餐。
- 是否有素材点击率和成交数据。
- 是否有可以复用的好素材。

如果机会不足，建议先做链接发散和素材测试。

### 3. 判断当前阶段目标

| 阶段 | 主要目标 | 不应过早追求 |
| --- | --- | --- |
| 初期 | 成交、加购、人群标签 | 最终盈利 ROI |
| 中期 | 转目标投产、校正人群 | 大幅加预算 |
| 放量期 | 稳定预算和投产 | 频繁切换 |
| 上限期 | 短视频拉新、内容种草 | 只靠搜索或全站硬拉 |

### 4. 判断预算和目标是否匹配

- 如果没有标签，不建议直接高目标投产。
- 如果保本投产是 3，可以考虑先从较低目标起步，再逐步提高。
- 每次提高目标后观察 8-24 小时。
- 如果预算拉高后人群变差，检查目标、素材、收割和市场体量。

### 5. 判断拉新和收割是否成对

- 是否有计划负责前端拉新。
- 是否有计划承接加购、浏览、搜索或已触达人群。
- 是否排除近期已购买人群。
- 是否存在自己种草、竞品成交的风险。

## 输出结构

建议按这个格式回答用户：

```text
1. 当前问题判断：
   你现在不是/是预算问题，而是【产品基础/成交标签/人群偏差/素材点击/收割缺失/市场体量】问题。

2. 证据：
   用用户提供的数据说明为什么。

3. 当前阶段：
   初期拿标签 / 中期转投产 / 稳定放量 / 上限补内容。

4. 下一步动作：
   给出 3-5 个动作，按优先级排序。

5. 观察周期：
   指明看 8-24 小时、3 天、7 天还是一个完整周期。

6. 禁止动作：
   说明现在不应做什么，例如盲目加预算、频繁切换、刷单、只追 ROI。
```
