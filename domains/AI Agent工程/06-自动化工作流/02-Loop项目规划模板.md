# Loop 项目规划模板

## 用途

当 Peter 要求 Agent “做一个 loop”“循环优化”“多轮自动改进”“先跑几轮再给报告”时，Agent 应先使用本模板规划，而不是直接开始执行。

本模板不限制 loop 的业务范围。下面的代码工具、电商运营、聚宽量化和视觉制作只是 Peter 已提供或常用的案例，用来帮助 Agent 快速类比；任何新领域只要能定义目标、证据、执行、评审、回流和停止条件，都可以套用。

本模板的目标是避免三类失败：

- 没有证据面：Agent 跑了很多轮，但无法证明真的变好。
- 没有停止条件：循环变成无限继续或成本失控。
- 没有回写机制：跑完一次后，经验没有进入 [[domains/AI Agent工程/03-Skill设计/00-Peter的Skill世界观|Skill]]、wiki 或工具。

## Loop 规划表

| 模块 | 必填问题 | 输出要求 |
| --- | --- | --- |
| Loop 名称 | 这是什么循环，不要只写任务名 | `领域/对象 + 目标 + 循环类型` |
| 业务目标 | 最终要改善什么指标、资产、工具、报告或决策质量 | 一句话，可验证 |
| 成功证据 | 用什么文件、测试、截图、数据、回测或人工 checklist 证明 | 至少一个外部证据 |
| 初始输入 | 用户已经给了什么，Agent 还需要读取什么 | 文件、链接、工具、知识库 |
| 工作基座 | Agent 可以使用哪些 CLI、MCP、浏览器、账号、API、目录、平台 | 明确允许和禁止 |
| 执行节点 | 每轮生产什么 | 代码、数据、报告、策略、图片、文档 |
| 证据节点 | 每轮如何采集证据 | 命令、截图、导出、回测、日志 |
| 评审节点 | 谁从什么视角评审 | reviewer 角色、subagent、脚本、用户 |
| 回流规则 | 评审意见如何进入下一轮 | 补数据、修代码、改报告、换假设、回滚 |
| 轮数预算 | 最多几轮，何时提前停止 | 最大轮数、时间、token、API 成本 |
| 风险边界 | 哪些动作必须确认 | 删除、发布、授权、发消息、真实交易、真实下单 |
| 留痕方式 | 每轮记录在哪里 | `PLAN.md`、`ITERATIONS.md`、`EVIDENCE/`、`FINAL_REPORT.md` |
| 入库规则 | 哪些经验写回 wiki/skill | 主线、分支、补丁、QA、工具规则 |

## 示例模板

以下模板是案例，不是范围边界。遇到新任务时，Agent 应先抽象出对应的目标契约、证据面、执行节点、评审节点和停止条件，再创建新的 loop 模板。

### 代码工具 Loop

```text
目标契约：让 <CLI/工具名> 达到可被 Codex/Hermes/SealSeek 调用的稳定状态。
成功证据：真实命令运行成功、结构化输出可解析、错误路径可解释、测试通过。
每轮执行：实现或修复一个功能点。
每轮评审：从独立 Agent 调用者视角检查接口、文档、错误处理和安全边界。
停止条件：达到 N 轮、测试全部通过、连续两轮无高价值改进、遇到需要用户确认的 destructive 操作。
```

### 电商运营/DMP Loop

```text
目标契约：围绕 <商品/竞品/店铺> 生成可行动的竞品分析报告。
成功证据：原始数据、Excel、HTML 报告、运营评审 checklist。
每轮执行：发现工具或补采数据，生成或更新报告。
每轮评审：以运营负责人视角判断是否能支持投放、人群、搜索、产品或价格动作。
回流规则：如果报告问题来自数据缺口，回到采集节点；如果来自表达问题，回到报告节点。
停止条件：达到 N 轮、核心问题已回答、关键数据无法获得、需要用户登录/授权。
```

### 聚宽量化 Loop

```text
目标契约：在不破坏风险约束的前提下，提升 <策略名> 的质量指标。
成功证据：基线回测、同口径新回测、分年度表现、风险指标、交易/持仓分析。
每轮执行：提出一个可解释假设并修改策略。
每轮评审：检查收益改善是否来自真实假设而非过拟合、未来函数、成本忽略或单一年份运气。
回流规则：有效修改保留，无效修改回滚或记录为失败实验。
停止条件：达到 N 轮、连续无改善、风险指标恶化、回测口径不可靠。
```

### 视觉制作 Loop

```text
目标契约：生成符合品牌/电商目标的视觉资产。
成功证据：图片、PSD、OCR、尺寸检查、品牌规范 checklist、转化信息完整性。
每轮执行：生成或编辑视觉资产。
每轮评审：从电商转化、品牌一致性、可编辑性和平台规范检查。
回流规则：缺风格回到提示词，缺转化信息回到策划，技术不合格回到 PSD/图片处理。
停止条件：达到 N 轮、通过视觉 QA、素材限制无法突破、需要用户确认审美方向。
```

## 每轮记录格式

```md
## Round <n>

- 本轮目标：
- 执行动作：
- 产物：
- 证据：
- 评审结论：
- 问题等级：
- 决策：保留 / 修复 / 回滚 / 补数据 / 停止
- 下一轮计划：
```
