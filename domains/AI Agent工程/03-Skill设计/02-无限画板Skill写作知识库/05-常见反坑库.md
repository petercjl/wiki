# 无限画板 Skill 常见反坑库

本页记录写无限画板 skill 时最容易重复犯的错误。未来创建新 skill 前，应把这里作为强制自检清单。

## 1. 把无限画板 skill 写成 SealSeek Skill

错误表现：写目录结构、脚本、CLI 命令、README 交付流程、Python/Shell 执行逻辑。

正确做法：无限画板 skill 是单个 `skill.md`，只写给无限画板 AI 执行的 Markdown 流程和工具调用规则。

## 2. 第一轮直接生成完整 skill.md

错误表现：用户刚描述需求，Agent 直接输出完整长文。

正确做法：默认先规划，用户确认后再生成。只有用户明确要求直接生成时才跳过确认。

## 3. 虚构工具或参数

错误表现：写不存在的工具、给 `reference_to_image` 写 `num`、不写必填参数、让模型自动选择工具。

正确做法：只使用无限画板已知工具；生成工具必须显式写 `model`、`aspect_ratio`、`resolution`，视频工具还要写 `duration`、`generate_audio`。

## 4. 参考图生图时复述产品细节

错误表现：详细描述参考图里的产品像素、纹理、把手、形状，导致模型按文字重画产品。

正确做法：少写产品长什么样，多写产品以参考图为准、哪些关键结构不能变；场景和风格可以详细写。

## 5. 风格参考图覆盖产品事实

错误表现：为了迁移风格，改变产品品类、结构、材质、功能或卖点。

正确做法：风格只迁移光影、色彩、空间、版式、氛围；产品和事实以待优化图/产品图为准。

## 6. 营销优化编造事实

错误表现：为了提升转化，编造检测报告、认证、功效、销量、评价、医生推荐、品牌背书。

正确做法：只能把已有参数转译成用户利益，不能创造未经提供的事实。

## 7. 多图任务混成一张生成

错误表现：把多张待优化图一起传给一次生成，希望一次出多张或混合优化。

正确做法：每张图独立分析、独立生成、独立质检；多图通过多次调用或并行调用完成。

## 8. 虚构图片 token

错误表现：写 `[产品图_2]`、`[产品图_3]` 等未定义 token。

正确做法：不确定多张产品图编号时，用 `[产品图]` 代表产品参考图集合，并在说明中写“包含用户上传的全部商品参考图”。

## 9. edit_image 指令过长

错误表现：把“保持产品不变”的完整约束塞进 `instruction`，导致真正编辑目标不突出。

正确做法：`instruction` 只写要改什么；保真约束放在流程规则、前置分析和质检中。

## 10. 图位任务不清

错误表现：一张图同时承担主图、痛点、卖点、证据、规格、促销，导致画面和文案失焦。

正确做法：每张图只确定一个主要销售任务。

## 11. 只做视觉判断，不做营销判断

错误表现：只说“更高级、更干净、更好看”，没有判断购买理由。

正确做法：电商图片优化必须先判断销售任务和说服逻辑，再谈视觉风格。

## 12. 模型策略不固定

错误表现：未指定模型，或写“推荐使用某模型”，但工具参数里没有显式传入。

正确做法：图片默认 `gpt-image-2`，视频默认 `domains/视觉制作/domains/视觉制作/05-小红书风格AI生图/index/06-AI视频/01-Doubao-Seedance-2.0视频生成模型卡`，除非用户明确指定。

## 13. 缺少失败与回退机制

错误表现：没有说明风格图缺失、产品无法识别、文案无法识别、生成失败、产品变形时怎么办。

正确做法：复杂 skill 必须写停止条件、追问条件、重试/改用编辑/请求用户确认的规则。
