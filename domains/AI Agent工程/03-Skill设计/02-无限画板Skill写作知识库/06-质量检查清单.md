# 无限画板 Skill 质量检查清单

本清单用于新建或优化无限画板 `skill.md` 前后的自检。

## 创建前检查

- [ ] 是否已经明确用户目标、输入、输出和任务类型？
- [ ] 是否判断这是生图、图生图、图片编辑、生视频、图生视频、参考视频还是混合流程？
- [ ] 是否需要先规划并等待用户确认？
- [ ] 是否需要读取产品图、风格参考图或已有生成图？
- [ ] 是否存在无法区分图片角色、产品不可识别、关键文字不可识别等必须先问的问题？

## 结构检查

- [ ] `skill.md` 是否是 Markdown 单文件说明？
- [ ] 是否没有写脚本、Shell、Python 或外部命令？
- [ ] 是否没有套用 SealSeek/OpenClaw skill 的目录和工具体系？
- [ ] 是否包含触发条件、核心目标、输入要求、执行流程、工具选择规则、Prompt 模板、输出规范、设计约束、参数建议？
- [ ] 复杂任务是否包含批次规则、失败回退、质量检查、多轮优化协议？

## 工具检查

- [ ] 是否只写无限画板支持的工具？
- [ ] 图片工具是否显式写 `model: "gpt-image-2"`？
- [ ] 视频工具是否显式写 `model: "domains/视觉制作/domains/视觉制作/05-小红书风格AI生图/index/06-AI视频/01-Doubao-Seedance-2.0视频生成模型卡"`？
- [ ] 生成工具是否写 `prompt_zh` 或 `instruction_zh`？
- [ ] 是否显式写 `aspect_ratio` 和 `resolution`？
- [ ] 视频工具是否显式写 `duration` 和 `generate_audio`？
- [ ] `reference_to_image` 是否没有写 `num`？
- [ ] 多图是否通过多次/并行调用表达？

## 产品保真检查

- [ ] 有产品参考图时，是否明确产品以参考图为准？
- [ ] 是否禁止改变产品主体、材质、整体形状、比例和关键结构？
- [ ] 是否避免大段复述参考图产品细节？
- [ ] 是否没有虚构未定义 token？
- [ ] 复杂产品是否设置保护模式或人工确认点？

## 营销真实性检查

- [ ] 是否区分图位任务？
- [ ] 是否先判断销售任务，再做视觉优化？
- [ ] 是否把参数转译为购买理由，而不是堆功能？
- [ ] 是否禁止虚构认证、检测、专利、销量、评价、功效承诺？
- [ ] 图上文案是否短、清楚、有层级、来源可靠？

## 批量/详情页检查

- [ ] 多图是否分批？
- [ ] 每张图是否单独分析、生成和质检？
- [ ] 每屏详情页是否只有一个主要任务？
- [ ] 是否规定每批完成后询问是否继续？
- [ ] 是否避免把多张待优化图混成一张生成？

## 生成后 QA 检查

- [ ] 产品是否变形？
- [ ] 风格是否符合参考方向？
- [ ] 销售任务是否完成？
- [ ] 文案是否真实且无错别字？
- [ ] 是否存在不合规夸大或虚构事实？
- [ ] 是否需要重新生成、局部编辑或让用户确认？

## 输出检查

- [ ] 对话中是否只输出总结、路径、改动点和下一步？
- [ ] 是否避免默认粘贴完整 `skill.md` 长文？
- [ ] 若是优化模式，是否保留原有核心逻辑，只做相关最小改动？
