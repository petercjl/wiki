# 无限画板 Skill 高频任务范式

本页把历史 15 个无限画板 skill 中反复出现的任务结构抽象成可复用范式。写新 skill 时，不应从空白页开始，而应先判断任务属于哪一类。

## 1. 电商图片营销优化

适用：用户上传商品图、详情图、卖点图、主图，要求“优化、重做、提升转化、变高级”。

核心流程：

1. 读取原图和必要参考图。
2. 判断图位任务：主图、详情页首屏、痛点图、卖点图、证据图、场景图、规格图、促销图。
3. 先做营销判断，再做视觉判断。
4. 把参数/功能转译成购买理由。
5. 生图或编辑。
6. 质检产品一致性、营销成立性、文案真实性。

关键规则：每张图只能有一个主要销售任务，不要一张图塞太多卖点。

## 2. 风格迁移 + 产品保真

适用：用户提供待优化图和风格参考图，希望迁移某种品牌视觉，例如白色松弛、杉居、极简北欧、静谧剧场。

核心流程：

1. 区分待优化图和风格参考图。
2. 提取风格参考图的光影、空间、色彩、版式、氛围。
3. 提取待优化图的产品、文案、图位任务和必须保留事实。
4. 用 `reference_to_image` 同时传入产品图和风格参考图。
5. 明确“风格服务营销，不能覆盖产品事实”。

关键规则：风格参考图只迁移视觉语言，不改变产品、品类、功能边界和核心卖点。

## 3. 商品详情页规划与分批生图

适用：做 10–12 屏详情页、按屏规划、分批生成。

核心流程：

1. 先分析产品、人群、使用场景、核心卖点和品牌调性。
2. 规划整套详情页结构。
3. 每屏定义一个图位任务。
4. 按批生成，历史上常用每批 4 张。
5. 每屏单独 prompt、单独生图、单独质检。
6. 每批结束后询问是否继续下一批。

常见屏幕结构：首屏承诺、痛点唤醒、核心卖点、场景代入、细节证据、材质/工艺、使用步骤、规格尺寸、信任背书、结尾转化。

## 4. 主图生成

适用：为商品生成 5 张主图或主图组。

核心流程：

1. 识别产品和平台主图要求。
2. 规划 5 张主图的不同任务。
3. 第一张主图负责抢注意和最强购买理由。
4. 后续主图负责场景、卖点、规格、信任或差异。
5. 保持风格统一，但每张图任务不同。

关键规则：主图不是详情页压缩版，不能每张都重复同一个卖点。

## 5. 批量图片优化

适用：用户一次上传多张图片，要求统一风格或批量优化。

核心流程：

1. 区分风格参考图和待优化图。
2. 自动分批。
3. 每张图独立读取、独立判断、独立生成。
4. 统一风格语言，但保留每张图原有销售任务。
5. 批次完成后汇报结果并询问是否继续。

关键规则：不要把多张待优化图混成一个 reference_to_image 任务。

## 6. 电商主图视频

适用：将产品图、主图或场景图动态化，生成短视频。

核心流程：

1. 明确视频目标：展示质感、使用方式、场景氛围、卖点演示。
2. 选择 `image_to_video` 或 `reference_to_video`。
3. 写清主体、动作、镜头运动、光线、氛围。
4. 控制时长 4–15 秒。
5. 默认不生成音频。

关键规则：视频不是图片 prompt 的简单延长，必须写镜头和动作。

## 7. 成套电商视觉生成

适用：一套主图 + SKU/白底 + 详情页完整视觉。

核心流程：

1. 产品解读。
2. 场景库构建。
3. 分模块叙事。
4. 模块一：主图组。
5. 模块二：SKU/白底/规格图。
6. 模块三：详情页。
7. 每个模块先规划、用户确认后生成。
8. 风格语言在模块间统一传承。

关键规则：成套视觉必须有总风格和模块级任务，不是连续生成一堆相似图。

## 8. 产品设计生图

适用：基于产品参考图做造型优化、概念设计、外观重设计。

核心流程：

1. 属性解构：功能、结构、材质、比例、关键部件。
2. 边界确认：哪些可以设计，哪些不能动。
3. 产品保真锚点：必须保留的识别点。
4. 未展示部分不臆想。
5. 冲突参考图先说明冲突，再请求用户确认。
6. 复杂产品进入保护模式，优先保守设计。

关键规则：产品设计类 skill 比营销图更容易改坏产品，必须先定义设计边界。
