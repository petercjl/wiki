# 无限画板 Skill Prompt 模板库

本页沉淀历史 skill 中可复用的 prompt 结构。写新 skill 时应按任务选择模板，并根据产品、图位、风格和工具调整。

## 产品保真模板

适用：任何有产品参考图的 `reference_to_image`。

```text
产品以参考图为准，必须保持产品主体不变；不得改变产品的材质、整体形状、比例关系、结构位置、提手/绑带/连接件等关键部件；只允许调整背景、场景、光线、构图和氛围。
```

如果产品关键部件明确，应具体化：

```text
产品以参考图为准，产品主体不能变；材质不能变，整体形状不能变，表面纹理不能被重新设计；提手/绳带必须保留原有位置、数量、连接方式和比例；只允许改变背景、场景、光线和构图。
```

注意：这里的“纹理不能被重新设计”是约束，不是让模型重新描述纹理细节。

## 参考图生图模板

```text
将参考产品放入[目标场景]，营造[光线/氛围/品牌风格]，突出[图位任务/购买理由]。产品以参考图为准，产品主体不能变；材质、整体形状、比例和关键结构必须保持不变；只允许调整背景、场景、光线、构图和氛围。
```

错误写法：

```text
参考图里有一个白色杯子，杯子左边有把手，杯口是圆的……
```

原因：这会诱导模型按文字重新画一个类似产品，导致参考产品变形。

## 风格迁移模板

```text
基于待优化产品图生成一张[图位类型]。视觉风格参考[风格参考图]，迁移其[光影/色彩/空间/版式/氛围]，但不得改变产品主体、品类、结构、材质、核心卖点和原图本意。画面应体现[品牌调性]，并服务于[销售任务]。
```

风格参考图只能提供视觉语言，不能覆盖产品事实。

## 营销转化模板

```text
这张图的主要销售任务是[主图/痛点图/卖点图/证据图/场景图/规格图]。画面需要让用户快速理解：[购买理由]。文案应从产品参数转译为用户利益，表达清楚、克制、可信，不虚构认证、销量、评价、检测报告或功效承诺。
```

适合写进 skill 的规划和 prompt 生成规则，而不是每次原样塞进最终图像 prompt。

## 文案入图模板

```text
图上文案只保留[必要短句/核心卖点]，层级清晰，字数克制，避免大段说明。文案必须来自用户提供信息或原图可识别事实，不得编造数据、认证、功效、销量、评价或品牌背书。
```

如果模型对文字稳定性不足，应在 skill 中提醒：重要文字可先生成无字图，再用编辑或设计工具处理文字。

## 详情页单屏模板

```text
生成详情页第 [N] 屏，图位任务是[本屏任务]，核心购买理由是[理由]。画面以参考产品为主体，产品主体保持不变；场景为[场景]，光线为[光线]，构图为[构图]，整体风格为[风格]。图上文案为[短句]，文案真实、克制、有层级。
```

每屏只承担一个主要任务，避免“首屏 + 痛点 + 证据 + 规格”混在一张图里。

## 主图组模板

```text
为同一产品规划 5 张主图：
1. 第一张：抢注意 + 最强购买理由。
2. 第二张：核心场景代入。
3. 第三张：关键卖点可视化。
4. 第四张：规格/结构/材质信任。
5. 第五张：差异化或使用结果感。
每张主图风格统一，但销售任务不同。
```

## 图片编辑模板

`edit_image` 的 `instruction` 应短：

```text
把背景替换成浅灰色高级影棚背景，增强产品底部阴影。
```

不要写：

```text
保持产品形状、颜色、大小、角度、材质不变，只把背景替换成……
```

保留约束应主要放在流程规则或前置分析中，不要塞满编辑 instruction。

## 视频 Prompt 模板

```text
[主体和外观] 放在 [环境] 中，[动作或状态变化]，镜头 [推进/环绕/平移/拉近]，光线 [自然光/柔光/夜景]，整体氛围 [高级/治愈/安静/有活力]。
```

示例：

```text
一只透明玻璃香薰瓶放在木质桌面上，瓶中液体微微晃动，镜头从左侧缓慢推进到产品正面，背景是清晨柔和自然光和虚化绿植，整体氛围高级、安静、治愈。
```

## 批量处理规划模板

```text
本次识别到 [N] 张待优化图片，将分为 [M] 批处理；每批最多 [X] 张。每张图都会单独做输入识别、营销判断、风格迁移、生图和质检。不会把多张待优化图混合成一张生成任务。
```
