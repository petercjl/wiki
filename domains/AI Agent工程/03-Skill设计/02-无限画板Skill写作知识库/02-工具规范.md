# 无限画板 Skill 工具规范

本页整理写无限画板 `skill.md` 时可使用的工具、适用场景、关键参数和禁用写法。原则是：只写无限画板真实支持的工具和参数，不虚构能力。

## 图片生成工具

### `text_to_image`

适用：纯文生图，无参考图。

必填/显式参数：

- `prompt`：中文图片描述。
- `prompt_zh`：展示给用户的中文说明。
- `model`：默认 `gpt-image-2`，除非用户明确指定。
- `aspect_ratio`：必须显式传入。
- `resolution`：必须显式传入，通常 `2K`。

适合：风格概念图、无产品参考的海报氛围图、创意视觉探索。

### `reference_to_image`

适用：有参考图，尤其是商品图、风格参考图、元素融合、产品场景图。

必填/显式参数：

- `prompt`：只写目标场景/意图/约束，不要复述参考图像素。
- `prompt_zh`
- `reference_images`：用 `[产品图]`、`[风格参考图]`、`[生成图]` 等 token。
- `model`: `gpt-image-2`
- `aspect_ratio`
- `resolution`

硬规则：

- 每次生成一张图。
- 不写 `num` 参数。
- 多图通过多次调用或并行调用表达。
- 同一产品有多张参考图时，全部放入 `reference_images`；无法确定编号时，用 `[产品图]` 代表产品参考图集合。

### `edit_image`

适用：对已有图做局部或整体编辑，如换背景、删元素、改色、增强阴影。

关键参数：

- `instruction`：只写要改什么。
- `instruction_zh`
- `base_image_path`
- `reference_images`：可选。
- `model`: `gpt-image-2`
- `aspect_ratio`
- `resolution`: `1K` 或 `2K`。

禁忌：不要在 `instruction` 里大段写“保持产品不变”。编辑工具的指令应短而明确，否则容易稀释真正修改目标。

## 视频生成工具

### `text_to_video`

适用：纯文本生成视频。

关键参数：

- `prompt`
- `prompt_zh`
- `duration`：`domains/视觉制作/domains/视觉制作/05-小红书风格AI生图/index/06-AI视频/01-Doubao-Seedance-2.0视频生成模型卡` 支持 4–15 秒。
- `aspect_ratio`
- `model`: `domains/视觉制作/domains/视觉制作/05-小红书风格AI生图/index/06-AI视频/01-Doubao-Seedance-2.0视频生成模型卡`
- `resolution`: 默认 `720p`，高清可用 `1080p`。
- `generate_audio`: 必须显式传入，默认 `false`。

### `image_to_video`

适用：静态图动态化、首尾帧视频、商品主图视频。

关键参数：

- `first_frame_image`：必填。
- `last_frame_image`：可选。
- `reference_images`：可选。
- 其他同 `text_to_video`。

### `reference_to_video`

适用：多模态参考、角色一致、分镜板或风格一致的视频。

关键参数：

- `first_frame_image`：可选。
- `reference_images`：可选。
- `reference_videos`：可选。
- `duration`、`aspect_ratio`、`model`、`resolution`、`generate_audio` 必须明确。

视频 Prompt 推荐结构：

```text
主体与外观 + 动作 + 镜头运动 + 环境/光线 + 氛围
```

## 辅助工具

### `view_image`

有参考图、产品图、风格图时，制定 prompt 前应优先读取。

应提取：产品、文案、场景、卖点、图位类型、必须保留信息、比例、风格、敏感信息。

### `select_image`

当用户指代“第一张”“刚才那张”“蓝色那张”时，先选择目标图再编辑。

### `present_files`

用于呈现输出文件。文件必须位于无限画板允许的输出路径。不要把 SealSeek 本地路径规则写进无限画板 skill。

## 搜索与规划工具

### `generate_prompt_plan`

适用：复杂任务先规划，例如多图、多视频、多分镜、电商成套视觉、角色一致性、需求模糊但不适合追问。

### `web_search` / `web_fetch`

适用：搜索设计趋势、配色、字体、品牌规范、技术参数。不要用于图片搜索。

### `image_search`

适用：搜索视觉参考图，可约束方向、类型和是否下载首图。

### `generate_mindmap`

适用：把对话或规划转成思维导图。

## Token 规则

可用角色 token：

| Token | 含义 |
| --- | --- |
| `[产品图]` | 用户上传的主产品参考图或产品参考图集合 |
| `[风格参考图]` | 用户上传的风格参考图 |
| `[生成图]` | 最近一张生成图 |
| `[生成图_N]` | 第 N 张生成图，1-based |

禁止虚构 `[产品图_2]`、`[产品图_3]` 等未定义 token。
