# 电商图片营销策划与 AI 生图 Playbook

## 用途

当 Agent 需要为商品设计主图、详情页、场景图、证据图、促销图或 AI 生图提示词时，先使用本 Playbook。它把 [[domains/visual-production/concepts/marketing-theory-for-ecommerce-images|营销学理论在电商图片中的应用]] 转成可执行流程。

核心原则：先做营销判断，再做视觉判断，最后才写生图提示词。

## 输入信息

最少需要收集：

| 信息 | 需要回答的问题 |
| --- | --- |
| 产品 | 是什么品类，有哪些特征、规格、价格带 |
| 目标用户 | 谁会买，为什么需要，在哪个场景使用 |
| 竞品 | 用户会和谁比较，竞品强弱点是什么 |
| 卖点 | Feature、Advantage、Benefit、Evidence 分别是什么 |
| 图片任务 | 这张图要抢注意、讲痛点、讲卖点、给证据、做对比、建场景、讲规格还是促下单 |
| 平台位置 | 淘宝/天猫/抖音/小红书/拼多多，主图/详情页/内容种草/广告图 |
| 品牌约束 | 调性、价格感、禁忌元素、是否有固定风格库 |

## Step 1：定义用户和购买情境

先判断 STP，而不是直接列卖点。

输出格式：

```text
目标用户：【人群】
核心场景：【使用/购买/送礼/替换/囤货场景】
用户痛点：【不买或不用时的麻烦】
用户欲望：【买了之后更轻松/体面/安全/高效/愉悦】
用户顾虑：【怕没用/怕不值/怕质量差/怕不适合/怕麻烦】
竞品参照：【用户正在比较的替代选择】
```

判断标准：如果目标用户换了，主卖点和画面也应该随之变化。

## Step 2：用 FABE 提炼卖点

不要把参数直接写成图上文案。先把参数翻译成利益和证据。

| Feature 特征 | Advantage 优势 | Benefit 用户利益 | Evidence 证明证据 | 图片表达 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

转译例子：

| 原始信息 | 不合格表达 | 合格表达 |
| --- | --- | --- |
| 1L 容量 | `1L大容量` | `通勤一天不用反复接水` |
| 加厚 20% | `加厚升级` | `承重更稳，不易变形` |
| 1000W | `1000W大功率` | `早餐加热更快，少等几分钟` |
| 316L 不锈钢 | `316L材质` | `长期接触水也更安心，不易生锈` |

## Step 3：确定每张图的唯一任务

一张图只承担一个主要任务。整套图完成完整销售逻辑。

| 图片位置 | 首要任务 | 推荐内容 |
| --- | --- | --- |
| 主图 | 抢注意 + 给核心购买理由 | 大主体、强利益标题、一个 USP、清晰产品 |
| 图2 | 痛点唤醒 | 不使用的麻烦、旧方案不足、损失厌恶 |
| 图3 | 核心卖点 | FABE 中最强 Benefit |
| 图4 | 对比证明 | Before/After、我们 vs 普通款、升级前后 |
| 图5 | 场景代入 | 目标用户真实生活、使用后状态 |
| 图6 | 信任背书 | 数据、测试、认证、评价、材质、工艺 |
| 图7 | 规格选择 | 尺寸、颜色、容量、套餐、安装、适用边界 |
| 图8 | 行动推动 | 保障、赠品、套装价值、限时利益 |

## Step 4：选择视觉表达策略

| 营销目的 | 优先视觉策略 | 画面要点 |
| --- | --- | --- |
| 让用户觉得与自己有关 | 场景化 | 用户、空间、动作、生活方式必须匹配人群 |
| 让差异一眼可见 | 对比化 | 左右/上下/前后对比，减少长文解释 |
| 让用户相信 | 证据化 | 实测、报告、认证、口碑、过程，不只堆图标 |
| 让复杂信息好理解 | 结构化 | 爆炸图、剖面、局部放大、步骤、参数表 |
| 让用户想拥有 | 情绪化 | 光影、色彩、质感、身份感、礼物感、生活改善 |

## Step 5：制定视觉美学规则

视觉风格要服务定位和价格带。

| 维度 | 决策规则 |
| --- | --- |
| 色彩 | 促销用高对比，健康自然用绿色，科技工具用蓝/灰，高端用黑/白/金/低饱和，家居可用白色和自然色 |
| 光影 | 日用品偏高亮柔光，家居食品偏自然窗光，高端数码偏低调硬光，功能证明偏冷光/实测 |
| 构图 | 主体足够大，文案靠近主体，第一眼产品，第二眼利益，第三眼证据 |
| 字体 | 功能产品清晰稳定，高端产品克制精致，促销产品粗体大数字，母婴家居柔和亲和 |
| 材质 | 用特写、剖面、触感、参照物让用户看见价值 |

## Step 6：写图上文案

文案要短、准、有利益、有证据。

推荐结构：

```text
主标题：一句话购买理由
副标题：解释利益或场景
证据点：数据 / 测试 / 认证 / 评价
行动点：优惠 / 保障 / 适用人群
```

检查：

- 文案是否是用户语言，而不是商家语言。
- 文案是否能被画面证明。
- 主标题是否只讲一个重点。
- 数据是否有真实依据或明确来源。

## Step 7：生成 AI 生图提示词

先输出营销方案，再输出视觉方案，最后输出 prompt。

通用提示词结构：

```text
请为【产品】生成一张【平台/比例/图片位置】电商图片。
目标用户是【人群】，核心购买场景是【场景】。
本图的营销任务是【抢注意/讲卖点/做对比/建信任/促下单】。
核心卖点是【卖点】，用户利益是【利益】，需要通过【视觉证据】来表达。
画面应突出【主体】，第一眼看到【主视觉】，第二眼看到【关键信息】。
使用【色彩/光影/构图/材质】来传达【定位/情绪/价格感】。
图上文案控制在【字数】以内，主标题强调【购买理由】。
避免【禁忌元素】，不要出现【错误品类/错误材质/不合规符号/过度促销噪音】。
```

主图 prompt 追加：

```text
产品主体清晰且占画面主要比例；背景不抢产品；文案只保留一个核心 USP；画面必须让用户三秒内看懂卖什么、为什么点进去。
```

证据图 prompt 追加：

```text
用可视化测试、局部放大、参数标注或真实使用结果证明卖点；证据必须和主标题承诺一致；避免只堆无意义图标。
```

场景图 prompt 追加：

```text
场景必须匹配目标用户的真实生活；产品正在被自然使用；画面表达拥有后的状态，而不是只做漂亮背景。
```

## Step 8：最终 QA

| 检查项 | 合格标准 |
| --- | --- |
| 看得见产品 | 主体清楚，品类明确，不被背景和文案遮挡 |
| 看得懂卖点 | 核心利益一眼能懂 |
| 看得出差异 | 与竞品或旧方案有明确区隔 |
| 看得到证据 | 数据、测试、材质、评价或对比能证明承诺 |
| 看得进场景 | 目标用户能想象自己使用 |
| 看得出价值 | 质感、光影、色彩匹配价格带 |
| 看得放心 | 降低风险、顾虑和选择成本 |
| 看得想买 | 推动点击、继续阅读或下单 |

## 常见失败诊断

| 症状 | 可能原因 | 修正方式 |
| --- | --- | --- |
| 图片好看但不转化 | 缺少购买理由 | 回到 FABE，重写 Benefit 和 Evidence |
| 主图信息很满但点击低 | 卖点堆砌，USP 不清 | 只保留一个主卖点，其他放详情页 |
| 场景图氛围好但没用 | 场景与用户和卖点无关 | 重新定义人群、场景和心理试用 |
| 用户不信 | 只有承诺，没有证据 | 加测试、对比、认证、评价、材质细节 |
| AI 图漂亮但像样片 | prompt 只有风格词 | 加营销任务、用户利益、证据方式和平台位置 |
| 价格显贵 | 没有价值锚点 | 加长期成本、套装价值、升级差异或服务保障 |
