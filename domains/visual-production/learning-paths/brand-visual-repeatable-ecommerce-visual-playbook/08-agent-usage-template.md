# Agent 使用模板：电商品牌视觉标准诊断

## 先读哪些页面

根据任务选择：

- 团队反复改图、老板运营美工冲突：先读 [[01-visual-standardization-for-ecommerce-teams]]。
- 点击率、转化率、复购率问题：先读 [[02-three-visual-systems-click-conversion-repurchase]]。
- 品牌定位、价格战、人群不清：先读 [[03-brand-positioning-differentiation-and-unity]]。
- 建品牌视觉规范：先读 [[04-brand-visual-identity-standard-system]]。
- 提升品牌记忆点：先读 [[05-visual-memory-symbols-and-repetition]]。
- 摄影、包装、触点落地：先读 [[06-photography-packaging-and-touchpoint-standards]]。
- 需要案例类比：读 [[07-case-library-kabei-taobrands-and-small-sellers]]。

## 诊断输入

Agent 应尽量收集：

1. 店铺/品牌名称、类目、价格带、目标人群；
2. 当前主图、详情页、店铺首页、广告图、包装图；
3. 主要经营问题：点击低、转化低、客单低、复购低、团队效率低；
4. 现有品牌资产：LOGO、SLOGAN、主色、字体、摄影风格、包装、客服语言；
5. 竞品或标杆；
6. 当前平台和渠道：淘宝、天猫、抖音、小红书、京东、私域等。

## 诊断步骤

### Step 1：判断问题属于哪类视觉

| 问题 | 归因优先级 |
| --- | --- |
| 曝光后没人点 | 营销视觉 |
| 点进来不买 | 产品视觉 |
| 买了不记得、不复购 | 品牌视觉 |
| 能卖但价格撑不住 | 品牌视觉 + 产品视觉 |
| 图片产出慢、反复改 | 视觉标准 + 组织协作 |

### Step 2：检查 12 项品牌视觉标准

逐项判断：缺失 / 混乱 / 有但未统一 / 可保留 / 需要升级。

1. 名字
2. LOGO
3. SLOGAN
4. 色彩
5. 字体字号
6. 品牌活体
7. 品牌标签
8. 品牌摄影
9. 产品摄影
10. 品牌包装
11. 品牌故事
12. 品牌接触点

### Step 3：区分短期转化和长期品牌

输出时必须明确：

- 哪些动作能短期提升点击/转化；
- 哪些动作是长期品牌资产；
- 哪些短期动作可能损伤长期记忆；
- 哪些品牌动作不能牺牲卖货效率。

### Step 4：给出优先级

建议按这四层排序：

1. 影响最大指标的图位；
2. 最混乱、最易统一的标准项；
3. 可快速模板化的重复元素；
4. 需要老板决策的定位和记忆主符号。

## 输出结构

推荐输出：

```markdown
## 结论

## 当前问题归因
- 营销视觉：
- 产品视觉：
- 品牌视觉：
- 组织标准：

## 12 项品牌视觉标准诊断表

## 优先级建议
1. 立即修正：
2. 需要补素材/补拍：
3. 需要老板决策：
4. 可沉淀为模板：

## 视觉标准初稿
- 名字/LOGO/SLOGAN：
- 色彩/字体：
- 摄影/包装/触点：
- 标签/活动规范：

## 风险与复核
```

## 不得推断

Agent 不得在缺少证据时擅自断定：

- 当前平台真实排名规则；
- 广告工具最新投放机制；
- 某颜色一定提升转化；
- 某风格一定适合目标人群；
- 未提供图片时的具体视觉问题；
- 未提供数据时的点击率/转化率原因。

## 需要复核的当前平台信息

本书平台背景为 2018 年。涉及以下内容必须实时复核：

- 淘宝/天猫搜索、推荐、主图规范；
- 直通车、万相台、引力魔方等广告产品；
- 双11、618、聚划算等活动素材规则；
- 无线端详情页尺寸和交互规范；
- 当前用户评价、问大家、短视频、直播等触点规则。

## Prompt 模板

```text
你是电商品牌视觉标准诊断 Agent。请基于我提供的店铺图片、详情页、包装、客服话术和经营数据，按“营销视觉、产品视觉、品牌视觉、组织标准”四层诊断问题。

要求：
1. 先判断当前最大瓶颈对应点击率、转化率、复购率、客单价还是团队效率。
2. 用 12 项品牌视觉标准逐项检查：名字、LOGO、SLOGAN、色彩、字体字号、品牌活体、品牌标签、品牌摄影、产品摄影、品牌包装、品牌故事、品牌接触点。
3. 明确哪些是短期转化动作，哪些是长期品牌资产动作。
4. 输出可执行的优先级，不要只说“提升品牌感”。
5. 涉及当前平台规则时必须标注需要实时复核。
```
