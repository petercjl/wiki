# 信息流与搜索承接图片策略

小红书图片生成前，先判断内容场景：用户是在“闲逛”还是“找答案”。两种场景决定封面任务、信息密度和画面表达方式。

## 对比表

| 维度 | 信息流素材 | 搜索承接素材 |
| --- | --- | --- |
| 用户状态 | 没有明确搜索需求，边刷边看 | 已经带着问题、关键词或购买意向 |
| 核心任务 | 抓停、激发兴趣、制造代入 | 给答案、建立信任、辅助决策 |
| 画面重点 | 真实生活瞬间、弱营销、轻反差、兴趣点 | 品类清晰、关键词明显、要点可读、攻略感 |
| 标题方式 | 场景化、痛点化、“没想到/谁懂/原来” | 答案型、避坑型、选购型、清单型 |
| 信息密度 | 中低，避免像说明书 | 中高，3-5 个清晰要点可接受 |
| 风险 | 太美但没有信息、产品露出弱 | 太像硬广、信息太满、像廉价详情页 |

## 信息流素材 Prompt 骨架

```text
小红书信息流种草封面，竖版 3:4，真实生活场景，主体是【产品/人物/空间】，目标人群是【人群】，画面呈现【具体场景】里的自然使用瞬间，弱营销感，不像广告摆拍，有兴趣点和轻微反差，产品露出清晰但不硬，融入【热点/季节/生活方式】元素。画面明亮干净，一个主视觉，背景有少量真实生活细节，手机端一眼能看懂。中文大标题：【场景化标题】，标题突出【人群词/场景词/痛点词】，保留文字安全区。负面提示：不要电商详情页，不要强促销，不要品牌硬广，不要复杂拼贴，不要密集小字。
```

## 搜索承接素材 Prompt 骨架

```text
小红书搜索承接封面，竖版 3:4，主题是【品类/产品】，目标用户正在搜索【关键词】，画面直接回答【痛点/选购问题】，包含清晰的产品/方案展示和 3-5 个可读要点。风格真实、干净、可信，像专业博主整理的实用攻略，标题：【答案型标题】，副标题：【利益点/避坑点】，画面中出现【品类词】、【人群词】、【场景词】或【功效词】标签，手机端高可读性，适合收藏。负面提示：不要空泛氛围图，不要只拍美图不讲信息，不要错误中文，不要小字过多。
```

## 决策规则

- 如果选题是“种草生活方式、制造兴趣、激发点击”，优先用信息流素材。
- 如果选题是“怎么选、避坑、测评、清单、对比”，优先用搜索承接素材。
- 如果笔记同时要信息流点击和搜索承接，封面可用信息流的真实场景做主视觉，但标题和标签要覆盖搜索关键词。
