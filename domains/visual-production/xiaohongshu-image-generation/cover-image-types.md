# 小红书封面类型库

## 选择规则

先判断笔记的主任务，再选择封面类型：

| 封面类型 | 适合任务 | 核心画面 | 标题方向 |
| --- | --- | --- | --- |
| 痛点型 | 抓痛点、制造代入 | 问题场景或错误状态 | 别再、总踩坑、先看 |
| 结果型 | 展示理想状态 | 使用后变化或结果 | 终于、这才是、想要的效果 |
| 场景型 | 强调生活代入 | 真实场景中自然使用 | 这个场景离不开它 |
| 清单/攻略型 | 搜索承接、收藏 | 实拍 + 信息图模块 | 选购、避坑、先看这几点 |
| 对比测评型 | 建立判断和信任 | 左右/上下对比 | 到底怎么选、差别在这里 |
| 爆文首图型 | 强点击与内容统一 | 痛点/痒点/结果浓缩 | 一眼知道解决什么问题 |
| 关键词封面型 | 搜索覆盖 | 品类词 + 人群词 + 场景词 | 包含核心搜索词 |

## 痛点型封面

适合用户已经有明显困扰或踩坑风险的品类。画面要把“问题”可视化，但不要负面到让人反感。

```text
小红书爆款封面风格，竖版 3:4，真实生活场景，主体是【产品/问题场景】，面向【人群】，画面突出【具体痛点】，情绪真实但不夸张，明亮干净，高对比但不刺眼，主体清晰，背景少量生活细节，手机端一眼可读，顶部大字标题：【痛点标题】，副标题：【解决方向】，红色或黑色重点字强调，避免杂乱拼贴
```

标题公式：

- 【人群】别再乱买【品类】了
- 用了才知道，原来【痛点】可以这样解决
- 【场景】总踩坑？先看这张

## 结果型封面

适合功能效果、体验改善、状态提升类产品。结果必须可视化，不能只写口号。

```text
小红书种草封面，竖版 3:4，主体是【产品】和【使用后结果】，画面展示清晰可见的变化或理想状态，真实自然，不像广告大片，干净背景，柔和光线，质感高级，突出一个核心结果：【结果】，中文标题醒目，标题：【结果型文案】，保留文字安全区，移动端清晰
```

标题公式：

- 坚持【周期】，终于把【问题】改善了
- 【人群】想要的【结果】，重点看这几个
- 这才是【品类】该有的效果

## 场景型封面

适合强调生活方式、使用情境和即时体验的内容。

```text
小红书生活方式种草图，竖版 3:4，真实【场景】中正在使用【产品】，人物或手部动作自然，场景有代入感，适合【人群】，画面明亮、温暖、干净，产品可见且不生硬，突出【场景需求】，标题区域清晰，中文大标题：【场景标题】，整体真实、有用、可收藏
```

标题公式：

- 【场景】真的离不开它
- 【人群】的【场景】好物清单
- 这个【场景】用【品类】才舒服

## 清单/攻略型封面

适合搜索承接、选购避坑、合集推荐和收藏型内容。信息密度可以高一点，但必须手机端可读。

```text
小红书攻略封面，竖版 3:4，信息图和实拍结合，主体是【品类/产品集合】，排版清楚，有 3-5 个要点模块，白底或浅色背景，红色重点标注，黑色清晰中文标题，适合收藏，手机端可读，标题：【攻略标题】，包含关键词：【品类词】、【人群词】、【场景词】，整体像真实博主整理的实用攻略
```

标题公式：

- 【品类】选购避坑清单
- 【人群】买【品类】先看这 5 点
- 【场景】好用的【品类】怎么选

## 对比测评型封面

适合竞品对比、使用前后、不同方案选择。核心是可信和克制，不要夸大。

```text
小红书测评封面，竖版 3:4，左右或上下对比构图，展示【产品A】和【产品B/使用前后/不同方案】，清晰标签区分，真实测评感，桌面自然光，背景简洁，重点突出差异，中文标题醒目：【对比标题】，有小标签：【适合人群】、【核心差异】，画面可信、克制、不夸张
```

标题公式：

- 【产品A】和【产品B】到底怎么选
- 【品类】别只看价格，差别在这里
- 实测后才发现【核心差异】

## 爆文首图型

适合需要提高点击率的封面。关键是让标题和主图共同浓缩痛点、痒点或结果。

```text
小红书爆文首图，竖版 3:4，封面主题是【选题】，核心人群是【人群】，主图直接浓缩【痛点/痒点/结果】，标题和画面信息统一。画面包含一个强主视觉，关键词醒目，中文大标题：【标题】，重点词：【人群词/功效词/场景词】，画面不要复杂，第一眼能判断这篇笔记解决什么问题。风格真实利他，像真实博主经验分享，不像品牌广告。负面提示：不要说明书式卖点罗列，不要大量品牌名，不要营销感过强，不要牛皮癣式排版。
```

## 关键词封面型

适合搜索流量承接。关键词不应堆砌，而要自然出现在标题、标签和画面语义中。

```text
小红书关键词封面，竖版 3:4，围绕【核心品类词】生成一张适合搜索承接的笔记首图。画面中自然体现【品牌/产品词】、【特征词】、【类目词】、【场景词】、【人群词】中的 3-4 类信息。目标人群是【人群】，需求是【需求/痛点】，场景是【场景】。标题：【包含关键词的标题】，标签文字：【关键词1】、【关键词2】、【关键词3】。画面真实、清晰、可信，像用户正在找答案时会点开的封面。
```
