# 07｜ABC 视觉分层系统

## 核心判断

品牌视觉不能只按“好看/不好看”判断，而要拆成 A、B、C 三层：A 层建立品牌态度和信任，B 层建立使用场景和向往感，C 层解释功能卖点和购买理由。

这套分层让详情页、主图、内容素材和广告画面从堆信息变成有秩序的沟通系统。

## A 层：品牌态度

A 层回答“这是一个什么层级的品牌、为什么值得信任、用户购买它是否有身份或心理收益”。

A 层通常承载：

1. 品牌理念和主张。
2. 品牌历史、背书、规模、渠道信任。
3. 调性、审美、价格感和身份感。
4. 品牌升级时需要传递给经销商、合作方或高价值用户的信心。

A 层不一定是宏大场景，也不一定必须出现在每一张图里。它的关键是让用户或渠道相信品牌值得被认真对待。例如高铁广告、城市大屏、品牌片、旗舰店首屏，都可能承担 A 层功能。

## B 层：使用场景

B 层回答“用户在什么生活场景中使用它，使用后获得什么感受”。

B 层通常承载：

1. 使用空间，如卧室、厨房、运动场、办公室、户外。
2. 使用人物，如妈妈、新中产女性、宠物主人、运动女孩、职场人。
3. 使用状态，如轻松、专业、放松、被照顾、效率提升。
4. 场景结果，如孩子愿意吃、宠物更健康、睡眠更舒适、穿着更有活力。

B 层的重点不是拍一个背景，而是把产品放回用户真实想象的生活里。它负责连接“产品功能”和“用户向往”。

## C 层：功能卖点

C 层回答“这个产品具体好在哪里，凭什么现在购买”。

C 层通常承载：

1. 功能点：材质、结构、参数、成分、工艺、技术。
2. 证明点：实验、对比、认证、数据、用户证据。
3. 行动点：规格、优惠、组合、适用人群、购买引导。

电商运营最容易使用 C 层，因为它最接近转化。但如果页面只有 C 层，品牌会变得像功能清单；如果只有 A 层和 B 层，用户又可能不知道为什么要买。

## 详情页编排

常见的详情页结构是头尾放 A，中段用 B 和 C 交替展开：

1. 首屏用 A 或 A+B 建立品牌感和整体信任。
2. 中段用 B 承接用户生活场景，再用 C 解释功能证据。
3. 尾段回到 A，强化品牌承诺、服务保障或长期信任。

这个结构不是固定模板。判断是否需要 A 层，取决于用户关系和购买阻力。

例如一个高端小家电品牌，如果主要购买者是老用户，用户已经信任品牌，详情页可以减少 A 层，把更多空间留给 B+C：场景和功能解释比重复品牌历史更有效。

## 单图测试

ABC 也可以用于主图和广告图测试。不要只测试不同版式，而要测试不同沟通层级：

1. A 图：品牌背书、调性、信任和价格感更强。
2. B 图：人物、场景、使用感更强。
3. C 图：功能、参数、利益点更强。

如果 A 图点击更高，说明用户对品牌层级和信任更敏感；如果 B 图更高，说明场景想象是购买入口；如果 C 图更高，说明用户处于明确需求和功效对比阶段。

## 给供应商的意义

ABC 分层能让品牌方更准确地向摄影、设计、AI 生图、短视频和详情页供应商下 brief。

错误 brief 通常是“高级一点、年轻一点、视觉冲击强一点”。有效 brief 应该明确这张图承担 A、B、C 中哪一层，哪些信息必须出现，哪些情绪必须建立，哪些元素不能破坏既有品牌记忆。

当团队共用 ABC 语言后，视觉讨论会从审美偏好回到经营目标。
